Question 320 / 361

You are developing an application that runs on a Compute Engine VM. The application needs to access data stored in Cloud Storage buckets in other Google Cloud projects. The required access to the buckets is variable. You need to provide access to these resources while following Google-recommended practices. What should you do?
  • A. Limit the VMs access to the Cloud Storage buckets by setting the relevant access scope of the VM.
  • B. Create IAM bindings for the VM's service account and the required buckets that allow appropriate access to the data stored in the buckets.
  • C. Grant the VM's service account access to the required buckets by using domain-wide delegation.
  • D. Create a group and assign IAM bindings to the group for each bucket that the application needs to access. Assign the VM's service account to the group.
Show Suggested Answer Hide Answer
Suggested Answer: B

Wyjaśnienie

Odpowiedź: B.

Wyjaśnienie (Dlaczego poprawna):
B. IAM bindings for the VM's service account: To jest 'Google-recommended practice' dla dostępu cross-project. Maszyna VM posiada swoją tożsamość (Service Account). Aby mogła czytać z bucketu w innym projekcie, administrator tamtego projektu musi dodać to konto serwisowe do polityki IAM bucketu z odpowiednią rolą (np. `storage.objectViewer`). Jest to bezpieczne, audytowalne i precyzyjne.

Dlaczego inne są błędne:
A. Access scope: To starszy mechanizm (legacy), który nie zastępuje IAM i jest mniej granularny.
C. Domain-wide delegation: Służy do impersonacji użytkowników Workspace (np. Drive), nie ma zastosowania do dostępu VM do Cloud Storage w innym projekcie GCP.
D. Create a group: Można tak zrobić dla wielu maszyn, ale dla pojedynczej aplikacji/VM bezpośrednie przypisanie roli do jej SA (B) jest najprostszym i najdokładniejszym rozwiązaniem.
