Question 168 / 361

Your security team wants to implement a defense-in-depth approach to protect sensitive data stored in a Cloud Storage bucket. Your team has the following requirements: - The Cloud Storage bucket in Project A can only be readable from Project B. - The Cloud Storage bucket in Project A cannot be accessed from outside the network. - Data in the Cloud Storage bucket cannot be copied to an external Cloud Storage bucket. What should the security team do?
  • A. Enable domain restricted sharing in an organization policy, and enable uniform bucket-level access on the Cloud Storage bucket.
  • B. Enable VPC Service Controls, create a perimeter around Projects A and B, and include the Cloud Storage API in the Service Perimeter configuration.
  • C. Enable Private Access in both Project A and B's networks with strict firewall rules that allow communication between the networks.
  • D. Enable VPC Peering between Project A and B's networks with strict firewall rules that allow communication between the networks.
Show Suggested Answer Hide Answer
Suggested Answer: B

Wyjaśnienie

Odpowiedź: B.

Wyjaśnienie (Dlaczego poprawna):
B. VPC Service Controls (Perimeter): To jest najsilniejsza odpowiedź na wszystkie trzy wymagania. Obwód VPC SC fizycznie uniemożliwia dostęp do API Cloud Storage spoza zdefiniowanych projektów (A i B) i sieci, nawet przy posiadaniu kluczy. Blokuje również operację kopiowania danych między bucketem wewnątrz obwodu a bucketem na zewnątrz (data exfiltration protection).

Dlaczego inne są błędne:
A. Domain Restricted Sharing: Chroni przed udostępnianiem nowym osobom, ale nie blokuje dostępu sieciowego ani kopiowania danych przez uprawnionych użytkowników.
C, D. VPC Peering/Private Access: To tylko routing sieciowy, nie chroni warstwy API usług Google przed nadużyciami.

Sugerowana odpowiedź: B

VPC Service Controls to jedyny mechanizm w GCP chroniący przed kradzieżą danych z usług 'managed' (API level protection).
