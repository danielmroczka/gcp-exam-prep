Question 247 / 361

You run a web application on top of Cloud Run that is exposed to the internet with an Application Load Balancer. You want to ensure that only privileged users from your organization can access the application. The proposed solution must support browser access with single sign-on. What should you do?
  • A. Change Cloud Run configuration to require authentication. Assign the role of Cloud Run Invoker to the group of privileged users.
  • B. Create a group of privileged users in Cloud Identity. Assign the role of Cloud Run User to the group directly on the Cloud Run service.
  • C. Change the Ingress Control configuration of Cloud Run to internal and create firewall rules to allow only access from known IP addresses.
  • D. Activate Identity-Aware Proxy (IAP) on the Application Load Balancer backend. Assign the role of IAP-secured Web App User to the group of privileged users.
Show Suggested Answer Hide Answer
Suggested Answer: D

Wyjaśnienie

Odpowiedź: D.

Wyjaśnienie (Dlaczego poprawna):
D. IAP on Application Load Balancer + IAP-secured Web App User: Cloud Run może być backendem dla globalnego Load Balancera (ALB). Włączenie IAP na tym Load Balancerze zapewnia centralne uwierzytelnianie (SSO) i autoryzację. Tylko użytkownicy z grupy 'privileged' posiadający odpowiednią rolę IAM zostaną przepuszczeni do aplikacji Cloud Run.

Dlaczego inne są błędne:
A. Cloud Run Invoker: Pozwala na autentykację service-to-service lub przy użyciu tokenów ID, ale IAP (D) oferuje znacznie lepsze doświadczenie dla użytkowników w przeglądarce (formularz logowania Google, obsługa sesji).
B. Cloud Run User: Ta rola pozwala na zarządzanie usługą, a nie na jej wywoływanie (do tego jest Invoker).
C. Ingress internal + Firewall: Cloud Run to usługa publiczna, nie można do niej stosować reguł firewall VPC w ten sam sposób co do maszyn VM.
