Question 119 / 361

You are a member of your company's security team. You have been asked to reduce your Linux bastion host external attack surface by removing all public IP addresses. Site Reliability Engineers (SREs) require access to the bastion host from public locations so they can access the internal VPC while off-site. How should you enable this access?
  • A. Implement Cloud VPN for the region where the bastion host lives.
  • B. Implement OS Login with 2-step verification for the bastion host.
  • C. Implement Identity-Aware Proxy TCP forwarding for the bastion host.
  • D. Implement Google Cloud Armor in front of the bastion host.
Show Suggested Answer Hide Answer
Suggested Answer: C

Wyjaśnienie

Odpowiedź: C.

Wyjaśnienie (Dlaczego poprawna):
C. Identity-Aware Proxy (IAP) TCP forwarding: To jest idealne rozwiązanie. IAP pozwala na tunelowanie ruchu SSH (TCP port 22) przez serwery Google. Inżynier loguje się do Google Cloud, a IAP przesyła jego ruch do maszyny bastion, która posiada TYLKO prywatny adres IP. Eliminuje to całkowicie ekspozycję bastionu na publiczny internet.

Dlaczego inne są błędne:
A. Cloud VPN: Rozwiązuje problem, ale wymaga zestawienia tunelu VPN po stronie inżyniera (trudniejsze dla osób 'off-site' na różnych łączach).
B. OS Login with 2SV: Bardzo dobra funkcja, ale sama w sobie wymaga, abyś mógł 'dosięgnąć' maszyny po IP (a maszyna nie ma publicznego IP).
D. Cloud Armor: Chroni HTTP(S), nie wspiera standardowo SSH (TCP 22) dla maszyn bez publicznego IP.

Sugerowana odpowiedź: C

IAP TCP Forwarding pozwala na bezpieczny dostęp administracyjny do zasobów prywatnych bez ryzyka skanowania portów przez boty z internetu.
