Question 322 / 361

You are responsible for managing identities in your company's Google Cloud organization. Employees are frequently using your organization's corporate domain name to create unmanaged Google accounts. You want to implement a practical and efficient solution to prevent employees from completing this action in the future. What should you do?
  • A. Create a Google Cloud identity for all users in your organization. Ensure that new users are added automatically.
  • B. Implement an automated process that scans all identities in your organization and disables any unmanaged accounts.
  • C. Register a new domain for your Google Cloud resources. Move all existing identities and resources to this domain.
  • D. Switch your corporate email system to another domain to avoid using the same domain for Google Cloud identities and corporate emails.
Show Suggested Answer Hide Answer
Suggested Answer: A

Wyjaśnienie

Odpowiedź: A.

Wyjaśnienie (Dlaczego poprawna):
A. Create Cloud Identity for all users + Auto-add: Najlepszym sposobem na uniknięcie ' Shadow IT' (kont osobistych na adresach firmowych) jest proaktywne stworzenie kont zarządzanych dla wszystkich pracowników. Kiedy adres email jest już zajęty przez konto zarządzane w Cloud Identity, pracownik nie może już użyć go do założenia konta osobistego. Automatyzacja tego procesu (np. przez synchronizację z HR/AD) gwarantuje, że problem nie powróci.

Dlaczego inne są błędne:
B. Disable unmanaged accounts: Nie można 'wyłączyć' konta, które nie należy do Twojej organizacji (Google chroni prywatność kont osobistych).
C. Register new domain: Bardzo kosztowne i uciążliwe dla brandingu i komunikacji firmy.
D. Switch email system: Podobnie jak C, jest to zbyt drastyczna zmiana biznesowa w odpowiedzi na problem techniczny, który można rozwiązać w chmurze (metoda A).
