Question 235 / 361

You are setting up a new Cloud Storage bucket in your environment that is encrypted with a customer managed encryption key (CMEK). The CMEK is stored in Cloud Key Management Service (KMS), in project "prj-a", and the Cloud Storage bucket will use project "prj-b". The key is backed by a Cloud Hardware Security Module (HSM) and resides in the region europe-west3. Your storage bucket will be located in the region europe-west1. When you create the bucket, you cannot access the key, and you need to troubleshoot why. What has caused the access issue?
  • A. A firewall rule prevents the key from being accessible.
  • B. Cloud HSM does not support Cloud Storage.
  • C. The CMEK is in a different project than the Cloud Storage bucket.
  • D. The CMEK is in a different region than the Cloud Storage bucket.
Show Suggested Answer Hide Answer
Suggested Answer: D

Wyjaśnienie

Odpowiedź: D.

Wyjaśnienie (Dlaczego poprawna):
D. CMEK in different region than bucket: W Google Cloud, aby użyć klucza CMEK do szyfrowania bucketu Cloud Storage, klucz MUSI znajdować się w tym samym regionie co bucket. Jeśli klucz jest w `europe-west3` (Frankfurt) a bucket w `europe-west1` (Belgia), Cloud Storage nie będzie mógł go użyć ze względów technicznych i wydajnościowych.

Dlaczego inne są błędne:
A. Firewall: KMS i Storage komunikują się przez wewnętrzne API Google, nie są blokowane przez standardowe reguły firewall VPC użytkownika.
B. HSM Support: Cloud Storage w pełni wspiera klucze KMS przechowywane w HSM.
C. Different project: GCP wspiera Cross-Project CMEK (klucz w jednym projekcie, zasób w innym), więc to nie jest przyczyną błędu.

Analiza i Podsumowanie Dyskusji (AI)

Analiza i Wyjaśnienie

Sugerowana odpowiedź: D

Zasada 'Lokalizacja klucza = Lokalizacja zasobu' jest kluczowa dla CMEK w wielu usługach GCP.
