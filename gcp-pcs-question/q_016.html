Question 16 / 361

An organization's typical network and security review consists of analyzing application transit routes, request handling, and firewall rules. They want to enable their developer teams to deploy new applications without the overhead of this full review. How should you advise this organization?
  • A. Use Forseti with Firewall filters to catch any unwanted configurations in production.
  • B. Mandate use of infrastructure as code and provide static analysis in the CI/CD pipelines to enforce policies.
  • C. Route all VPC traffic through customer-managed routers to detect malicious patterns in production.
  • D. All production applications will run on-premises. Allow developers free rein in GCP as their dev and QA platforms.
Show Suggested Answer Hide Answer
Suggested Answer: B

Wyjaśnienie

Odpowiedź: B.

Wyjaśnienie (Dlaczego poprawna):
B. IaC + static analysis: Przeniesienie kontroli na etap budowania (Shift Left) za pomocą Infrastructure as Code (np. Terraform) i automatycznych narzędzi do analizy statycznej (np. Terraform Validator) pozwala na automatyczne odrzucanie niezgodnych konfiguracji bez angażowania zespołu security w każde wdrożenie.

Dlaczego inne są błędne:
A. Forseti: Wykrywa błędy JUŻ PO ich wdrożeniu na produkcję (Detective control). Firma chce umożliwić wdrożenia przy jednoczesnym zachowaniu standardów, co lepiej adresuje prewencyjne podejście B.
C. Customer-managed routers: Wprowadza ogromny narzut wydajnościowy i operacyjny, co jest sprzeczne z celem 'enabling teams'.
D. Run on-premises: To ucieczka od problemu, a nie rozwiązanie go w chmurze.

Konsensus społeczności ExamTopics: Silne poparcie dla B. Automatyzacja polityk w pipeline'ie to klucz do nowoczesnego DevSecOps.

Analiza i Podsumowanie Dyskusji (AI)

Analiza i Wyjaśnienie

Sugerowana odpowiedź: B

Głęboka analiza: Statyczna analiza IaC pozwala na sprawdzanie reguł takich jak: 'czy ten firewall otwiera port 22 na cały świat?' jeszcze zanim jakikolwiek zasób powstanie w chmurze.
