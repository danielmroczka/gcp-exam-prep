Question 343 / 361

You manage the security logs within your cloud environment. You have configured a continuous export of security logs to Cloud Storage buckets for long-term retention. You need to provide auditors the ability to analyze the logs that were exported to Cloud Storage. Your solution must be cost-effective and quickly implemented. What should you do?
  • A. Use a VM instance to download the data from Cloud Storage. Provide the auditors access to the VM and allow them to install their preferred analytics toolset.
  • B. Use the data in the Cloud Storage bucket as an external table in BigQuery. Provide the auditors access to the BigQuery dataset.
  • C. Use Dataflow to import the data from Cloud Storage into Elasticsearch. Provide the auditors with access to Elasticsearch.
  • D. Use a Cloud Run job to import the log data from Cloud Storage to Cloud Logging. Provide the auditors access through Log Analytics.
Show Suggested Answer Hide Answer
Suggested Answer: B

Wyjaśnienie

Odpowiedź: B.

Wyjaśnienie (Dlaczego poprawna):
B. Cloud Storage as BigQuery External Table: To najszybszy i najtańszy sposób udostępnienia logów do analizy. Nie musisz fizycznie kopiować (importować) danych z GCS do BigQuery. Tworzysz 'External Table' w BigQuery wskazującą na pliki w GCS. Audytorzy mogą wtedy używać standardowego SQL do przeszukiwania logów, płacąc tylko za wykonane zapytania, bez kosztów podwójnego przechowywania danych.

Dlaczego inne są błędne:
A. Download to VM: Bardzo wolne, nieefektywne i trudne do audytowania (co audytor robi z danymi na tej maszynie?).
C. Dataflow to Elasticsearch: Bardzo skomplikowane wdrożenie i wysokie koszty utrzymania klastra Elastic.
D. Cloud Run to Cloud Logging: Niepotrzebna komplikacja; BigQuery (B) natywnie wspiera odczyt z GCS i jest standardowym narzędziem analitycznym.

Sugerowana odpowiedź: B

BigQuery Federated Queries (External Tables) pozwalają na analizę danych w GCS bez ich przemieszczania.
