Question 25 / 361

Your team needs to configure their Google Cloud Platform (GCP) environment so they can centralize the control over networking resources like firewall rules, subnets, and routes. They also have an on-premises environment where resources need access back to the GCP resources through a private VPN connection. The networking resources will need to be controlled by the network security team. Which type of networking design should your team use to meet these requirements?
  • A. Shared VPC Network with a host project and service projects
  • B. Grant Compute Admin role to the networking team for each engineering project
  • C. VPC peering between all engineering projects using a hub and spoke model
  • D. Cloud VPN Gateway between all engineering projects using a hub and spoke model
Show Suggested Answer Hide Answer
Suggested Answer: A

Wyjaśnienie

Odpowiedź: A.

Wyjaśnienie (Dlaczego poprawna):
A. Shared VPC (Host + Service projects): Shared VPC pozwala na oddzielenie administracji siecią (Host Project zarządzany przez network security team) od administracji zasobami (Service Projects dla deweloperów). Pozwala na centralne zarządzanie firewallami, subnetami i połączeniem VPN z jednego miejsca, udostępniając tylko subnety do projektów inżynierskich.

Dlaczego inne są błędne:
B. Compute Admin for all: Zbyt szerokie uprawnienia i brak centralizacji zasobów sieciowych.
C. VPC Peering hub-and-spoke: Trudniejsze w zarządzaniu i ma limity liczby połączeń peeringowych (nie jest to Shared VPC).
D. VPN between all projects: Bardzo skomplikowane i nieefektywne kosztowo.

Analiza i Podsumowanie Dyskusji (AI)

Analiza i Wyjaśnienie

Sugerowana odpowiedź: A

Shared VPC to 'enterprise-grade' rozwiązanie do zarządzania siecią w GCP, zapewniające spójność i centralną kontrolę.
