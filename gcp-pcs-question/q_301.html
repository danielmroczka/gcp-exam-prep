Question 301 / 361

Your organization is implementing separation of duties in a Google Cloud project. A group of developers must deploy new code, but cannot have permission to change network firewall rules. What should you do?
  • A. Assign the network administrator IAM role to all developers. Tell developers not to change firewall settings.
  • B. Use Access Context Manager to create conditions that allow only authorized administrators to change firewall rules based on attributes such as IP address or device security posture.
  • C. Create and assign two custom IAM roles. Assign the deployer role to control Compute Engine and deployment-related permissions. Assign the network administrator role to manage firewall permissions.
  • D. Grant the editor IAM role to the developer group. Explicitly negate any firewall modification permissions by using IAM deny policies.
Show Suggested Answer Hide Answer
Suggested Answer: C

Wyjaśnienie

Odpowiedź: C.

Wyjaśnienie (Dlaczego poprawna):
C. Create two custom IAM roles (Deployer vs Network Admin): To jest klasyczna implementacja zasady 'Separation of Duties'. Tworzysz rolę dla deweloperów, która zawiera tylko uprawnienia do wdrażania kodu (np. `compute.instances.setMetadata` dla skryptów, uprawnienia GAE lub GKE), oraz osobną rolę dla administratorów sieci z uprawnieniami do firewalli. Dzięki temu deweloperzy nie mogą sami otwierać portów w sieci.

Dlaczego inne są błędne:
A. Manual instructions: Zasady bezpieczeństwa muszą być wymuszone technicznie, a nie tylko 'powiedziane'.
B. Access Context Manager: Służy do kontekstowego dostępu do API (np. z konkretnego IP), a nie do granulacji uprawnień wewnątrz projektu między rolami dewelopera a admina sieci.
D. Editor role + Deny policies: Chociaż IAM Deny Policies istnieją, tworzenie precyzyjnych ról (C) jest znacznie czytelniejszą i standardową metodą ograniczania uprawnień w GCP.

Analiza i Wyjaśnienie

Sugerowana odpowiedź: C

Separacja obowiązków (Separation of Duties) poprzez dedykowane role IAM to podstawa bezpieczeństwa operacyjnego.
