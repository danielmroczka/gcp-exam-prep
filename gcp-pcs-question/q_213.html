Question 213 / 361

Your organization is using GitHub Actions as a continuous integration and delivery (CI/CD) platform. You must enable access to Google Cloud resources from the CI/CD pipelines in the most secure way. What should you do?
  • A. Create a service account key, and add it to the GitHub pipeline configuration file.
  • B. Create a service account key, and add it to the GitHub repository content.
  • C. Configure a Google Kubernetes Engine cluster that uses Workload Identity to supply credentials to GitHub.
  • D. Configure workload identity federation to use GitHub as an identity pool provider.
Show Suggested Answer Hide Answer
Suggested Answer: D

Wyjaśnienie

Odpowiedź: D.

Wyjaśnienie (Dlaczego poprawna):
D. Workload Identity Federation with GitHub: To jest najbezpieczniejsza metoda (keyless). Pozwala na uwierzytelnianie GitHub Actions w GCP bez użycia długowiecznych kluczy JSON. GitHub wystawia token OIDC, który GCP wymienia na tymczasowy token dostępu do konta serwisowego.

Dlaczego inne są błędne:
A, B. Service account keys: Przechowywanie kluczy JSON w repozytoriach lub konfiguracjach CI/CD jest uważane za niebezpieczne i niezalecane przez Google.
C. GKE Workload Identity: Służy do podów działających WEWNĄTRZ GKE, a nie do zewnętrznych systemów jak GitHub Actions (chyba że GitHub Actions runner działa wewnątrz Twojego GKE, ale WIF (D) jest ogólniejszym i właściwszym rozwiązaniem dla zewnętrznych platform CI/CD).
