Your company is deploying a large number of containerized applications to GKE. The existing CI/CD pipeline uses Cloud Build to construct container images, transfers the images to Artifact Registry, and then deploys the images to GKE. You need to ensure that only images that have passed vulnerability scanning and meet specific corporate policies are allowed to be deployed. The process needs to be automated and integrated into the existing CI/CD pipeline. What should you do?
  • A. Configure a policy in Binary Authorization to use Artifact Analysis vulnerability scanning to only allow images that pass the scan to deploy to your GKE clusters.
  • B. Enable Artifact Analysis vulnerability scanning and regularly scan images in Artifact Registry. Remove any images that do not meet the vulnerability requirements before deployment.
  • C. Implement a custom script in the Cloud Build pipeline that uses a third-party vulnerability scanning tool. Fail the build if vulnerabilities are found.
  • D. Configure GKE to use only images from a specific, trusted Artifact Registry repository. Manually inspect all images before pushing them to this repository.
Show Suggested Answer Hide Answer
Suggested Answer: A

Wyjaśnienie

Odpowiedź: A.

Wyjaśnienie (Dlaczego poprawna):
A. Binary Authorization + Artifact Analysis: To jest standardowy model 'Trusted Deployment' w GCP. 1. Artifact Analysis automatycznie skanuje obrazy w Artifact Registry. 2. Binary Authorization pozwala na zdefiniowanie polityki: 'Zezwalaj na deployment tylko jeśli obraz przeszedł skanowanie i nie posiada krytycznych luk'. Całość jest w pełni zintegrowana i zautomatyzowana.

Dlaczego inne są błędne:
B. Manual removal: Mało wydajne i nie gwarantuje, że obraz nie zostanie uruchomiony przed jego usunięciem.
C. Custom scripts: Zwiększają nakład pracy operacyjnej, gdy GCP oferuje gotowy system Binary Authorization.
D. Manual inspection: Skrajnie nieefektywne przy dużej skali wdrożeń kontenerowych.

Sugerowana odpowiedź: A

Binary Authorization to 'polityka jako kod' dla bezpieczeństwa obrazów kontenerowych.
