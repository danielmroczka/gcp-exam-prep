Question 358 / 361

Your organization has established a highly sensitive project within a VPC Service Controls perimeter. You need to ensure that only users meeting specific contextual requirements such as having a company-managed device, a specific location, and a valid user identity can access resources within this perimeter. You want to evaluate the impact of this change without blocking legitimate access. What should you do?
  • A. Establish a Context-Aware Access policy that specifies the required contextual attributes, and associate the policy with the VPC Service Controls perimeter in dry run mode.
  • B. Use the VPC Service Control Violation dashboard to identify the impact of details about access denials by service perimeters.
  • C. Configure a VPC Service Controls perimeter in dry run mode, and enforce strict network segmentation using firewall rules. Use multi-factor authentication (MFA) for user verification.
  • D. Use Cloud Audit Logs to monitor user access to the project resources. Use post-incident analysis to identify unauthorized access attempts.
Show Suggested Answer Hide Answer
Suggested Answer: A

Wyjaśnienie

Odpowiedź: A.

Wyjaśnienie (Dlaczego poprawna):
A. Context-Aware Access + Dry Run Mode: To jest kompletna odpowiedź. 1. Tworzysz politykę sprawdzającą stan urządzenia i lokalizację (Context-Aware Access). 2. Przypisujesz ją do obwodu VPC Service Controls. 3. Kluczowym krokiem jest 'Dry run mode' - pozwala on na sprawdzenie w logach, kto ZOSTAŁBY zablokowany, bez faktycznego blokowania nikogo. Pozwala to na walidację polityki przed wdrożeniem produkcyjnym.

Dlaczego inne są błędne:
B. Violation dashboard: To narzędzie do przeglądania wyników, ale samo w sobie nie 'ewaluuje wpływu zmiany' przed jej wprowadzeniem (musisz najpierw włączyć dry-run).
C. Enforce strict firewall: Firewalle nie widzą 'company-managed device attribute', to domena warstwy aplikacyjnej IAP/BeyondCorp.
D. Post-incident analysis: To podejście reaktywne, nie spełnia celu 'evaluate impact without blocking legitimate access' przed wdrożeniem.

Analiza i Wyjaśnienie

Sugerowana odpowiedź: A

Tryb 'Dry Run' w VPC Service Controls to niezbędne narzędzie dla bezpiecznego wprowadzania zmian w dużych organizacjach.
