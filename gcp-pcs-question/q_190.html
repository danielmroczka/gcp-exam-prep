Your organization is using Active Directory and wants to configure Security Assertion Markup Language (SAML). You must set up and enforce single sign-on (SSO) for all users. What should you do?
  • A. 1. Create a new SAML profile. 2. Populate the sign-in and sign-out page URLs. 3. Upload the X.509 certificate. 4. Configure Entity ID and ACS URL in your IdP.
  • B. 1. Configure prerequisites for OpenID Connect (OIDC) in your Active Directory (AD) tenant. 2. Verify the AD domain. 3. Decide which users should use SAML. 4. Assign the pre-configured profile to the select organizational units (OUs) and groups.
  • C. 1. Create a new SAML profile. 2. Upload the X.509 certificate. 3. Enable the change password URL. 4. Configure Entity ID and ACS URL in your IdP.
  • D. 1. Manage SAML profile assignments. 2. Enable OpenID Connect (OIDC) in your Active Directory (AD) tenant. 3. Verify the domain.
Show Suggested Answer Hide Answer
Suggested Answer: A

Wyjaśnienie

Odpowiedź: A.

Wyjaśnienie (Dlaczego poprawna):
A. SAML profile configuration steps: To jest poprawna kolejność konfiguracji SSO w Google Cloud Identity / Workspace: 1. Tworzysz profil SAML. 2. Podajesz adresy URL logowania/wylogowania Twojego IdP (Active Directory / AD FS). 3. Wgrywasz certyfikat publiczny IdP (do weryfikacji podpisów SAML). 4. Konfigurujesz po stronie AD dane Google (Entity ID i ACS URL), aby AD wiedziało, dokąd wysłać użytkownika.

Dlaczego inne są błędne:
B, D. OpenID Connect (OIDC): Pytanie specyficznie wymaga konfiguracji SAML.
C. Change password URL: Choć przydatne, nie jest krytycznym krokiem technicznym do samego 'ustanowienia' SSO tak jak poprawne skonfigurowanie Entity ID i ACS URL (wymiana metadanych).

Analiza i Wyjaśnienie

Sugerowana odpowiedź: A

Konfiguracja SAML SSO wymaga obustronnej wymiany identyfikatorów i certyfikatów między Google (SP) a Active Directory (IdP).
