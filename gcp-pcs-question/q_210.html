Question 210 / 361

Your company uses Google Cloud and has publicly exposed network assets. You want to discover the assets and perform a security audit on these assets by using a software tool in the least amount of time. What should you do?
  • A. Run a platform security scanner on all instances in the organization.
  • B. Identify all external assets by using Cloud Asset Inventory, and then run a network security scanner against them.
  • C. Contact a Google approved security vendor to perform the audit.
  • D. Notify Google about the pending audit, and wait for confirmation before performing the scan.
Show Suggested Answer Hide Answer
Suggested Answer: B

Wyjaśnienie

Odpowiedź: B.

Wyjaśnienie (Dlaczego poprawna):
B. Cloud Asset Inventory + Network Security Scanner: Cloud Asset Inventory pozwala błyskawicznie wygenerować listę wszystkich zasobów posiadających publiczne IP (External Assets). Mając tę listę, możesz skierować na nie skaner (np. Cloud Security Scanner lub zewnętrzne narzędzia), co jest znacznie szybsze niż skanowanie wszystkiego 'na ślepo'.

Dlaczego inne są błędne:
A. Scan all instances: W dużej organizacji skanowanie 100% instancji (również tych prywatnych) zajmie mnóstwo czasu i zasobów.
C. Security vendor: Zajmie to znacznie więcej czasu (kontraktowanie, planowanie) niż użycie natywnych narzędzi chmurowych.
D. Notify Google: Od 2021 roku Google nie wymaga już wcześniejszego powiadamiania o standardowych testach penetracyjnych i skanach podatności własnych zasobów (z pewnymi wyjątkami dotyczącymi DoS).
