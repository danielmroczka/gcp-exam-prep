Question 22 / 361

A website design company recently migrated all customer sites to App Engine. Some sites are still in progress and should only be visible to customers and company employees from any location. Which solution will restrict access to the in-progress sites?
  • A. Upload an .htaccess file containing the customer and employee user accounts to App Engine.
  • B. Create an App Engine firewall rule that allows access from the customer and employee networks and denies all other traffic.
  • C. Enable Cloud Identity-Aware Proxy (IAP), and allow access to a Google Group that contains the customer and employee user accounts.
  • D. Use Cloud VPN to create a VPN connection between the relevant on-premises networks and the company's GCP Virtual Private Cloud (VPC) network.
Show Suggested Answer Hide Answer
Suggested Answer: C

Wyjaśnienie

Odpowiedź: C.

Wyjaśnienie (Dlaczego poprawna):
C. Enable Cloud IAP + Google Group: Identity-Aware Proxy (IAP) to idealne rozwiązanie dla App Engine. Pozwala na zablokowanie dostępu do aplikacji na poziomie tożsamości (użytkownik musi się zalogować). Dodanie grupy Google do IAP pozwala na łatwe zarządzanie listą osób uprawnionych bez względu na ich lokalizację (pracuje z każdego miejsca w internecie).

Dlaczego inne są błędne:
A. .htaccess: Nie jest wspierany przez standardowe środowisko App Engine (to rozwiązanie dla Apache).
B. Firewall rule: Blokowanie po IP jest trudne, gdy klienci/pracownicy łączą się 'z dowolnej lokalizacji' (zmienne IP).
D. Cloud VPN: Wymagałoby, aby użytkownicy byli wpięci do konkretnej sieci, co jest trudniejsze niż proste logowanie przez IAP.

Sugerowana odpowiedź: C

Cloud IAP realizuje model 'Zero Trust'. Tożsamość jest nowym obwodem bezpieczeństwa, zastępującym stare metody oparte na adresach IP.
