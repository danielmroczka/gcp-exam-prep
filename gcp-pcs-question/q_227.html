Question 227 / 361

You are migrating your users to Google Cloud. There are cookie replay attacks with Google web and Google Cloud CLI SDK sessions on endpoint devices. You need to reduce the risk of these threats. What should you do? (Choose two.)
  • A. Configure Google session control to a shorter duration.
  • B. Set an organizational policy for OAuth 2.0 access token with a shorter duration.
  • C. Set a reauthentication policy for Google Cloud services to a shorter duration.
  • D. Configure a third-party identity provider with session management.
  • E. Enforce Security Key Authentication with 2SV.
Show Suggested Answer Hide Answer
Suggested Answer: AE

Wyjaśnienie

Odpowiedź: AE.

Wyjaśnienie (Dlaczego poprawna):
A. Session control to a shorter duration: Skrócenie czasu trwania sesji ciasteczka (cookie) sprawia, że skradzione ciasteczko szybciej staje się bezużyteczne dla napastnika.
E. Security Key Authentication (2SV): Ataki typu 'cookie replay' są często blokowane przez wymóg użycia klucza bezpieczeństwa przy każdej krytycznej akcji lub po prostu poprzez fakt, że nowoczesne systemy MFA (FIDO2) lepiej wiążą sesję z fizycznym urządzeniem użytkownika. Ponadto, wymuszenie re-autentykacji sprzętowej utrudnia napastnikowi utrzymanie dostępu.

Dlaczego inne są błędne:
B. OAuth token duration: Dotyczy aplikacji trzecich, a nie głównej sesji logowania użytkownika do konsoli/SDK.
C. Reauthentication policy: Brzmi podobnie do A, ale 'Session Control' (A) jest specyficznym mechanizmem Google do zarządzania czasem życia ciasteczek logowania.
D. Third-party session management: Może pomóc, ale natywne mechanizmy Google (A i E) są pierwszą rekomendacją dla ochrony sesji GCP.

Analiza i Wyjaśnienie

Sugerowana odpowiedź: AE

Ochrona sesji przed kradzieżą (Session Hijacking) polega na minimalizacji czasu ich trwania i wymuszaniu silnej tożsamości.
