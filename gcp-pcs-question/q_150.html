Question 150 / 361

Your organization's Google Cloud VMs are deployed via an instance template that configures them with a public IP address in order to host web services for external users. The VMs reside in a service project that is attached to a host (VPC) project containing one custom Shared VPC for the VMs. You have been asked to reduce the exposure of the VMs to the internet while continuing to service external users. You have already recreated the instance template without a public IP address configuration to launch the managed instance group (MIG). What should you do?
  • A. Deploy a Cloud NAT Gateway in the service project for the MIG.
  • B. Deploy a Cloud NAT Gateway in the host (VPC) project for the MIG.
  • C. Deploy an external HTTP(S) load balancer in the service project with the MIG as a backend.
  • D. Deploy an external HTTP(S) load balancer in the host (VPC) project with the MIG as a backend.
Show Suggested Answer Hide Answer
Suggested Answer: C

Wyjaśnienie

Odpowiedź: C.

Wyjaśnienie (Dlaczego poprawna):
C. External HTTP(S) load balancer in the service project: To jest zalecana architektura. Twoje maszyny VM nie mają publicznych adresów IP (bezpieczeństwo), a ruch z internetu trafia najpierw do Load Balancera. Load Balancer przekazuje ruch do backendu (MIG) przez wewnętrzną sieć GCP. Ponieważ aplikacja należy do zespołu w projekcie serwisowym, Load Balancer powinien znajdować się tam, gdzie jego backendy, korzystając z sieci Shared VPC udostępnionej z projektu hosta.

Dlaczego inne są błędne:
A, B. Cloud NAT: Pozwala maszynom WYJŚĆ do internetu, ale nie pozwala użytkownikom zewnętrznym WEJŚĆ do Twoich usług webowych.
D. Load balancer in host project: Choć technicznie możliwe, łamie zasadę separacji obowiązków (Separation of Duties). Zespół aplikacji w projekcie serwisowym powinien zarządzać swoim Load Balancerem, a zespół sieci w projekcie hosta tylko samą siecią VPC.

Sugerowana odpowiedź: C

Użycie Load Balancera jako 'frontu' pozwala na całkowite ukrycie adresów IP serwerów aplikacji przed internetem.
