Question 181 / 361

Your organization operates Virtual Machines (VMs) with only private IPs in the Virtual Private Cloud (VPC) with internet access through Cloud NAT. Everyday, you must patch all VMs with critical OS updates and provide summary reports. What should you do?
  • A. Validate that the egress firewall rules allow any outgoing traffic. Log in to each VM and execute OS specific update commands. Configure the Cloud Scheduler job to update with critical patches daily for daily updates.
  • B. Copy the latest patches to the Cloud Storage bucket. Log in to each VM, download the patches from the bucket, and install them.
  • C. Assign public IPs to VMs. Validate that the egress firewall rules allow any outgoing traffic. Log in to each VM, and configure a daily cron job to enable for OS updates at night during low activity periods.
  • D. Ensure that VM Manager is installed and running on the VMs. In the OS patch management service, configure the patch jobs to update with critical patches daily.
Show Suggested Answer Hide Answer
Suggested Answer: D

Wyjaśnienie

Odpowiedź: D.

Wyjaśnienie (Dlaczego poprawna):
D. VM Manager + OS patch management: VM Manager to natywny zestaw narzędzi w GCP do zarządzania flotą maszyn VM. Moduł 'OS patch management' pozwala na pełną automatyzację procesu patchowania (zgodnie z harmonogramem, np. codziennie) oraz generowanie raportów o stanie poprawek we wszystkich maszynach, co idealnie spełnia wymagania bez konieczności ręcznego logowania się do każdej VM.

Dlaczego inne są błędne:
A, B, C: Wszystkie sugerują ręczne logowanie się do maszyn ('Log in to each VM'), co jest skrajnie nieefektywne, podatne na błędy i nie skaluje się przy dużej liczbie instancji.
