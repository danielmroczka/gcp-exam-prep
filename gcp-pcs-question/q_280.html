Cloud Run access via IAP: only AppDev group, restricted to internal IPs. What to do?
  • A. VPN gateway.
  • B. Access level with IP ranges and groups, apply to IAP.
  • C. Firewall rules.
  • D. IAP MFA + NIDS.
Show Suggested Answer Hide Answer
Suggested Answer: B

Wyjaśnienie

Odpowiedź: B.

Wyjaśnienie (Dlaczego poprawna):
B. Access Context Manager + IAP: To połączenie pozwala na zdefiniowanie 'Context-Aware Access' (CAA). Tworzysz Access Level, który wymaga przynależności do grupy 'AppDev' ORAZ połączenia z określonych zakresów adresów IP. Następnie przypinasz ten poziom dostępu do polityki IAP chroniącej Twój serwis Cloud Run.

Dlaczego inne są błędne:
A. VPN gateway: Sam VPN nie kontroluje uprawnień grupowych IAM w warstwie aplikacji Cloud Run.
C. Firewall rules: Cloud Run jest usługą serverless i nie można do niej przypisać standardowych reguł firewalla VPC w celu filtrowania IP użytkowników przeglądarkowych. Tylko IAP/WAF to potrafi.
D. NIDS: To system detekcji włamań, nie służy do kontroli dostępu opartej na grupach i IP.

Konsensus społeczności ExamTopics: 100% na B. CAA to jedyna metoda na wymuszenie IP przy dostępie do serverless przez IAP.

Analiza i Wyjaśnienie

Sugerowana odpowiedź: B

Głęboka analiza: IAP najpierw sprawdza 'kto' (IAM), a potem 'skąd/jak' (Context-Aware Access). Tylko pomyślne przejście obu testów pozwala na dostęp do aplikacji.
