Question 330 / 361

Your organization has Google Cloud applications that require access to external web services. You must monitor, control, and log access to these services. What should you do?
  • A. Set up a Secure Web Proxy that allows access to the specific external web services. Configure applications to use the proxy for the web service requests.
  • B. Set up a Cloud NAT instance to allow egress traffic from your VPC.
  • C. Configure VPC firewall rules to allow the services to access the IP addresses of required external web services.
  • D. Configure Google Cloud Armor to monitor and protect your applications by checking incoming traffic patterns for attack patterns.
Show Suggested Answer Hide Answer
Suggested Answer: A

Wyjaśnienie

Odpowiedź: A.

Wyjaśnienie (Dlaczego poprawna):
A. Secure Web Proxy: Jest to nowa, natywna usługa GCP (część Cloud NGFW) zaprojektowana specyficznie do kontroli i audytu ruchu WYCHODZĄCEGO (egress) do internetu. Pozwala na definiowanie reguł opartych na domenach (FQDN), a nie tylko adresach IP, oraz dostarcza szczegółowe logi z każdej próby połączenia z zewnętrznym serwisem.

Dlaczego inne są błędne:
B. Cloud NAT: Pozwala na wyjście, ale nie oferuje zaawansowanej kontroli na poziomie domen (FQDN) ani szczegółowego logowania 'gdzie dokładnie' maszyna się łączy (widzisz tylko ruch IP).
C. VPC Firewall: Podobnie jak NAT, reguły firewall oparte na IP są bardzo trudne do utrzymania dla zewnętrznych serwisów webowych, których adresy IP często się zmieniają.
D. Cloud Armor: Chroni Twoje aplikacje przed ruchem przychodzącym (Ingress), nie służy do kontroli wyjścia Twoich aplikacji do internetu.

Sugerowana odpowiedź: A

Secure Web Proxy to 'zarządzany firewall wychodzący', niezbędny do zabezpieczenia komunikacji z zewnętrznymi API i usługami.
