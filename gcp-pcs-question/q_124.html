Question 124 / 361

You are working with protected health information (PHI) for an electronic health record system. The privacy officer is concerned that sensitive data is stored in the analytics system. You are tasked with anonymizing the sensitive data in a way that is not reversible. Also, the anonymized data should not preserve the character set and length. Which Google Cloud solution should you use?
  • A. Cloud Data Loss Prevention with deterministic encryption using AES-SIV
  • B. Cloud Data Loss Prevention with format-preserving encryption
  • C. Cloud Data Loss Prevention with cryptographic hashing
  • D. Cloud Data Loss Prevention with Cloud Key Management Service wrapped cryptographic keys
Show Suggested Answer Hide Answer
Suggested Answer: C

Wyjaśnienie

Odpowiedź: C.

Wyjaśnienie (Dlaczego poprawna):
C. Cloud DLP with cryptographic hashing: Haszowanie kryptograficzne jest procesem jednostronnym (nieodwracalnym). Po zamianie danych (np. nazwiska) na skrót (hash), nie można odzyskać oryginału bez posiadania klucza i przeprowadzenia ataku brute-force. Ponadto, hash zazwyczaj ma stałą długość i inny zestaw znaków niż wejście, co spełnia wymóg 'should not preserve character set and length'.

Dlaczego inne są błędne:
A. Deterministic encryption: Jest odwracalna (jeśli masz klucz).
B. Format-preserving encryption (FPE): Z definicji zachowuje zestaw znaków i długość danych (np. numer karty kredytowej zamienia na inny numer o tej samej długości).
D. KMS wrapped keys: To metoda zarządzania kluczami, a nie sam sposób anonimizacji danych (choć może być częścią procesu, samo haszowanie (C) jest sednem odpowiedzi).

Analiza i Podsumowanie Dyskusji (AI)

Analiza i Wyjaśnienie

Sugerowana odpowiedź: C

Haszowanie kryptograficzne to najsilniejsza forma de-identyfikacji danych, gdy nie planujemy powrotu do danych jawnych.
