Question 4 / 361

Your team wants to centrally manage GCP IAM permissions from their on-premises Active Directory Service. Your team wants to manage permissions by AD group membership. What should your team do to meet these requirements?
  • A. Set up Cloud Directory Sync to sync groups, and set IAM permissions on the groups.
  • B. Set up SAML 2.0 Single Sign-On (SSO), and assign IAM permissions to the groups.
  • C. Use the Cloud Identity and Access Management API to create groups and IAM permissions from Active Directory.
  • D. Use the Admin SDK to create groups and assign IAM permissions from Active Directory.
Show Suggested Answer Hide Answer
Suggested Answer: A

Wyjaśnienie

Odpowiedź: A.

Wyjaśnienie (Dlaczego poprawna):
A. Cloud Directory Sync (GCDS): Jest to darmowe narzędzie Google, które synchronizuje użytkowników i grupy z LDAP/Active Directory do Google Cloud Identity. Pozwala to na zarządzanie uprawnieniami w GCP bezpośrednio na grupach, które odzwierciedlają strukturę AD.

Dlaczego inne są błędne:
B. SAML SSO: SAML służy do uwierzytelniania (logowania), ale sam w sobie nie synchronizuje składu grup z AD do systemu IAM w GCP. Bez GCDS grupy w GCP nie wiedziałyby, jacy użytkownicy do nich należą.
C i D: Pisanie własnych skryptów przy użyciu API jest nieefektywne i błędotwórcze, gdy istnieje gotowe, wspierane narzędzie GCDS.

Konsensus społeczności ExamTopics: Zdecydowana większość wskazuje na A. GCDS to 'prawy ramię' AD w ekosystemie Google.

Analiza i Podsumowanie Dyskusji (AI)

Analiza i Wyjaśnienie

Sugerowana odpowiedź: A

Głęboka analiza: Kluczowe jest rozróżnienie między Provisioningiem (GCDS) a Federacją (SAML). GCDS tworzy 'tożsamości' w Google, a SAML pozwala na 'logowanie' się nimi przy użyciu hasła z AD.
