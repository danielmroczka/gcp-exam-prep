Question 53 / 361

A customer deployed an application on Compute Engine that takes advantage of the elastic nature of cloud computing. How can you work with Infrastructure Operations Engineers to best ensure that Windows Compute Engine VMs are up to date with all the latest OS patches?
  • A. Build new base images when patches are available, and use a CI/CD pipeline to rebuild VMs, deploying incrementally.
  • B. Federate a Domain Controller into Compute Engine, and roll out weekly patches via Group Policy Object.
  • C. Use Deployment Manager to provision updated VMs into new serving Instance Groups (IGs).
  • D. Reboot all VMs during the weekly maintenance window and allow the StartUp Script to download the latest patches from the internet.
Show Suggested Answer Hide Answer
Suggested Answer: A

Wyjaśnienie

Odpowiedź: A.

Wyjaśnienie (Dlaczego poprawna):
A. Build new base images and rebuild VMs: W świecie chmury zaleca się podejście 'Immutable Infrastructure'. Zamiast patchować działające maszyny (co jest ryzykowne i niespójne), buduje się nowy, czysty obraz 'złoty' (Golden Image) z poprawkami i podmienia maszyny w grupach IG (Rolling Update).

Dlaczego inne są błędne:
B. GPO: Tradycyjne podejście on-premises, wolniejsze i trudniejsze do śledzenia w dynamicznym środowisku chmurowym.
C. Deployment Manager: To narzędzie IaC, samo w sobie nie zarządza procesem patchowania wewnątrz OS.
D. StartUp Script: Pobieranie patchy przy każdym starcie spowalnia skalowanie maszyn i może zawieść, jeśli repozytorium patchy będzie niedostępne.

Analiza i Wyjaśnienie

Sugerowana odpowiedź: A

Infrastruktura niezmienna (Immutable) gwarantuje, że to co przetestowano w środowisku testowym, jest identyczne z tym co działa na produkcji.
