Question 157 / 361

Audit every single access to sensitive data in BigQuery, including SQL queries and people who viewed the results. Which log?
  • A. Admin Activity logs
  • B. Data Access logs
  • C. System Event logs
  • D. BigQuery Job logs
Show Suggested Answer Hide Answer
Suggested Answer: B

Wyjaśnienie

Odpowiedź: B.

Wyjaśnienie (Dlaczego poprawna):
B. Data Access logs: Dla BigQuery logi te rejestrują zdarzenia `query_job.completed`, które zawierają treść zapytania SQL, oraz zdarzenia `tabledata.list` (kto przeglądał wyniki). Jest to jedyny pełny ślad audytowy dostępu do danych.

Dlaczego inne są błędne:
A. Admin Activity: Rejestrują tylko zmiany w strukturze (np. stworzenie nowej tabeli), a nie odczyt danych.
C. System Event: Dotyczą problemów infrastruktury Google.
D. Job logs: To techniczny status zadań, niepełny dla audytu tożsamości i bezpieczeństwa danych tak jak ustrukturyzowane logi audytowe (B).

Konsensus społeczności ExamTopics: 100% na B. To kręgosłup audytu Big Data.

Analiza i Podsumowanie Dyskusji (AI)

Analiza i Wyjaśnienie

Sugerowana odpowiedź: B

Analiza spornych kwestii (AI): Eksperci ostrzegają: logi Data Access dla BigQuery potrafią generować gigantyczne koszty przy intensywnym użytkowaniu bazy. Zaleca się ich filtrowanie lub eksport tylko dla wybranych, najbardziej wrażliwych zbiorów danych (Datasets).
