Question 158 / 361

Protect your VPC from data exfiltration where an authorized user tries to copy data from your Cloud Storage to a bucket in their personal GCP project. Which service?
  • A. Cloud IAM
  • B. VPC Service Controls
  • C. VPC Firewall rules
  • D. Cloud Armor
Show Suggested Answer Hide Answer
Suggested Answer: B

Wyjaśnienie

Odpowiedź: B.

Wyjaśnienie (Dlaczego poprawna):
B. VPC Service Controls: Jest to jedyna usługa, która potrafi zapobiec kopiowaniu danych Między projektami na poziomie API Google. Tworzy perymetr, który blokuje dostęp do danych z zewnątrz oraz uniemożliwia wysyłanie danych do zasobów (bucketów) niebędących częścią tego perymetru.

Dlaczego inne są błędne:
A. IAM: Uprawniony użytkownik MOŻE czytać dane. IAM nie zabroni mu wysłać tych danych do innego bucketu, do którego też ma uprawnienia (np. prywatnego).
C. Firewall: Chroni sieć, nie 'rozumie' operacji kopiowania między API Google.
D. Cloud Armor: Chroni aplikacje webowe przed atakami typu SQLi/DDoS.

Konsensus społeczności ExamTopics: 100% na B. VPC SC to definicja ochrony przed eksfiltracją.

Analiza i Podsumowanie Dyskusji (AI)

Analiza i Wyjaśnienie

Sugerowana odpowiedź: B

Analiza spornych kwestii (AI): To pytanie jest kluczowe dla zrozumienia różnicy między Autoryzacją (IAM) a Granicą Sieciową (VPC SC). Na forach inżynierowie nazywają VPC SC 'zaporą ogniową dla API', która chroni dane przed 'zaufanymi, ale nieuczciwymi' użytkownikami.
