Question 54 / 361

Your team needs to make sure that their backend database can only be accessed by the frontend application and no other instances on the network. How should your team design this network?
  • A. Create an ingress firewall rule to allow access only from the application to the database using firewall tags.
  • B. Create a different subnet for the frontend application and database to ensure network isolation.
  • C. Create two VPC networks, and connect the two networks using Cloud VPN gateways to ensure network isolation.
  • D. Create two VPC networks, and connect the two networks using VPC peering to ensure network isolation.
Show Suggested Answer Hide Answer
Suggested Answer: A

Wyjaśnienie

Odpowiedź: A.

Wyjaśnienie (Dlaczego poprawna):
A. Ingress firewall rule + tags: To jest najbardziej elegancki i natywny sposób w GCP. Nadajesz tag `frontend` maszynom aplikacji i tag `database` maszynom bazy. Tworzysz regułę firewall zezwalającą na ruch na port bazy (np. 5432) tylko jeśli źródłem jest tag `frontend` i celem jest tag `database`.

Dlaczego inne są błędne:
B. Different subnets: Podsieci w GCP nie izolują ruchu (routing między subnetami w tej samej VPC jest domyślnie otwarty).
C, D. Separate VPCs: Bardzo skomplikowane rozwiązanie dla tak prostego problemu izolacji wewnątrz jednej aplikacji.

Analiza i Podsumowanie Dyskusji (AI)

Analiza i Wyjaśnienie

Sugerowana odpowiedź: A

Tagi sieciowe (lub Service Accounts w nowszych wersjach) pozwalają na tworzenie reguł firewall opartych na rolach maszyn, a nie na ich adresach IP.
