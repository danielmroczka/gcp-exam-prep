Question 44 / 361

You want to prevent users from creating public IP addresses for their Compute Engine instances. Which tool should you use?
  • A. VPC Firewall Rules
  • B. Organization Policy Service
  • C. Cloud IAM
  • D. Access Context Manager
Show Suggested Answer Hide Answer
Suggested Answer: B

Wyjaśnienie

Odpowiedź: B.

Wyjaśnienie (Dlaczego poprawna):
B. Organization Policy Service: Pozwala na nakładanie odgórnych ograniczeń (constraints) na poziomie całej firmy lub folderu. Ograniczenie `compute.restrictPublicIp` (lub podobne) fizycznie uniemożliwia przypisanie zewnętrznego adresu IP do interfejsu sieciowego VM.

Dlaczego inne są błędne:
A. VPC Firewall: Blokuje ruch na portach, ale nie uniemożliwia SAMEGO przypisania adresu IP do maszyny.
C. Cloud IAM: Można by odebrać uprawnienia do edycji IP, ale to skomplikowane i trudne do wymuszenia globalnie przy wielu rolach. Org Policy jest bardziej precyzyjna i prewencyjna.
D. Access Context Manager: Służy do kontroli dostępu do konsoli/API na podstawie kontekstu (np. skąd loguje się admin), a nie do ograniczania konfiguracji zasobów.

Konsensus społeczności ExamTopics: Zdecydowane poparcie dla B. Słowo 'prevent' w kontekście konfiguracji infrastruktury często wskazuje na Org Policy.

Analiza i Podsumowanie Dyskusji (AI)

Analiza i Wyjaśnienie

Sugerowana odpowiedź: B

Głęboka analiza: Polityki organizacji działają prewencyjnie - operacja API utworzenia VM z publicznym IP zostanie odrzucona przez system Google, zanim maszyna powstanie.
