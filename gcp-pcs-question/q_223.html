Question 223 / 361

Analyze VPC network traffic to detect data exfiltration via DNS tunneling. Which service?
  • A. BigQuery via Log Sink
  • B. Event Threat Detection (ETD) with Cloud DNS logs
  • C. Cloud Trace
  • D. Packet Mirroring
Show Suggested Answer Hide Answer
Suggested Answer: B

Wyjaśnienie

Odpowiedź: B.

Wyjaśnienie (Dlaczego poprawna):
B. ETD + DNS Logs: DNS Tunneling to metoda ukrywania danych w zapytaniach DNS. ETD posiada dedykowany detektor, który analizuje logi z Cloud DNS i szuka nietypowo długich lub licznych zapytań o dziwne subdomeny, co jest sygnaturą tego ataku.

Dlaczego inne są błędne:
A. BigQuery: Wymagałoby ręcznego pisania bardzo trudnych algorytmów analitycznych.
C. Cloud Trace: Nie widzi ruchu sieciowego DNS.
D. Packet Mirroring: Zadziałałoby, ale analiza surowego ruchu UDP/53 jest znacznie trudniejsza i droższa niż analiza logów przez gotowy silnik ETD (B).

Konsensus społeczności ExamTopics: 100% na B. DNS Tunneling detection to mocna strona ETD.

Analiza i Podsumowanie Dyskusji (AI)

Analiza i Wyjaśnienie

Sugerowana odpowiedź: B

Analiza spornych kwestii (AI): Eksperci na forach zauważają, że hakerzy uwielbiają DNS, bo jest on zazwyczaj otwarty na każdym firewallu. ETD (B) to jedyny automat w GCP, który potrafi to 'wyłowić' bez zapychania sieci kopiami pakietów.
