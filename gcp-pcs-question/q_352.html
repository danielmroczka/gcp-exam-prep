Question 352 / 361

Your company hosts a critical web application on Google Cloud. The application is experiencing an increasing number of sophisticated layer 7 attacks, including cross-site scripting (XSS) and SQL injection attempts. You need to protect the application from these attacks while minimizing the impact on legitimate traffic and ensuring high availability. What should you do?
  • A. Implement a load balancer in front of the web application instances, and enable Adaptive Protection and throttling to mitigate the occurrence of these malicious requests.
  • B. Configure Cloud Next Generation Firewall to block known malicious IP addresses targeting /32 addresses.
  • C. Enable Google Cloud Armor's pre-configured WAF rules for OWASP Top 10 vulnerabilities at the backend service.
  • D. Configure a Cloud Armor security policy with customized and pre-configured WAF rules for OWASP Top 10 vulnerabilities at the load balancer.
Show Suggested Answer Hide Answer
Suggested Answer: D

Wyjaśnienie

Odpowiedź: D.

Wyjaśnienie (Dlaczego poprawna):
D. Cloud Armor policy + pre-configured WAF rules: Cloud Armor posiada zestaw gotowych reguł (WAF pre-configured rules) chroniących przed atakami z listy OWASP Top 10 (SQLi, XSS, LFI itp.). Konfiguracja tej polityki na Load Balancerze zapewnia ochronę całego ruchu przychodzącego, a możliwość 'customizacji' pozwala na wykluczenie reguł powodujących błędy (false positives), co zapewnia wysoką dostępność dla legalnych użytkowników.

Dlaczego inne są błędne:
A. Adaptive Protection: To potężna funkcja ML do wykrywania DDoS, ale dla konkretnych ataków typu SQLi/XSS standardową i pierwszą linią obrony są reguły WAF (D).
B. NGFW: Działa na warstwie 3/4, nie analizuje ładunku HTTP pod kątem skomplikowanych ataków L7 (XSS).
C. Backend service only: Choć technicznie poprawne, odpowiedź D wspomina o Load Balancerze jako punkcie styku, co jest pełniejszym opisem architektury perymetru.

Sugerowana odpowiedź: D

Cloud Armor WAF rules to 'tarcza' chroniąca kod aplikacji przed wykorzystaniem znanych luk w zabezpieczeniach webowych.
