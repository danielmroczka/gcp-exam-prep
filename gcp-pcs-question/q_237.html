Question 237 / 361

Your organization wants full control of the keys used to encrypt data at rest in their Google Cloud environments. Keys must be generated and stored outside of Google and integrate with many Google Services including BigQuery. What should you do?
  • A. Use customer-supplied encryption keys (CSEK) with keys generated on trusted external systems. Provide the raw CSEK as part of the API call.
  • B. Create a KMS key that is stored on a Google managed FIPS 140-2 level 3 Hardware Security Module (HSM). Manage the Identity and Access Management (IAM) permissions settings, and set up the key rotation period.
  • C. Use Cloud External Key Management (EKM) that integrates with an external Hardware Security Module (HSM) system from supported vendors.
  • D. Create a Cloud Key Management Service (KMS) key with imported key material. Wrap the key for protection during import. Import the key generated on a trusted system in Cloud KMS.
Show Suggested Answer Hide Answer
Suggested Answer: C

Wyjaśnienie

Odpowiedź: C.

Wyjaśnienie (Dlaczego poprawna):
C. Cloud External Key Management (EKM): Jak w pytaniu 160, EKM to jedyny mechanizm, który pozwala na używanie kluczy generowanych i przechowywanych FIZYCZNIE poza Google (on-premises lub u zewnętrznego partnera), jednocześnie integrując się z wysokopoziomowymi usługami takimi jak BigQuery.

Dlaczego inne są błędne:
A. CSEK: BigQuery NIE wspiera CSEK (kluczy dostarczanych w API call). Wspiera tylko CMEK (KMS) oraz EKM.
B. Cloud HSM: Klucze w HSM są generowane i zarządzane przez Google (choć w certyfikowanym sprzęcie), co nie spełnia wymogu 'stored outside of Google'.
D. Imported Key: Materiał jest generowany na zewnątrz, ale po imporcie trafia do chmury Google (KMS), tracisz nad nim fizyczną kontrolę.

Sugerowana odpowiedź: C

Cloud EKM jest dedykowany dla klientów wymagających najwyższego stopnia fizycznej suwerenności nad materiałem klucza.
