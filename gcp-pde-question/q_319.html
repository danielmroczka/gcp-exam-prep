You are preparing an organization-wide dataset. You need to preprocess customer data stored in a restricted bucket in Cloud Storage. The data will be used to create consumer analyses. You need to follow data privacy requirements, including protecting certain sensitive data elements, while also retaining all of the data for potential future use cases. What should you do?

  • A. Use the Cloud Data Loss Prevention API and Dataflow to detect and remove sensitive fields from the data in Cloud Storage. Write the filtered data in BigQuery.
  • B. Use customer-managed encryption keys (CMEK) to directly encrypt the data in Cloud Storage. Use federated queries from BigQuery. Share the encryption key by following the principle of least privilege.
  • C. Use Dataflow and the Cloud Data Loss Prevention API to mask sensitive data. Write the processed data in BigQuery. Most Voted
  • D. Use Dataflow and Cloud KMS to encrypt sensitive fields and write the encrypted data in BigQuery. Share the encryption key by following the principle of least privilege.

Odpowiedź: C.

Wykorzystanie Dataflow w połączeniu z Cloud Data Loss Prevention (DLP) API do maskowania danych wrażliwych jest najlepszym rozwiązaniem, ponieważ pozwala na spełnienie wymagań dotyczących prywatności danych, jednocześnie zachowując wszystkie dane dla przyszłych zastosowań. DLP API identyfikuje i maskuje dane wrażliwe (np. numery kart kredytowych, adresy e-mail) bez ich trwałego usuwania. Dataflow umożliwia przetwarzanie danych w dużej skali, co jest istotne przy przetwarzaniu danych organizacji. Zapisanie przetworzonych danych w BigQuery zapewnia łatwy dostęp do nich dla analiz. Maskowanie, a nie usuwanie, pozwala na potencjalne wykorzystanie danych w przyszłości po zastosowaniu odpowiednich technik odzyskiwania informacji.

A. Usunięcie danych wrażliwych jest sprzeczne z wymogiem zachowania wszystkich danych dla przyszłych zastosowań.
B. CMEK chroni dane w spoczynku, ale nie maskuje ich, co nie spełnia wymagań dotyczących prywatności podczas analiz.
D. Szyfrowanie danych wrażliwych, choć poprawne z punktu widzenia bezpieczeństwa, nie chroni przed dostępem do nich po odszyfrowaniu, a maskowanie jest bardziej odpowiednie dla celów analizy.

*   **Dataflow:** https://cloud.google.com/dataflow
*   **Cloud KMS:** https://cloud.google.com/kms
*   **BigQuery:** https://cloud.google.com/bigquery

Sugerowana odpowiedź: C

Odpowiedź C jest prawidłowa, ponieważ najlepiej spełnia wszystkie wymagania zadania. Kluczowe jest tutaj maskowanie danych wrażliwych, a nie ich usuwanie (eliminując opcję A) ani jedynie szyfrowanie (eliminując opcję B i D). Maskowanie pozwala na zachowanie danych do przyszłych analiz, jednocześnie chroniąc prywatność użytkowników. Wykorzystanie Dataflow umożliwia przetwarzanie danych w dużej skali, a Cloud Data Loss Prevention (DLP) API skutecznie identyfikuje i maskuje wrażliwe informacje. Zapisanie przetworzonych danych w BigQuery zapewnia łatwy dostęp do nich dla celów analitycznych.

Dyskusja użytkowników w pełni potwierdza poprawność odpowiedzi C. Większość użytkowników (Pime13, Nagamanikanta, 22c1725, HectorLeon2099) wskazuje na tę opcję, argumentując, że maskowanie danych jest zgodne z wymogiem zachowania wszystkich danych, a usuwanie ich uniemożliwiłoby przyszłe analizy. Komentarze podkreślają również, że dane konsumenckie często zawierają informacje PII (Personally Identifiable Information), co dodatkowo uzasadnia potrzebę maskowania.

Dlaczego inne odpowiedzi są nieprawidłowe:

  • A: Usuwanie danych wrażliwych narusza wymóg zachowania wszystkich danych.
  • B: Szyfrowanie (CMEK) chroni dane w spoczynku, ale nie maskuje ich, co nie spełnia wymagań dotyczących prywatności podczas analiz. Dostęp do odszyfrowanych danych nadal byłby możliwy.
  • D: Szyfrowanie za pomocą Cloud KMS również nie maskuje danych. Chociaż poprawne z punktu widzenia bezpieczeństwa, nie chroni przed dostępem do informacji po odszyfrowaniu.
