You want to create a machine learning model using BigQuery ML and create an endpoint for hosting the model using Vertex AI. This will enable the processing of continuous streaming data in near-real time from multiple vendors. The data may contain invalid values. What should you do?

  • A. Create a new BigQuery dataset and use streaming inserts to land the data from multiple vendors. Configure your BigQuery ML model to use the "ingestion" dataset as the framing data.
  • B. Use BigQuery streaming inserts to land the data from multiple vendors where your BigQuery dataset ML model is deployed.
  • C. Create a Pub/Sub topic and send all vendor data to it. Connect a Cloud Function to the topic to process the data and store it in BigQuery.
  • D. Create a Pub/Sub topic and send all vendor data to it. Use Dataflow to process and sanitize the Pub/Sub data and stream it to BigQuery. Most Voted

Odpowiedź: D.

Opcja D jest najlepszym rozwiązaniem, ponieważ łączy elastyczność i skalowalność Pub/Sub z możliwościami przetwarzania i czyszczenia danych Dataflow. Pub/Sub pozwala na niezawodne odbieranie strumieni danych od wielu dostawców. Dataflow umożliwia transformację i sanitację danych, radząc sobie z potencjalnymi wartościami nieprawidłowymi, co jest kluczowe dla jakości modelu uczenia maszynowego. Następnie, oczyszczone dane są strumieniowo przesyłane do BigQuery, gdzie model BigQuery ML może z nich korzystać w czasie zbliżonym do rzeczywistego. To podejście zapewnia solidną architekturę dla przetwarzania strumieniowego i utrzymania wysokiej jakości danych wejściowych dla modelu.

A. Użycie "ingestion" datasetu jako framing data nie rozwiązuje problemu walidacji danych, a jedynie pozwala na przechowywanie surowych danych.
B. Bezpośrednie strumieniowe wstawianie do BigQuery bez walidacji może prowadzić do problemów z jakością modelu.
C. Cloud Functions są mniej skalowalne i elastyczne niż Dataflow w przypadku przetwarzania dużych strumieni danych i bardziej skomplikowanych transformacji.

Analiza i Wyjaśnienie

Sugerowana odpowiedź: D

Odpowiedź D jest prawidłowa i najczęściej wskazywana przez użytkowników. Kluczowym wymaganiem zadania jest obsługa strumieni danych z wielu źródeł, potencjalnie zawierających nieprawidłowe wartości, oraz wykorzystanie BigQuery ML i Vertex AI do przetwarzania w czasie zbliżonym do rzeczywistego. Pub/Sub zapewnia niezawodny mechanizm odbioru danych od różnych dostawców. Dataflow umożliwia skalowalne i elastyczne przetwarzanie strumieni danych, w tym ich sanitację i transformację, co jest niezbędne do zapewnienia wysokiej jakości danych wejściowych dla modelu uczenia maszynowego. Oczyszczone dane są następnie strumieniowo przesyłane do BigQuery, gdzie model BigQuery ML może z nich korzystać.

Użytkownicy, w tym Atnafu (autor pytania), wielokrotnie potwierdzają poprawność odpowiedzi D. Komentarze wskazują, że Dataflow jest odpowiednim narzędziem do filtrowania i przetwarzania danych w strumieniu, co jest kluczowe w kontekście potencjalnych nieprawidłowości w danych wejściowych.

Dlaczego inne odpowiedzi są nieprawidłowe:

  • A: Samo przechowywanie surowych danych w BigQuery (nawet w dedykowanym zbiorze) nie rozwiązuje problemu walidacji i czyszczenia danych.
  • B: Bezpośrednie strumieniowe wstawianie danych do BigQuery bez wcześniejszego przetworzenia może prowadzić do problemów z jakością modelu i potencjalnych błędów.
  • C: Cloud Functions są mniej skalowalne i elastyczne niż Dataflow w przypadku przetwarzania dużych strumieni danych i bardziej skomplikowanych transformacji. Dataflow jest zaprojektowany do obsługi dużych wolumenów danych w sposób niezawodny i efektywny.
