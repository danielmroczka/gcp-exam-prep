You are designing a cloud-native historical data processing system to meet the following conditions:
✑ The data being analyzed is in CSV, Avro, and PDF formats and will be accessed by multiple analysis tools including Dataproc, BigQuery, and Compute
Engine.
✑ A batch pipeline moves daily data.
✑ Performance is not a factor in the solution.
✑ The solution design should maximize availability.
How should you design data storage for this solution?

  • A. Create a Dataproc cluster with high availability. Store the data in HDFS, and perform analysis as needed.
  • B. Store the data in BigQuery. Access the data using the BigQuery Connector on Dataproc and Compute Engine.
  • C. Store the data in a regional Cloud Storage bucket. Access the bucket directly using Dataproc, BigQuery, and Compute Engine.
  • D. Store the data in a multi-regional Cloud Storage bucket. Access the data directly using Dataproc, BigQuery, and Compute Engine. Most Voted

Odpowiedź: D.

Wymagania zadania wskazują na potrzebę wysokiej dostępności przechowywania danych historycznych, przy jednoczesnym braku nacisku na wydajność. Multi-regional Cloud Storage zapewnia najwyższy poziom dostępności w porównaniu do innych opcji, ponieważ replikuje dane w wielu regionach geograficznych. Dzięki temu, w przypadku awarii w jednym regionie, dane pozostają dostępne z innego. Bezpośredni dostęp do danych z Cloud Storage przez Dataproc, BigQuery i Compute Engine jest prosty i efektywny, a obsługa różnych formatów (CSV, Avro, PDF) jest natywna dla Cloud Storage. Wybór multi-regional zamiast regionalnego dodatkowo zwiększa odporność na awarie.

A. HDFS w Dataproc jest zależny od dostępności klastra Dataproc i nie zapewnia tak wysokiej dostępności jak Cloud Storage.
B. BigQuery jest świetne do analizy, ale przechowywanie wszystkich formatów danych (w tym PDF) w BigQuery może być mniej efektywne i wymagać dodatkowych kroków transformacji.
C. Regionalny Cloud Storage oferuje mniejszą dostępność niż multi-regionalny, co nie spełnia kluczowego wymagania zadania.

Analiza i Wyjaśnienie

Sugerowana odpowiedź: D

Prawidłową odpowiedzią jest D. Store the data in a multi-regional Cloud Storage bucket. Access the data directly....

Dlaczego?

Wymagania: 1. Multiple formats (CSV, Avro, PDF): Cloud Storage (GCS) jest uniwersalnym magazynem obiektowym, który idealnie nadaje się do przechowywania dowolnych typów plików (zarówno strukturalnych jak Avro, jak i niestrukturalnych jak PDF). 2. Accessed by multiple tools (Dataproc, BigQuery, GCE): GCS jest centralnym punktem integracji w GCP. BigQuery może odpytywać dane w GCS (External Tables), Dataproc korzysta z GCS przez GCS Connector, a GCE montuje lub pobiera pliki przez API/CLI. 3. Maximize availability: To kluczowe słowo. Bucket Multi-Regional oferuje najwyższą dostępność (99.95% vs 99.9% regional) oraz georedundancję (dane są replikowane w co najmniej dwóch regionach oddalonych od siebie o min. 160 km). 4. Performance is not a factor: Potwierdza to wybór GCS (który ma wyższą latencję niż lokalne dyski lub BigQuery native storage) oraz lokalizacji Multi-Regional (gdzie dane mogą być dalej od compute).

Dlaczego inne odpowiedzi są nieprawidłowe?

  • A. HDFS on Dataproc: HDFS jest przywiązany do klastra. Jeśli usuniesz klaster (co jest dobrą praktyką w chmurze), stracisz dane (chyba że używasz Persistent Disks, ale to drogie i skomplikowane). GCS zapewnia lepszą separację składowania od obliczeń.
  • B. Store in BigQuery: BigQuery nie nadaje się do składowania plików PDF.
  • C. Regional Cloud Storage: Oferuje mniejszą dostępność i odporność na katastrofy regionalne niż Multi-Regional (D). Skoro celem jest "maximize availability", wybieramy Multi-Regional.
