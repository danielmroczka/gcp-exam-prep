Your company is in a highly regulated industry. One of your requirements is to ensure individual users have access only to the minimum amount of information required to do their jobs. You want to enforce this requirement with Google BigQuery for a multi-tenant environment. Which three approaches can you take? (Choose three.)

  • A. Load data for each client into a different dataset.
  • B. Use row-level security to filter data based on the user's identity.
  • C. Put each client's BigQuery dataset into a different table.
  • D. Restrict a client's dataset to approved users using IAM.
  • E. Only allow a service account to access the datasets and handle all user queries through a middle tier.
  • F. Use the appropriate identity and access management (IAM) roles for each client's users at the dataset level.
Show Suggested Answer
Suggested Answer: A, D, F

Wyjaśnienie

Correct Answers: A, D, F

A, D, F. These options describe the standard BigQuery approach for multi-tenancy: isolating each client's data into its own dataset (A) and then using IAM roles (F) to grant access only to specific authorized users (D) for that dataset. This ensures strong isolation and adheres to the principle of least privilege.

Why other options are incorrect:
B. While row-level security is a valid way to filter data within a single table, for a highly regulated multi-tenant environment, dataset-level isolation is often preferred for more robust security and compliance.
C. This option is confusingly phrased ("dataset into a different table"). Tables are contained within datasets, not vice versa.
E. While a middle-tier service account approach is possible, it adds complexity and bypasses native BigQuery IAM features, making auditing and individual user management more difficult.
