You are planning to migrate your current on-premises Apache Hadoop deployment to the cloud. You need to ensure that the deployment is as fault-tolerant and cost-effective as possible for long-running batch jobs. You want to use a managed service. What should you do?

  • A. Deploy a Dataproc cluster. Use a standard persistent disk and 50% preemptible workers. Store data in Cloud Storage, and change references in scripts from hdfs:// to gs:// Most Voted
  • B. Deploy a Dataproc cluster. Use an SSD persistent disk and 50% preemptible workers. Store data in Cloud Storage, and change references in scripts from hdfs:// to gs://
  • C. Install Hadoop and Spark on a 10-node Compute Engine instance group with standard instances. Install the Cloud Storage connector, and store the data in Cloud Storage. Change references in scripts from hdfs:// to gs://
  • D. Install Hadoop and Spark on a 10-node Compute Engine instance group with preemptible instances. Store data in HDFS. Change references in scripts from hdfs:// to gs://

Odpowiedź: A.

Wykorzystanie Dataproc w połączeniu ze standardowymi dyskami persistent i 50% preemptible workerami jest najbardziej efektywnym rozwiązaniem dla długotrwałych zadań batchowych. Dataproc to w pełni zarządzana usługa Hadoop, co minimalizuje nakład pracy administracyjnej. Standardowe dyski persistent oferują dobry stosunek ceny do wydajności dla danych, które nie wymagają ekstremalnie szybkiego dostępu. Użycie 50% preemptible workerów pozwala na znaczące obniżenie kosztów, ponieważ te instancje są tańsze, ale mogą być przerywane. Przechowywanie danych w Cloud Storage zapewnia wysoką dostępność i skalowalność, a zmiana odniesień w skryptach z `hdfs://` na `gs://` umożliwia bezproblemowe korzystanie z Cloud Storage jako systemu plików.

A. Jest to poprawna odpowiedź, jak wyjaśniono powyżej.
B. SSD dyski są droższe niż standardowe, a dla długotrwałych zadań batchowych, gdzie szybkość dostępu do danych nie jest krytyczna, nie uzasadniają dodatkowych kosztów.
C. Instalacja i zarządzanie Hadoop i Spark na instancjach Compute Engine wymaga znacznie więcej pracy administracyjnej niż korzystanie z zarządzanej usługi Dataproc.
D. Przechowywanie danych w HDFS na Compute Engine ogranicza skalowalność i dostępność w porównaniu do Cloud Storage, a użycie wyłącznie preemptible instancji może prowadzić do częstych przerw w działaniu zadań.

Dodatkowe źródła:

*   **Google Cloud Dataproc:** https://cloud.google.com/dataproc
*   **Cloud Storage:** https://cloud.google.com/storage
*   **Preemptible VM instances:** https://cloud.google.com/compute/docs/instances/preemptible

Wykorzystanie Dataproc jako zarządzanej usługi Hadoop minimalizuje nakład pracy administracyjnej. Wykorzystanie 50% instancji preemptible pozwala na znaczne obniżenie kosztów, a przechowywanie danych w Cloud Storage zapewnia wysoką dostępność i skalowalność.
