You need to establish a unified data governance framework across multiple Cloud Storage buckets and BigQuery datasets. The requirement is for centralized metadata discovery, data quality monitoring, and consistent security policy application without physically moving or duplicating data. What should you do?

  • A. Export metadata out of Dataplex Universal Catalog by running a metadata export job. Implement Dataproc Metastore to manage table schemas.
  • B. Use Dataplex to organize the BigQuery datasets and Cloud Storage buckets into lakes and zones. Then, use Dataplex for automated metadata discovery, centralized security policy management, data profiling, and data quality tasks.
  • C. Deploy a centralized Cloud SQL database to store metadata extracted from BigQuery and Cloud Storage using custom scripts.
  • D. Create a Looker Studio dashboard on BigQuery INFORMATION_SCHEMA views to visualize and monitor data quality.
Show Suggested Answer
Suggested Answer: B

Wyjaśnienie

Correct Answer: B

Dataplex is Google Cloud's intelligent data fabric that allows you to manage, monitor, and govern data across data lakes, data warehouses, and data marts. By organizing BigQuery datasets and GCS buckets into Dataplex lakes and zones, you can achieve centralized governance. Dataplex provides automated metadata discovery, centralized security policy management (via IAM and policy tags), and built-in data profiling and quality features.

Why other options are incorrect:
A. Dataproc Metastore is primarily for Apache Hive/Spark metadata management, not a general-purpose governance framework for BQ and GCS.
C. Using custom scripts and Cloud SQL for metadata is a high-overhead, manual approach that doesn't leverage Google's managed governance tools.
D. Looker Studio on INFORMATION_SCHEMA can provide some visualization of metadata, but it is not a governance or policy enforcement framework.
