You work for a manufacturing company that sources up to 750 different components, each from a different supplier. You've collected a labeled dataset that has on average 1000 examples for each unique component. Your team wants to implement an app to help warehouse workers recognize incoming components based on a photo of the component. You want to implement the first working version of this app (as Proof-Of-Concept) within a few working days. What should you do?

  • A. Use Cloud Vision AutoML with the existing dataset. Most Voted
  • B. Use Cloud Vision AutoML, but reduce your dataset twice.
  • C. Use Cloud Vision API by providing custom labels as recognition hints.
  • D. Train your own image recognition model leveraging transfer learning techniques.

Odpowiedź: A.

Użycie Cloud Vision AutoML z istniejącym zbiorem danych jest najrozsądniejszym podejściem do szybkiego stworzenia działającego prototypu (Proof-of-Concept). AutoML automatyzuje proces trenowania i wdrażania modeli uczenia maszynowego, co pozwala uniknąć czasochłonnego ręcznego kodowania i konfiguracji. Posiadając 750 unikalnych komponentów z średnio 1000 przykładów na komponent, mamy wystarczająco dużo danych, aby AutoML mogło wytrenować model o akceptowalnej dokładności. AutoML jest zaprojektowane do szybkiego prototypowania i dostarczania działających rozwiązań w krótkim czasie, co idealnie odpowiada wymaganiom zadania.  Ponadto, usługa ta jest łatwa w użyciu i nie wymaga głębokiej wiedzy z zakresu uczenia maszynowego.

A. Redukcja zbioru danych może negatywnie wpłynąć na dokładność modelu.
B. Cloud Vision API z niestandardowymi etykietami nie jest tak efektywne jak trenowanie dedykowanego modelu.
C. Trenowanie własnego modelu od podstaw, nawet z transfer learningiem, jest zbyt czasochłonne dla szybkiego prototypu.
D. Wymagałoby to znacznego nakładu pracy i czasu, co jest sprzeczne z celem szybkiego wdrożenia Proof-of-Concept.

Analiza i Wyjaśnienie

Sugerowana odpowiedź: A

Prawidłową odpowiedzią jest A. Use Cloud Vision AutoML with the existing dataset.

Dlaczego?

Wymagania: 750 unikalnych komponentów, 1000 przykładów na każdy (duży, zbalansowany zbiór), "Proof-of-Concept within a few working days". 1. AutoML Vision: Jest idealny, gdy masz własny zbiór obrazów specyficznych dla Twojej domeny (części fabryczne), których ogólne modele (jak Vision API) nie rozpoznają. AutoML automatycznie dobiera architekturę sieci neuronowej i trenuje model na Twoich danych. 2. Time-to-market: Kilka dni to za mało na pisanie własnego modelu od zera (D). AutoML wymaga jedynie wrzucenia zdjęć i kliknięcia "Train". 3. Dataset size: 1000 zdjęć na kategorię to "sweet spot" dla AutoML, zapewniający bardzo wysoką dokładność (min. wymagane to 10-100, ale 1000 jest zalecane dla systemów produkcyjnych).

Dlaczego inne odpowiedzi są nieprawidłowe?

  • B. Reduce dataset: Nie ma powodu, by zmniejszać zbiór danych, skoro go już masz. Więcej danych w AutoML zazwyczaj oznacza lepszy model.
  • C. Vision API with hints: Vision API świetnie rozpoznaje ogólne obiekty (kot, pies, samochód), ale 750 specyficznych komponentów technicznych wykracza poza jego standardowe możliwości. "Hints" (podpowiedzi) nie zastąpią pełnego treningu na konkretnych obrazach części.
  • D. Train own model (Transfer Learning): Nawet z transfer learningiem, budowa własnego pipeline'u w TensorFlow/Keras, strojenie hiperparametrów i wdrożenie serwera modelu zajmie więcej niż "kilka dni roboczych" dla zespołu, który chce szybkiego POC.
