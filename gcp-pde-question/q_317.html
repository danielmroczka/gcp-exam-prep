You have several different unstructured data sources, within your on-premises data center as well as in the cloud. The data is in various formats, such as Apache Parquet and CSV. You want to centralize this data in Cloud Storage. You need to set up an object sink for your data that allows you to use your own encryption keys. You want to use a GUI-based solution. What should you do?

  • A. Use BigQuery Data Transfer Service to move files into BigQuery.
  • B. Use Storage Transfer Service to move files into Cloud Storage
  • C. Use Dataflow to move files into Cloud Storage
  • D. Use Cloud Data Fusion to move files into Cloud Storage. Most Voted

Odpowiedź: D.

Cloud Data Fusion oferuje graficzny interfejs użytkownika (GUI) do budowania i zarządzania potokami danych. Pozwala na integrację z różnymi źródłami danych, w tym z on-premise i chmurą, obsługuje różne formaty plików takie jak Parquet i CSV, a co najważniejsze, umożliwia konfigurację szyfrowania po stronie klienta (Customer-Managed Encryption Keys - CMEK) w Cloud Storage. Dzięki temu spełnia wszystkie wymagania zadania: centralizację danych w Cloud Storage, obsługę różnych formatów, użycie własnych kluczy szyfrujących i wykorzystanie GUI. Cloud Data Fusion jest zaprojektowane do tego typu zadań integracyjnych i transformacyjnych.

A. BigQuery Data Transfer Service służy do transferu danych *do* BigQuery, a nie bezpośrednio do Cloud Storage.
B. Storage Transfer Service, choć pozwala na transfer do Cloud Storage, nie oferuje tak elastycznej konfiguracji szyfrowania po stronie klienta poprzez GUI.
C. Dataflow wymaga pisania kodu (Python lub Java) i nie jest rozwiązaniem opartym na GUI, co jest kluczowym wymaganiem zadania.

Dodatkowe źródła:

*   **Cloud Data Fusion Documentation:** https://cloud.google.com/data-fusion/docs
*   **Storage Transfer Service Documentation:** https://cloud.google.com/storage-transfer/docs
*   **BigQuery Data Transfer Service Documentation:** https://cloud.google.com/bigquery-transfer/docs
*   **Dataflow Documentation:** https://cloud.google.com/dataflow/docs

Analiza i Wyjaśnienie

Sugerowana odpowiedź: D

Prawidłową odpowiedzią jest D, czyli użycie Cloud Data Fusion. Pytanie wyraźnie wymaga rozwiązania z graficznym interfejsem użytkownika (GUI) oraz możliwością wykorzystania własnych kluczy szyfrujących (CMEK) do zabezpieczenia danych w Cloud Storage. Cloud Data Fusion oferuje właśnie takie możliwości – posiada interfejs GUI do budowania potoków danych, obsługuje różne formaty plików (Parquet, CSV) i umożliwia konfigurację szyfrowania po stronie klienta (CMEK) w Cloud Storage.

Dyskusja użytkowników w większości poprawnie wskazuje odpowiedź D, podkreślając brak GUI w Storage Transfer Service oraz fakt, że Dataflow wymaga pisania kodu, co stoi w sprzeczności z wymaganiem GUI. Użytkownik apoio.certificacoes.closer słusznie zauważa, że Storage Transfer Service oferuje CMEK tylko w trakcie transferu (in-transit), a nie w spoczynku (at-rest) w Cloud Storage, co nie spełnia w pełni wymagań zadania.

Dlaczego inne odpowiedzi są nieprawidłowe:

  • A (BigQuery Data Transfer Service): Służy do transferu danych do BigQuery, a nie bezpośrednio do Cloud Storage.
  • B (Storage Transfer Service): Chociaż umożliwia transfer do Cloud Storage, nie oferuje GUI i zapewnia CMEK tylko podczas transferu, a nie w spoczynku.
  • C (Dataflow): Wymaga pisania kodu (Python lub Java) i nie jest rozwiązaniem opartym na GUI.

Najnowsze komentarze (np. od 22c1725) potwierdzają, że Storage Transfer Service nie posiada interfejsu GUI, co dodatkowo umacnia poprawność odpowiedzi D.
