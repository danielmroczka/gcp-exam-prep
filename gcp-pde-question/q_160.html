You work for a mid-sized enterprise that needs to move its operational system transaction data from an on-premises database to GCP. The database is about 20
TB in size. Which database should you choose?

Odpowiedź: A.

Cloud SQL jest najlepszym wyborem dla migracji istniejących relacyjnych baz danych o rozmiarze 20 TB z on-premises do GCP. Oferuje on w pełni zarządzaną usługę baz danych relacyjnych, kompatybilną z popularnymi silnikami takimi jak MySQL, PostgreSQL i SQL Server.  Migracja do Cloud SQL jest stosunkowo prosta, ponieważ można wykorzystać istniejące narzędzia i procesy migracji danych.  Cloud SQL zapewnia skalowalność i niezawodność, a także integrację z innymi usługami GCP.  Dla tej wielkości danych i potrzeby migracji istniejącej bazy relacyjnej, Cloud SQL oferuje najlepszy balans między kosztami, wydajnością i łatwością zarządzania.

B. Cloud Bigtable jest bazą NoSQL, zoptymalizowaną pod kątem dużych obciążeń analitycznych i danych szeregów czasowych, a nie transakcyjnych.
C. Cloud Spanner jest bardzo skalowalną bazą danych, ale jest droższa i bardziej skomplikowana w konfiguracji niż Cloud SQL, co nie jest konieczne dla tego scenariusza.
D. Cloud Datastore (obecnie znany jako Firestore w trybie Datastore) jest bazą NoSQL, przeznaczoną dla aplikacji mobilnych i webowych, a nie dla dużych transakcyjnych obciążeń.

Analiza i Wyjaśnienie

Sugerowana odpowiedź: A

Cloud SQL jest najbardziej odpowiednim wyborem dla migracji istniejącej relacyjnej bazy danych o rozmiarze 20 TB do GCP, biorąc pod uwagę wymagania operacyjnego systemu transakcyjnego. Początkowo pytanie sugerowało limit 30 TB dla Cloud SQL, jednak dyskusja użytkowników, a szczególnie komentarze z ostatnich miesięcy (sierpień 2024, listopad 2022, wrzesień 2021), jednoznacznie wskazują, że limit ten został zwiększony do 64 TB. To sprawia, że Cloud SQL jest w pełni wystarczający dla bazy danych o rozmiarze 20 TB, z zapasem na przyszły wzrost.

Cloud SQL oferuje w pełni zarządzaną usługę baz danych relacyjnych, kompatybilną z popularnymi silnikami (MySQL, PostgreSQL, SQL Server), co ułatwia migrację. Jest również bardziej opłacalny i prostszy w konfiguracji niż Cloud Spanner, który jest przeznaczony dla aplikacji wymagających globalnej dystrybucji i ekstremalnej skalowalności, co nie wynika z treści pytania.

Cloud Bigtable i Cloud Datastore to bazy NoSQL, które nie są odpowiednie dla transakcyjnych danych operacyjnych, które wymagają silnej konsystencji i obsługi SQL.

Komentarze użytkowników, takie jak te od Satishjuly18, Atnafu, zellck i ashu381, potwierdzają, że Cloud SQL jest obecnie w stanie obsłużyć 64 TB danych, co czyni go najlepszym wyborem w tym scenariuszu. Ważne jest, aby brać pod uwagę aktualne informacje o limitach i możliwościach usług GCP, a nie polegać na starych danych.

Użytkownik imrane1995 wybrał odpowiedź C, argumentując potrzebą silnej konsystencji i skalowalności, ale Cloud SQL również zapewnia silną konsystencję i jest skalowalny w ramach swojego limitu 64 TB.
