You use Google Cloud Dataflow to process data and decide if a message should be sent based on a set of criteria. You want to ensure that messages are grouped together for processing based on the time they arrived. How should you design the pipeline?

  • A. Use a fixed-time window with a duration of 60 minutes.
  • B. Use a sliding time window with a duration of 60 minutes.
  • C. Use a session window with a gap time duration of 60 minutes.
  • D. Use a global window with a time-based trigger with a delay of 60 minutes.
Show Suggested Answer
Suggested Answer: A

Wyjaśnienie

Correct Answer: A

Fixed windows (also known as tumbling windows) represent a consistent, non-overlapping time interval in the data stream. For example, if you set a 60-minute fixed window, all data with timestamps between 0:00:00 and 0:59:59 is collected in one window, and data from 1:00:00 to 1:59:59 is in the next. This is the most straightforward way to group messages based on their arrival time into discrete blocks for processing.

Why other options are incorrect:
B. Sliding windows can overlap. A message could belong to multiple windows if the slide period is less than the window duration.
C. Session windows are defined by periods of activity followed by a gap of inactivity. They are used for grouping data from a specific user or source that occurs in bursts.
D. A global window covers the entire duration of the pipeline. A time-based trigger in a global window would process all data received so far every 60 minutes, which is different from grouping them into 60-minute windows.
