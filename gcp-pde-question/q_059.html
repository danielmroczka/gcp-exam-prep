An online retailer has built their current application on Google App Engine. A new initiative at the company mandates that they extend their application to allow their customers to transact directly via the application. They need to manage their shopping transactions and analyze combined data from multiple datasets using a business intelligence (BI) tool. They want to use only a single database for this purpose. Which Google Cloud database should they choose?

  • A. BigQuery
  • B. Cloud SQL Most Voted
  • C. Cloud BigTable
  • D. Cloud Datastore

Odpowiedź: B.

Wyjaśnienie:
Cloud SQL to w pełni zarządzana usługa relacyjnej bazy danych (obsługująca PostgreSQL, MySQL i SQL Server), która doskonale spełnia oba kluczowe wymagania scenariusza. Jest idealna do zarządzania transakcjami handlowymi, ponieważ oferuje pełne wsparcie dla transakcji ACID (Atomicity, Consistency, Isolation, Durability), co jest niezbędne dla zapewnienia integralności danych w systemach transakcyjnych online. Dodatkowo, jako baza danych relacyjna, Cloud SQL jest szeroko kompatybilna z większością narzędzi Business Intelligence (BI). Umożliwia łatwe łączenie i analizowanie danych z wielu zestawów danych (czyli tabel w bazie), co jest typowe dla potrzeb analitycznych, wszystko w ramach jednej, spójnej bazy danych.

Dlaczego pozostałe opcje są gorsze/błędne:
*   **A. BigQuery:** Jest to hurtownia danych klasy enterprise, zaprojektowana głównie do analizy danych (OLAP) na dużą skalę i nie nadaje się do zarządzania transakcjami w czasie rzeczywistym (OLTP), takimi jak pojedyncze zamówienia klientów.
*   **C. Cloud Bigtable:** To nierelacyjna baza danych o szerokich kolumnach, zoptymalizowana pod kątem bardzo dużych obciążeń operacyjnych i analitycznych, ale brakuje jej silnych gwarancji transakcyjnych niezbędnych do zarządzania złożonymi transakcjami zakupowymi.
*   **D. Cloud Datastore:** Jest nierelacyjną bazą danych dokumentową, która choć obsługuje transakcje, to nie jest optymalna do wykonywania złożonych zapytań analitycznych i łączenia danych z wielu zestawów dla narzędzi BI bez eksportowania danych do innej usługi (np. BigQuery).

Analiza i Wyjaśnienie

Sugerowana odpowiedź: B. Cloud SQL

Najbardziej odpowiednią bazą danych w tym scenariuszu jest Cloud SQL. Kluczowe wymagania to obsługa transakcji handlowych oraz możliwość analizy danych za pomocą narzędzia BI. Cloud SQL, jako w pełni zarządzana usługa relacyjnej bazy danych (obsługująca PostgreSQL, MySQL i SQL Server), zapewnia pełne wsparcie dla transakcji ACID, co jest niezbędne dla integralności danych w systemach transakcyjnych. Dodatkowo, jako relacyjna baza danych, jest kompatybilna z większością narzędzi BI, umożliwiając łatwe łączenie i analizowanie danych z wielu tabel.

Dyskusja użytkowników potwierdza tę odpowiedź. Wiele osób, w tym oficjalni trenerzy Google (Fotofilico), wskazuje Cloud SQL jako najlepszy wybór. Dodatkowo, użytkownicy (DipT, DGames, zellck) zwracają uwagę na możliwość wykorzystania federowanych zapytań BigQuery do analizy danych z Cloud SQL, co pozwala na spełnienie obu wymagań w ramach jednego rozwiązania.

Dlaczego pozostałe opcje są nieprawidłowe:

  • BigQuery: Jest to hurtownia danych zoptymalizowana pod kątem analizy (OLAP), a nie zarządzania transakcjami w czasie rzeczywistym (OLTP).
  • Cloud Bigtable: To baza danych NoSQL o szerokich kolumnach, która nie oferuje silnych gwarancji transakcyjnych.
  • Cloud Datastore: Jest to baza danych dokumentowa, która nie jest optymalna do złożonych zapytań analitycznych i łączenia danych dla narzędzi BI.

Chociaż BigQuery może być używane do analizy danych z Cloud SQL poprzez federowane zapytania, to Cloud SQL pozostaje podstawową bazą danych do zarządzania transakcjami, co jest kluczowe w tym scenariuszu.
