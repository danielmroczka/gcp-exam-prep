Odpowiedź: D

Wyjaśnienie: Cloud Firestore jest idealnym wyborem do migracji 2TB relacyjnego bazy danych do Google Cloud Platform, biorąc pod uwagę ograniczenia zasobów i kosztów.  To rozwiązanie oferuje skalowalność, łatwość zarządzania i niskie opóźnienia, co jest kluczowe w kontekście migracji dużego bazy danych.  Firestone jest odpowiedni dla dużych, dynamicznych danych, ale nie jest zaprojektowany do migracji relacyjnych baz danych, a Cloud SQL jest bardziej odpowiedni dla baz danych SQL, które wymagają bardziej szczegółowej konfiguracji.  Cloud Spanner oferuje wysoką spójność i dostępność, ale może być droższy i bardziej skomplikowany w zarządzaniu niż Firestore.  Cloud Bigtable jest przeznaczony do danych o wysokiej przepustowości i niskiej latencji, a nie do migracji relacyjnych baz danych.  Zatem, Firestore oferuje najlepszy kompromis między skalowalnością, łatwością zarządzania i kosztami w kontekście migracji relacyjnego bazy danych.

Link do dodatkowych żródeł: https://cloud.google.com/firestore/

Analiza i Wyjaśnienie

Sugerowana odpowiedź: D

Najbardziej odpowiednią usługą do migracji 2TB relacyjnej bazy danych do Google Cloud Platform, z uwzględnieniem ograniczeń zasobów i kosztów, jest Cloud SQL. Kluczowym czynnikiem jest fakt, że aplikacja nie może być znacząco refaktoryzowana. Cloud SQL oferuje wsparcie dla popularnych silników bazodanowych, takich jak MySQL, PostgreSQL i SQL Server, co minimalizuje potrzebę zmian w kodzie aplikacji.

Dyskusja użytkowników potwierdza tę odpowiedź. Wiele osób wskazuje Cloud SQL jako prawidłowe rozwiązanie, a nowsze komentarze (np. Preetmehta1234, LaxmanTiwari) potwierdzają, że Cloud SQL obsługuje pojemność do 64TB (dla instancji z dedykowanymi rdzeniami), co jest wystarczające dla 2TB bazy danych. Użytkownik xrun słusznie zauważa, że Cloud Spanner wymagałby refaktoryzacji aplikacji ze względu na różnice w implementacji bazodanowej (brak procedur składowanych i triggerów).

Dlaczego inne odpowiedzi są nieprawidłowe:

  • Cloud Spanner: Jest droższy i bardziej skomplikowany w zarządzaniu niż Cloud SQL, a także wymagałby refaktoryzacji aplikacji.
  • Cloud Bigtable: Jest to baza danych NoSQL, zoptymalizowana pod kątem wysokiej przepustowości i niskiej latencji, a nie relacyjnych danych.
  • Cloud Firestore: Jest to baza danych dokumentowa NoSQL, nie nadaje się do migracji relacyjnych baz danych bez znaczącej refaktoryzacji.

Początkowe komentarze sugerujące ograniczenia pojemności Cloud SQL (np. Barniyah) są nieaktualne. Obecnie Cloud SQL oferuje pojemność wystarczającą dla 2TB bazy danych.
