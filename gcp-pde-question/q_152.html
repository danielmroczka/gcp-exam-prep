You work for a global shipping company. You want to train a model on 40 TB of data to predict which ships in each geographic region are likely to cause delivery delays on any given day. The model will be based on multiple attributes collected from multiple sources. Telemetry data, including location in GeoJSON format, will be pulled from each ship and loaded every hour. You want to have a dashboard that shows how many and which ships are likely to cause delays within a region. You want to use a storage solution that has native functionality for prediction and geospatial processing. Which storage solution should you use?

  • A. BigQuery Most Voted
  • B. Cloud Bigtable
  • C. Cloud Datastore
  • D. Cloud SQL for PostgreSQL

Odpowiedź: A.

BigQuery jest najlepszym rozwiązaniem w tym scenariuszu, ponieważ oferuje natywną obsługę zarówno analizy predykcyjnej (integracja z BigQuery ML), jak i przetwarzania danych geoprzestrzennych (funkcje GeoJSON).  40 TB danych to duża skala, a BigQuery jest zaprojektowany do obsługi takich wolumenów danych w sposób wydajny i skalowalny. Możliwość ładowania danych co godzinę i wykonywania zapytań w czasie zbliżonym do rzeczywistego jest kluczowa dla monitorowania i przewidywania opóźnień.  BigQuery ML pozwala na trenowanie modeli bezpośrednio w BigQuery, eliminując potrzebę przenoszenia danych do innych systemów.  Dodatkowo, BigQuery integruje się z narzędziami do wizualizacji danych, umożliwiając tworzenie dashboardów prezentujących wyniki predykcji.

A. Cloud Bigtable jest bazą danych NoSQL, zoptymalizowaną pod kątem dużych wolumenów danych i niskich opóźnień, ale nie oferuje natywnej obsługi predykcji ani przetwarzania geoprzestrzennego.
B. Cloud Datastore to baza danych NoSQL, odpowiednia dla aplikacji webowych i mobilnych, ale nie jest zoptymalizowana pod kątem analizy dużych zbiorów danych i nie posiada wbudowanych funkcji predykcyjnych ani geoprzestrzennych.
C. Cloud SQL for PostgreSQL to relacyjna baza danych, która może być używana do przechowywania danych, ale nie oferuje natywnej obsługi predykcji ani przetwarzania geoprzestrzennego na poziomie BigQuery.

Analiza i Wyjaśnienie

Sugerowana odpowiedź: A

BigQuery jest zdecydowanie najlepszym rozwiązaniem w tym scenariuszu. Pytanie wyraźnie wskazuje na potrzebę analizy predykcyjnej (model przewidujący opóźnienia) oraz przetwarzania danych geoprzestrzennych (dane telemetryczne w formacie GeoJSON). BigQuery oferuje natywną integrację z BigQuery ML, umożliwiającą trenowanie modeli bezpośrednio w systemie, oraz rozbudowane funkcje GIS do obsługi danych geoprzestrzennych. Skala danych (40 TB) jest również dobrze obsługiwana przez BigQuery, który jest zaprojektowany do analizy dużych zbiorów danych.

Dyskusja użytkowników w większości potwierdza tę odpowiedź. Większość głosów (100% w podanym rozkładzie) popiera BigQuery. Komentarze takie jak te od barnac1es, zellck, Atnafu i Chihhanyu podkreślają kluczowe zalety BigQuery w kontekście tego zadania: skalowalność, wsparcie dla SQL i ML, przetwarzanie geoprzestrzenne oraz integrację z narzędziami do wizualizacji.

Dlaczego inne odpowiedzi są nieprawidłowe:

  • Cloud Bigtable: Jest to baza danych NoSQL zoptymalizowana pod kątem niskich opóźnień i dużych wolumenów danych, ale brakuje jej natywnej obsługi predykcji i przetwarzania geoprzestrzennego. Komentarz musumusu początkowo sugerował BigTable, ale później sam się z niego wycofał, uznając, że pytanie dotyczy rozwiązania do przechowywania, a nie całego workflow.
  • Cloud Datastore: Jest to baza danych NoSQL odpowiednia dla aplikacji webowych i mobilnych, ale nie jest zoptymalizowana pod kątem analizy dużych zbiorów danych i nie posiada wbudowanych funkcji predykcyjnych ani geoprzestrzennych.
  • Cloud SQL for PostgreSQL: Jest to relacyjna baza danych, która może być używana do przechowywania danych, ale nie oferuje natywnej obsługi predykcji ani przetwarzania geoprzestrzennego na poziomie BigQuery.

Komentarz PM17 porusza kwestię wielkości danych, ale ostatecznie zgadza się, że BigQuery jest lepszym wyborem ze względu na funkcje ML i GIS. Pytanie koncentruje się na rozwiązaniu do przechowywania z natywnymi funkcjami, a BigQuery idealnie spełnia te kryteria.
