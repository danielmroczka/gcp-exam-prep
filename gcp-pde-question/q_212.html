You are troubleshooting your Dataflow pipeline that processes data from Cloud Storage to BigQuery. You have discovered that the Dataflow worker nodes cannot communicate with one another. Your networking team relies on Google Cloud network tags to define firewall rules. You need to identify the issue while following Google-recommended networking security practices. What should you do?

  • A. Determine whether your Dataflow pipeline has a custom network tag set.
  • B. Determine whether there is a firewall rule set to allow traffic on TCP ports 12345 and 12346 for the Dataflow network tag. Most Voted
  • C. Determine whether there is a firewall rule set to allow traffic on TCP ports 12345 and 12346 on the subnet used by Dataflow workers.
  • D. Determine whether your Dataflow pipeline is deployed with the external IP address option enabled.

Odpowiedź: B.

Wyjaśnienie: Dataflow wykorzystuje wewnętrzną komunikację między workerami do przetwarzania danych. Domyślnie, Dataflow używa portów TCP 12345 i 12346 do tej komunikacji. Skoro networking team używa tagów sieciowych do definiowania reguł firewall, to kluczowe jest sprawdzenie, czy istnieje reguła firewall zezwalająca na ruch na tych portach dla tagu sieciowego przypisanego do Dataflow. Brak takiej reguły uniemożliwi workerom komunikację, powodując problemy z działaniem pipeline. Zastosowanie tagów sieciowych jest zgodne z rekomendowanymi praktykami bezpieczeństwa Google Cloud, ponieważ pozwala na precyzyjne kontrolowanie dostępu do zasobów.

A. Sprawdzenie, czy pipeline ma ustawiony customowy tag sieciowy jest mniej istotne niż sprawdzenie, czy istnieją reguły firewall dla *jakiegokolwiek* tagu używanego przez Dataflow.
C. Reguły firewall powinny być definiowane na poziomie tagów sieciowych, a nie bezpośrednio na subnetach, co jest mniej elastyczne i trudniejsze w zarządzaniu.
D. Włączanie zewnętrznych adresów IP dla Dataflow jest generalnie niezalecane ze względów bezpieczeństwa i nie rozwiązuje problemu komunikacji wewnętrznej.

Dodatkowe źródła:

*   **Dataflow Networking Overview:** https://cloud.google.com/dataflow/docs/guides/networking
*   **Firewall rules overview:** https://cloud.google.com/vpc/docs/firewalls
