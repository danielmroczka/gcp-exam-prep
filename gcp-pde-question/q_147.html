You are implementing several batch jobs that must be executed on a schedule. These jobs have many interdependent steps that must be executed in a specific order. Portions of the jobs involve executing shell scripts, running Hadoop jobs, and running queries in BigQuery. The jobs are expected to run for many minutes up to several hours. If the steps fail, they must be retried a fixed number of times. Which service should you use to manage the execution of these jobs?

  • A. Cloud Scheduler
  • B. Cloud Dataflow
  • C. Cloud Functions
  • D. Cloud Composer Most Voted

Odpowiedź: D.

Cloud Composer to w pełni zarządzany przepływ pracy (workflow orchestration) oparty na Apache Airflow. Jest idealny do zarządzania złożonymi zadaniami wsadowymi z wieloma zależnościami, takimi jak te opisane w pytaniu. Pozwala na definiowanie zadań jako DAGów (Directed Acyclic Graphs), co umożliwia precyzyjne określenie kolejności wykonywania kroków, w tym skryptów shell, zadań Hadoop i zapytań BigQuery. Dodatkowo, Cloud Composer oferuje wbudowane mechanizmy retry dla nieudanych zadań, co jest kluczowe w tym scenariuszu.  Dzięki temu zapewnia niezawodne i powtarzalne wykonywanie długotrwałych procesów.

A. Cloud Scheduler służy do uruchamiania zadań w oparciu o harmonogram, ale nie oferuje zaawansowanych możliwości zarządzania zależnościami i retry.
B. Cloud Dataflow jest przeznaczony do przetwarzania strumieniowego i wsadowego danych, ale nie jest optymalny do orkiestracji złożonych przepływów pracy z różnymi typami zadań.
C. Cloud Functions to bezserwerowe funkcje, które są odpowiednie do prostych, krótkotrwałych zadań, a nie do długotrwałych, złożonych przepływów pracy z zależnościami.

Dodatkowe źródła:

*   **Cloud Composer Documentation:** https://cloud.google.com/composer/docs
*   **Apache Airflow:** https://airflow.apache.org/

Analiza i Wyjaśnienie

Sugerowana odpowiedź: D

Poprawną odpowiedzią jest Cloud Composer. Pytanie opisuje scenariusz wymagający orkiestracji złożonych zadań wsadowych z wieloma zależnościami między krokami, w tym wykonywania skryptów shell, zadań Hadoop i zapytań BigQuery. Cloud Composer, oparty na Apache Airflow, jest w pełni zarządzaną usługą do orkiestracji przepływów pracy, która idealnie nadaje się do tego typu zadań.

Kluczowe cechy Cloud Composer, które odpowiadają wymaganiom z pytania to:

  • Definiowanie zależności: Możliwość definiowania zależności między zadaniami za pomocą DAGów (Directed Acyclic Graphs).
  • Obsługa różnych typów zadań: Możliwość wykonywania skryptów shell, zadań Hadoop i zapytań BigQuery.
  • Mechanizm retry: Wbudowany mechanizm ponawiania prób w przypadku niepowodzenia zadań.
  • Harmonogramowanie: Możliwość planowania wykonywania zadań w oparciu o harmonogram.

Dyskusja użytkowników w większości potwierdza tę odpowiedź, z dominującą przewagą głosów (90%) dla Cloud Composer. Komentarze takie jak te od barnac1es, zellck, nadavw i daghayeghi podkreślają kluczowe zalety Cloud Composer w kontekście tego pytania.

Dlaczego inne odpowiedzi są nieprawidłowe:

  • Cloud Scheduler (A): Służy do uruchamiania zadań w oparciu o harmonogram, ale nie oferuje zaawansowanych możliwości zarządzania zależnościami i retry, które są kluczowe w tym scenariuszu.
  • Cloud Dataflow (B): Jest przeznaczony do przetwarzania strumieniowego i wsadowego danych, ale nie jest optymalny do orkiestracji złożonych przepływów pracy z różnymi typami zadań.
  • Cloud Functions (C): To bezserwerowe funkcje, które są odpowiednie do prostych, krótkotrwałych zadań, a nie do długotrwałych, złożonych przepływów pracy z zależnościami.

Chociaż początkowo niektórzy użytkownicy (np. Alasmindas, Abby1356, kavs) sugerowali Cloud Scheduler, późniejsze komentarze i analiza wskazują, że Cloud Composer jest znacznie lepszym rozwiązaniem ze względu na złożoność i zależności między zadaniami.
