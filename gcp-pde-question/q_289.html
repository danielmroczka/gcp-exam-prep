You have data located in BigQuery that is used to generate reports for your company. You have noticed some weekly executive report fields do not correspond to format according to company standards. For example, report errors include different telephone formats and different country code identifiers. This is a frequent issue, so you need to create a recurring job to normalize the data. You want a quick solution that requires no coding. What should you do?

  • A. Use Cloud Data Fusion and Wrangler to normalize the data, and set up a recurring job. Most Voted
  • B. Use Dataflow SQL to create a job that normalizes the data, and that after the first run of the job, schedule the pipeline to execute recurrently.
  • C. Create a Spark job and submit it to Dataproc Serverless.
  • D. Use BigQuery and GoogleSQL to normalize the data, and schedule recurring queries in BigQuery.

Odpowiedź: A.

Cloud Data Fusion w połączeniu z Wranglerem oferuje wizualny interfejs do transformacji danych, który pozwala na normalizację danych bez konieczności pisania kodu. Wrangler, będący częścią Cloud Data Fusion, specjalizuje się w profilowaniu danych i automatycznym proponowaniu transformacji, co idealnie pasuje do problemu z różnymi formatami danych (telefonów, kodów krajów). Można łatwo zdefiniować reguły normalizacji i zastosować je do danych w BigQuery. Następnie, Cloud Data Fusion umożliwia zaplanowanie regularnego uruchamiania tej transformacji jako zadania cyklicznego, co rozwiązuje problem częstych błędów w raportach. To szybkie i efektywne rozwiązanie, które spełnia wszystkie wymagania zadania.

B. Dataflow SQL wymaga znajomości SQL i konfiguracji potoku, co jest sprzeczne z wymogiem braku kodowania.
C. Spark i Dataproc Serverless wymagają pisania kodu w Pythonie, Scala lub Javie, co również nie spełnia kryterium braku kodowania.
D. BigQuery i GoogleSQL, choć mogą normalizować dane, wymagają pisania zapytań SQL, co nie jest rozwiązaniem "bez kodowania" i może być bardziej czasochłonne w przypadku złożonych transformacji.

Dodatkowe źródła:

*   **Cloud Data Fusion:** https://cloud.google.com/data-fusion
*   **Cloud Data Fusion Wrangler:** https://cloud.google.com/data-fusion/docs/how-to/use-wrangler
*   **BigQuery Scheduled Queries:** https://cloud.google.com/bigquery/docs/scheduled-queries

Sugerowana odpowiedź: A

Najbardziej odpowiednią odpowiedzią jest A, czyli użycie Cloud Data Fusion w połączeniu z Wranglerem do normalizacji danych i ustawienia cyklicznego zadania. Kluczowym wymaganiem zadania jest "szybkie rozwiązanie, które nie wymaga kodowania". Cloud Data Fusion z Wranglerem oferuje wizualny interfejs, który umożliwia transformację danych bez pisania kodu. Wrangler specjalizuje się w profilowaniu danych i automatycznym proponowaniu transformacji, co idealnie odpowiada na problem z różnymi formatami danych.

Dyskusja użytkowników potwierdza tę odpowiedź. Większość głosów (83%) popiera opcję A. Użytkownicy, którzy wybrali opcję A, podkreślają brak konieczności kodowania i możliwość wizualnego projektowania przepływu pracy normalizacji danych.

Dlaczego inne odpowiedzi są nieprawidłowe:

  • B (Dataflow SQL): Mimo że Dataflow SQL pozwala na transformację danych, wymaga znajomości SQL i konfiguracji potoku, co nie spełnia wymogu braku kodowania.
  • C (Spark i Dataproc Serverless): Wymaga pisania kodu w językach takich jak Python, Scala lub Java, co również nie spełnia kryterium braku kodowania.
  • D (BigQuery i GoogleSQL): Chociaż BigQuery i GoogleSQL mogą normalizować dane, pisanie zapytań SQL jest formą kodowania, co stoi w sprzeczności z wymaganiem "bez kodowania". Użytkownik RenePetersen słusznie zauważa, że pisanie transformacji SQL jest kodowaniem.

Należy zwrócić uwagę na komentarze użytkowników, którzy początkowo wskazywali na odpowiedź D, ale po dalszej dyskusji i uwzględnieniu wymogu braku kodowania, zmienili zdanie lub potwierdzili poprawność odpowiedzi A.
