Your company is planning to migrate a large on-premises data warehouse (e.g., Teradata or Netezza) to BigQuery. The data is currently stored in a proprietary format. You need to perform a batch migration of this data to BigQuery with minimal custom development. What should you do?

  • A. Use the bq command-line tool to load the data directly from the on-premises data warehouse.
  • B. Export the data to CSV files, upload the files to Cloud Storage, then load the files into BigQuery.
  • C. Use the BigQuery Data Transfer Service (Migration Service) to automate the transfer from the source data warehouse.
  • D. Use Datastream to replicate the data in real time from the on-premises warehouse.
Show Suggested Answer
Suggested Answer: C

Wyjaśnienie

Correct Answer: C

The BigQuery Data Transfer Service (DTS) includes a migration service specifically designed to automate the data movement from on-premises data warehouses like Teradata, Netezza, and Oracle to BigQuery. It handles schema mapping and data transfer, minimizing the need for custom scripts or manual export/import processes.

Why other options are incorrect:
A. The bq tool can load data from files, but it doesn't directly connect to and extract from proprietary on-premises data warehouses.
B. Manual export to CSV and then loading is a labor-intensive and error-prone process for large-scale migrations.
D. Datastream is primarily for real-time change data capture (CDC) from relational databases (MySQL, Oracle, PostgreSQL) into BigQuery or GCS, not for batch migration of large data warehouses.
