Question 12 / 319

Your company handles data processing for a number of different clients. Each client prefers to use their own suite of analytics tools, with some allowing direct query access via Google BigQuery. You need to secure the data so that clients cannot see each other's data. You want to ensure appropriate access to the data.
Which three steps should you take? (Choose three.)

  • A. Load data into different partitions.
  • B. Load data into a different dataset for each client. Most Voted
  • C. Put each client's BigQuery dataset into a different table.
  • D. Restrict a client's dataset to approved users. Most Voted
  • E. Only allow a service account to access the datasets.
  • F. Use the appropriate identity and access management (IAM) roles for each client's users. Most Voted

Odpowiedź: B, D, F.

Aby zabezpieczyć dane i zapobiec dostępowi klientów do danych innych klientów, należy zastosować izolację danych na poziomie zbioru (dataset). Umieszczenie danych każdego klienta w osobnym zbiorze w BigQuery jest kluczowe dla tej izolacji. Następnie, należy ograniczyć dostęp do zbiorów danych tylko do zatwierdzonych użytkowników i zapewnić właściwe role IAM dla każdego klienta. Te kroki skutecznie zapobiegną nieautoryzowanemu dostępowi.

A. Partycje służą do organizowania danych wewnątrz tabeli w celu optymalizacji zapytań, a nie do izolacji danych między klientami.
C. Umieszczanie danych każdego klienta w osobnych tabelach w ramach tego samego zbioru nadal nie zapewnia wystarczającej izolacji, ponieważ klienci mogliby mieć potencjalnie dostęp do całego zbioru.
E. Używanie tylko konta usługi ogranicza dostęp do wszystkich użytkowników, co jest sprzeczne z potrzebą dostępu dla poszczególnych klientów.

Sugerowana odpowiedź: B, D, F

Prawidłowe odpowiedzi to B, D, F.

Dlaczego?

Scenariusz wymaga izolacji danych wielu klientów ("multitenancy") przy jednoczesnym zapewnieniu im bezpośredniego dostępu (query access). Najlepszą architekturą w BigQuery dla tego przypadku jest:

  • B. Load data into a different dataset for each client: Dataset (Zbiór danych) to podstawowa jednostka uprawnień w BigQuery. Izolacja na poziomie datasetu jest czysta, bezpieczna i łatwa w zarządzaniu.
  • D. Restrict a client's dataset to approved users: (Wspiera B) Po utworzeniu datasetu należy skonfigurować jego ACL (Access Control List), aby tylko użytkownicy danej firmy mieli do niego dostęp.
  • F. Use appropriate IAM roles: (Wspiera B i D) Należy nadać użytkownikom klienta konkretne role (np. roles/bigquery.dataViewer) tylko do ich datasetu, a nie do całego projektu.

Dlaczego inne odpowiedzi są nieprawidłowe?

  • A. Partitions: Partycje służą wydajności, a nie bezpieczeństwu. Nie da się nadać uprawnień IAM do konkretnej partycji wewnątrz tabeli (przynajmniej nie w standardowy sposób bez Row-Level Security, które jest bardziej skomplikowane w utrzymaniu dla całkowitej separacji klientów).
  • C. Different table: Choć można mieć osobne tabele w jednym datasecie, zarządzanie uprawnieniami per-tabela jest znacznie trudniejsze (możliwe, ale uciążliwe przy setkach tabel) niż zarządzanie per-dataset. Izolacja na poziomie datasetu jest "Best Practice".
  • E. Only service account: Jeśli damy dostęp tylko kontu serwisowemu (używanemu przez naszą aplikację), klienci nie będą mogli wykonywać "direct query access" (bezpośrednich zapytań), co jest wymogiem w pytaniu.
