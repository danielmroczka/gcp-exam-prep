You store and analyze your relational data in BigQuery on Google Cloud with all data that resides in US regions. You also have a variety of object stores across Microsoft Azure and Amazon Web Services (AWS), also in US regions. You want to query all your data in BigQuery daily with as little movement of data as possible. What should you do?

  • A. Use BigQuery Data Transfer Service to load files from Azure and AWS into BigQuery.
  • B. Create a Dataflow pipeline to ingest files from Azure and AWS to BigQuery.
  • C. Load files from AWS and Azure to Cloud Storage with Cloud Shell gsutil rsync arguments.
  • D. Use the BigQuery Omni functionality and BigLake tables to query files in Azure and AWS. Most Voted

Odpowiedź: D.

BigQuery Omni i tabele BigLake pozwalają na bezpośrednie wykonywanie zapytań do danych przechowywanych w chmurach AWS i Azure bez konieczności ich kopiowania do BigQuery. BigLake zapewnia jednolity dostęp do danych, niezależnie od ich lokalizacji, a BigQuery Omni umożliwia wykonywanie zapytań w tych chmurach przy użyciu składni SQL BigQuery. To minimalizuje ruch danych, co jest kluczowe w wymaganiach zadania. Rozwiązanie to pozwala na analizę danych w miejscu ich przechowywania, co obniża koszty transferu i przyspiesza czas dostępu. Dzięki temu można codziennie wykonywać zapytania bez dużych operacji ETL.

A. BigQuery Data Transfer Service jest przeznaczone do transferu danych z konkretnych źródeł, a nie ogólnych obiektów w Azure i AWS, co wymagałoby dodatkowej konfiguracji i transferu danych.
B. Dataflow pipeline wymagałby napisania kodu i zarządzania infrastrukturą do przesyłania danych, co zwiększa złożoność i ruch danych.
C. Użycie Cloud Shell i `gsutil rsync` wymagałoby skopiowania danych do Cloud Storage, co jest sprzeczne z wymaganiem minimalnego ruchu danych.

Dodatkowe źródła:

*   **BigQuery Omni:** https://cloud.google.com/bigquery/docs/omni
*   **BigLake:** https://cloud.google.com/biglake/docs
*   **BigQuery Data Transfer Service:** https://cloud.google.com/bigquery/docs/data-transfer-service
*   **Dataflow:** https://cloud.google.com/dataflow

Sugerowana odpowiedź: D

Odpowiedź D, wykorzystująca BigQuery Omni i tabele BigLake, jest zdecydowanie najbardziej odpowiednia. Zgodnie z treścią pytania, kluczowe jest zminimalizowanie ruchu danych. BigQuery Omni umożliwia bezpośrednie wykonywanie zapytań do danych przechowywanych w Azure i AWS bez ich kopiowania do BigQuery. BigLake zapewnia jednolity dostęp do tych danych, niezależnie od ich lokalizacji. To rozwiązanie spełnia wszystkie wymagania zadania.

Komentarze użytkowników, w szczególności te z większą liczbą głosów (raaad, e70ea9e, JyoGCP) jednoznacznie potwierdzają poprawność odpowiedzi D, podkreślając możliwość analizy danych w miejscu ich przechowywania bez konieczności transferu.

Pozostałe odpowiedzi są nieprawidłowe z następujących powodów:

  • A (BigQuery Data Transfer Service): Wymaga transferu danych do BigQuery, co jest sprzeczne z wymaganiami zadania.
  • B (Dataflow pipeline): Wymaga napisania kodu i zarządzania infrastrukturą do przesyłania danych, co zwiększa złożoność i generuje ruch danych.
  • C (Cloud Shell gsutil rsync): Wymaga skopiowania danych do Cloud Storage, co również jest sprzeczne z wymaganiem minimalnego ruchu danych.

Użytkownik Ramon98 trafnie zauważył, że opcje A, B i C wiążą się z przemieszczaniem danych, co jest niedopuszczalne w kontekście tego pytania.
