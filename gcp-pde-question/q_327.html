Your company wants to implement a Retrieval-Augmented Generation (RAG) system to allow employees to query an extensive knowledge base of internal documents. You need to prepare this unstructured text for retrieval. What should you do to ensure the system can retrieve the most relevant information?

  • A. Use Cloud Data Loss Prevention (Cloud DLP) to scan and redact sensitive information.
  • B. Store the documents as compressed files in a traditional relational database.
  • C. Convert the unstructured documents into high-dimensional numerical vectors (embeddings) that capture semantic meaning.
  • D. Index each word from the documents into a search engine to enable keyword-based search.
Show Suggested Answer
Suggested Answer: C

Wyjaśnienie

Correct Answer: C

In a Retrieval-Augmented Generation (RAG) system, the retrieval step depends on finding pieces of text that are semantically similar to the user's query. This is achieved by converting both the documents and the query into high-dimensional numerical vectors (embeddings) using an embedding model (like Vertex AI Text Embeddings). Vector search allows the system to find relevant context even if the exact keywords don't match.

Why other options are incorrect:
A. While Cloud DLP is good for privacy, it's a preprocessing step for security, not for improving the quality of information retrieval.
B. Relational databases are not optimized for searching across unstructured text documents.
D. Keyword-based search (BM25, etc.) is less effective than semantic search for RAG systems because it misses context and synonyms that embeddings capture.
