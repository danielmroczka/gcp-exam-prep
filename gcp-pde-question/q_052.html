You are implementing security best practices on your data pipeline. Currently, you are manually executing jobs as the Project Owner. You want to automate these jobs by taking nightly batch files containing non-public information from Google Cloud Storage, processing them with a Spark Scala job on a Google Cloud
Dataproc cluster, and depositing the results into Google BigQuery.
How should you securely run this workload?

  • A. Restrict the Google Cloud Storage bucket so only you can see the files
  • B. Grant the Project Owner role to a service account, and run the job with it
  • C. Use a service account with the ability to read the batch files and to write to BigQuery Most Voted
  • D. Use a user account with the Project Viewer role on the Cloud Dataproc cluster to read the batch files and write to BigQuery

Odpowiedź: C.

Wyjaśnienie:
Opcja C jest najbezpieczniejszym i najbardziej zalecanym podejściem zgodnie z najlepszymi praktykami bezpieczeństwa w Google Cloud Platform. Do zautomatyzowanych zadań, takich jak nocne procesy wsadowe, zawsze należy używać kont usługi (service accounts), a nie kont użytkowników. Konto usługi pozwala na wykonywanie operacji w imieniu aplikacji lub maszyny. Kluczową zasadą bezpieczeństwa jest zasada najmniejszych uprawnień (Principle of Least Privilege), która mówi, że dany podmiot powinien mieć tylko te uprawnienia, które są absolutnie niezbędne do wykonania swojego zadania. W tym przypadku zadanie wymaga jedynie odczytu plików z Google Cloud Storage i zapisu wyników do Google BigQuery. Utworzenie konta usługi z precyzyjnie określonymi uprawnieniami do tych dwóch zasobów zapewnia, że klaster Dataproc, wykonujący zadanie Spark Scala, będzie miał niezbędny dostęp, jednocześnie minimalizując potencjalne ryzyka bezpieczeństwa związane z nadmiernymi uprawnieniami.

Pozostałe opcje są gorsze lub błędne:
A. Ograniczenie widoczności plików w zasobniku Google Cloud Storage tylko dla właściciela projektu nie rozwiązuje problemu, jak zautomatyzowane zadanie ma uzyskać do nich dostęp ani jak ma zapisać dane do BigQuery, nadal wymagając bezpiecznego sposobu uwierzytelnienia i autoryzacji dla samego zadania.
B. Nadanie roli Właściciela Projektu (Project Owner) kontu usługi stanowi poważne naruszenie zasady najmniejszych uprawnień, ponieważ rola ta daje pełen dostęp administracyjny do wszystkich zasobów w projekcie, co jest znacznie więcej niż potrzeba do wykonania tego konkretnego zadania.
D. Używanie konta użytkownika do zautomatyzowanych zadań jest niewłaściwą praktyką, a rola Przeglądającego Projektu (Project Viewer) nie zapewnia uprawnień do zapisu danych do BigQuery, co jest wymagane przez zadanie.

Dodatkowe źródła:
*   Zasada najmniejszych uprawnień
*   Konta usługi (Service Accounts) w Google Cloud

Sugerowana odpowiedź: C

Odpowiedź C jest najbezpieczniejszym i najbardziej zgodnym z najlepszymi praktykami rozwiązaniem. Kluczowe jest użycie konta usługi (service account) z minimalnymi wymaganymi uprawnieniami (zasada najmniejszych uprawnień). Konto usługi powinno mieć dostęp do odczytu plików z Google Cloud Storage oraz do zapisu danych do Google BigQuery. Wykorzystanie konta usługi pozwala na automatyzację zadań bez konieczności używania kont użytkowników, co zwiększa bezpieczeństwo.

Dyskusja użytkowników potwierdza tę odpowiedź. Wiele osób, w tym nowsze komentarze (np. od ThomasChoy, Bhawantha, medeis_jar), poprawnie wskazuje, że konto usługi z odpowiednimi uprawnieniami jest najlepszym rozwiązaniem. Komentarze podkreślają, że rola Project Owner (odpowiedź B) jest zbyt szeroka i narusza zasadę najmniejszych uprawnień. Dodatkowo, komentarze wskazują, że konto usługi może być używane do uruchamiania zadań Dataproc, a inne konta usługi mogą być używane do uruchamiania zadań (Cron, Composer) z różnymi uprawnieniami.

Odpowiedź A jest niewłaściwa, ponieważ ograniczenie dostępu tylko do właściciela projektu nie umożliwia automatyzacji zadań. Odpowiedź D jest niewłaściwa, ponieważ rola Project Viewer nie zapewnia uprawnień do zapisu do BigQuery, a używanie kont użytkowników do automatyzacji jest złą praktyką.
