You have an Oracle database deployed in a VM as part of a Virtual Private Cloud (VPC) network. You want to replicate and continuously synchronize 50 tables to BigQuery. You want to minimize the need to manage infrastructure. What should you do?

  • A. Deploy Apache Kafka in the same VPC network, use Kafka Connect Oracle Change Data Capture (CDC), and Dataflow to stream the Kafka topic to BigQuery.
  • B. Create a Pub/Sub subscription to write to BigQuery directly. Deploy the Debezium Oracle connector to capture changes in the Oracle database, and sink to the Pub/Sub topic.
  • C. Deploy Apache Kafka in the same VPC network, use Kafka Connect Oracle change data capture (CDC), and the Kafka Connect Google BigQuery Sink Connector.
  • D. Create a Datastream service from Oracle to BigQuery, use a private connectivity configuration to the same VPC network, and a connection profile to BigQuery. Most Voted

Odpowiedź: D.

Datastream to usługa Google Cloud zaprojektowana specjalnie do replikacji danych z różnych źródeł, w tym Oracle, do BigQuery. Wykorzystanie Datastream minimalizuje potrzebę zarządzania infrastrukturą, ponieważ Google zajmuje się orkiestracją i skalowaniem. Konfiguracja prywatnej łączności w tej samej VPC zapewnia bezpieczny transfer danych bez wystawiania ich na publiczny internet. Profil połączenia z BigQuery upraszcza proces konfiguracji i utrzymania połączenia. Datastream oferuje również ciągłą synchronizację danych, co jest kluczowe dla spełnienia wymagań zadania.

A. Rozwiązanie z Kafką wymaga zarządzania klastrem Kafka, Kafka Connect oraz Dataflow, co zwiększa złożoność operacyjną.
B. Bezpośrednie pisanie do BigQuery z Pub/Sub może być mniej wydajne i trudniejsze w konfiguracji dla ciągłej synchronizacji dużych ilości danych.
C. Użycie Kafka Connect i BigQuery Sink Connector również wymaga zarządzania infrastrukturą Kafka i konfiguracji konektorów, co jest bardziej skomplikowane niż Datastream.

Analiza i Wyjaśnienie

Sugerowana odpowiedź: D

Odpowiedź D jest zdecydowanie poprawna i najbardziej efektywna w kontekście minimalizacji zarządzania infrastrukturą. Usługa Google Cloud Datastream została zaprojektowana specjalnie do replikacji danych z baz danych Oracle do BigQuery. Wykorzystuje ona mechanizm Change Data Capture (CDC) do ciągłej synchronizacji danych, a konfiguracja prywatnej łączności w tej samej VPC zapewnia bezpieczeństwo transferu danych.

Kluczowe korzyści z użycia Datastream w tym scenariuszu to:

  • Brak konieczności zarządzania infrastrukturą: Datastream jest usługą zarządzaną, co oznacza, że Google zajmuje się orkiestracją, skalowaniem i utrzymaniem.
  • Ciągła synchronizacja: Datastream zapewnia ciągłą replikację danych, co jest kluczowe dla utrzymania aktualności danych w BigQuery.
  • Bezpieczeństwo: Prywatna łączność w VPC zapewnia bezpieczny transfer danych.
  • Uproszczona konfiguracja: Profile połączeń upraszczają proces konfiguracji i utrzymania połączenia z BigQuery.

Dyskusja użytkowników potwierdza poprawność odpowiedzi D. Użytkownicy, w tym scaenruy (autor pytania), raaad i JyoGCP, jednogłośnie wskazują Datastream jako najlepsze rozwiązanie. Zek dodaje link do tutorialu pokazującego replikację danych z Oracle do BigQuery przy użyciu Cloud Data Fusion, który jest oparty na Datastream.

Dlaczego inne odpowiedzi są nieprawidłowe:

  • A i C: Rozwiązania oparte na Apache Kafka i Kafka Connect wymagają zarządzania klastrem Kafka, Kafka Connect oraz konfiguracji konektorów, co znacznie zwiększa złożoność operacyjną i obciążenie administracyjne.
  • B: Bezpośrednie pisanie do BigQuery z Pub/Sub może być mniej wydajne i trudniejsze w konfiguracji dla ciągłej synchronizacji dużych ilości danych, szczególnie w porównaniu do dedykowanej usługi Datastream.
