You work for a manufacturing plant that batches application log files together into a single log file once a day at 2:00 AM. You have written a Google Cloud
Dataflow job to process that log file. You need to make sure the log file in processed once per day as inexpensively as possible. What should you do?

  • A. Change the processing job to use Google Cloud Dataproc instead.
  • B. Manually start the Cloud Dataflow job each morning when you get into the office.
  • C. Create a cron job with Google App Engine Cron Service to run the Cloud Dataflow job. Most Voted
  • D. Configure the Cloud Dataflow job as a streaming job so that it processes the log data immediately.

Odpowiedź: C.

**Wyjaśnienie:**
Pliki logów są agregowane w jeden plik raz dziennie o godzinie 2:00 AM, co idealnie pasuje do przetwarzania wsadowego. Celem jest uruchamianie zadania Dataflow raz dziennie i jak najtaniej. Usługa Google App Engine Cron Service (lub nowszy Google Cloud Scheduler) jest standardowym i najbardziej efektywnym kosztowo sposobem na automatyczne planowanie zadań cyklicznych w Google Cloud. Skonfigurowanie zadania cron tak, aby uruchamiało job Dataflow krótko po 2:00 AM każdego dnia, zapewnia niezawodne i automatyczne przetwarzanie bez konieczności interwencji manualnej, spełniając tym samym wszystkie wymagania.

**Dlaczego pozostałe opcje są gorsze/błędne:**
*   **A. Zmień zadanie przetwarzania, aby używało zamiast tego Google Cloud Dataproc.** Zmiana technologii przetwarzania nie rozwiązuje problemu planowania, a jeśli zadanie Dataflow jest już napisane i działa, Dataproc może wprowadzić dodatkowe koszty i złożoność.
*   **B. Ręcznie uruchamiaj zadanie Cloud Dataflow każdego ranka, gdy dotrzesz do biura.** Ręczne uruchamianie jest zawodne, podatne na błędy ludzkie, nieefektywne kosztowo pod względem czasu pracy i nie spełnia wymagań automatyzacji oraz niezawodności.
*   **D. Skonfiguruj zadanie Cloud Dataflow jako zadanie strumieniowe, aby przetwarzało dane logów natychmiast.** Zadanie strumieniowe jest przeznaczone do ciągłego przetwarzania danych w czasie rzeczywistym i byłoby znacznie droższe oraz nieefektywne w tym scenariuszu, ponieważ dane pojawiają się tylko raz dziennie w formie pojedynczego pliku wsadowego.

**Dodatkowe źródła:**
*   Dokumentacja Google Cloud Scheduler: https://cloud.google.com/scheduler/docs
*   Dokumentacja App Engine Cron Service: https://cloud.google.com/appengine/docs/standard/python/scheduling-jobs-with-cron

Sugerowana odpowiedź: C

Poprawną odpowiedzią jest C, czyli użycie Google App Engine Cron Service (lub nowszej usługi Google Cloud Scheduler) do uruchomienia zadania Dataflow. Zadanie Dataflow przetwarza pliki logów, które są generowane raz dziennie o 2:00 AM. Automatyczne uruchamianie zadania o określonej godzinie jest najbardziej efektywne kosztowo i niezawodne. Usługa Cloud Scheduler (która zastąpiła App Engine Cron Service) jest idealna do tego celu.

Uruchomienie zadania Dataflow jako strumieniowego (D) jest nieefektywne, ponieważ dane są dostępne tylko raz dziennie w formie wsadowej.
