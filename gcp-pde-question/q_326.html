Your team runs a complex analytical query daily that processes terabytes of data. Recently, after running for 20 minutes, the query fails with a “Resources exceeded” error. You need to resolve this issue and ensure the query completes. What should you do?

  • A. Move from BigQuery on-demand to slot reservations (capacity-based billing).
  • B. Analyze the SQL syntax for errors.
  • C. Increase the maximum table size limit.
  • D. Increase your project's BigQuery API request quota.
Show Suggested Answer
Suggested Answer: A

Wyjaśnienie

Correct Answer: A

A “Resources exceeded” error in BigQuery typically means the query is too complex or requires more compute capacity (slots) than what is available in the shared on-demand pool (which is capped at 2,000 slots per project). Switching to slot reservations (Editions) allows you to allocate more dedicated slots to your project, providing the necessary resources for heavy analytical queries.

Why other options are incorrect:
B. The query is failing due to resource limits, not syntax errors. If there were syntax errors, it would fail immediately, not after 20 minutes.
C. There is no "maximum table size limit" that causes a "Resources exceeded" error during query execution.
D. API request quotas limit the number of API calls per second, not the internal compute resources used by a single long-running query.
