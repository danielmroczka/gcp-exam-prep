You have several Spark jobs that run on a Cloud Dataproc cluster on a schedule. Some of the jobs run in sequence, and some of the jobs run concurrently. You need to automate this process. What should you do?

  • A. Create a Cloud Dataproc Workflow Template
  • B. Create an initialization action to execute the jobs
  • C. Create a Directed Acyclic Graph in Cloud Composer Most Voted
  • D. Create a Bash script that uses the Cloud SDK to create a cluster, execute jobs, and then tear down the cluster

Odpowiedź: C.

Stworzenie Directed Acyclic Graph (DAG) w Cloud Composer jest najlepszym rozwiązaniem do automatyzacji sekwencyjnych i współbieżnych zadań Spark na klastrze Cloud Dataproc. Cloud Composer, oparty na Apache Airflow, pozwala na definiowanie zależności między zadaniami, co umożliwia precyzyjne kontrolowanie kolejności ich wykonywania. DAG wizualizuje przepływ pracy, ułatwiając monitorowanie i zarządzanie złożonymi procesami. Dzięki temu można łatwo zautomatyzować harmonogram zadań, uwzględniając zarówno sekwencję, jak i współbieżność, co jest kluczowe w opisanym scenariuszu. Cloud Composer integruje się natywnie z usługami Google Cloud, w tym z Cloud Dataproc, co upraszcza konfigurację i uruchamianie zadań.

A. Cloud Dataproc Workflow Templates są bardziej odpowiednie do powtarzalnych zadań o stałej konfiguracji, a nie do złożonych przepływów pracy z zależnościami.
B. Initialization actions służą do konfigurowania klastra podczas jego tworzenia, a nie do automatyzacji wykonywania zadań po uruchomieniu klastra.
D. Bash script jest mniej elastyczny i trudniejszy w utrzymaniu niż DAG w Cloud Composer, szczególnie w przypadku złożonych zależności i monitoringu.

Dodatkowe źródła:

*   **Cloud Composer Documentation:** https://cloud.google.com/composer/docs
*   **Apache Airflow Documentation:** https://airflow.apache.org/docs/
*   **Cloud Dataproc and Cloud Composer Integration:** https://cloud.google.com/composer/docs/concepts/integrations/dataproc

Sugerowana odpowiedź: C

Najlepszym rozwiązaniem do automatyzacji sekwencyjnych i współbieżnych zadań Spark na klastrze Cloud Dataproc jest stworzenie Directed Acyclic Graph (DAG) w Cloud Composer. Cloud Composer, bazujący na Apache Airflow, oferuje zaawansowane możliwości orkiestracji przepływów pracy, w tym definiowanie zależności między zadaniami, co pozwala na precyzyjne kontrolowanie kolejności ich wykonywania i obsługę zadań współbieżnych. Dyskusja użytkowników, szczególnie te nowsze komentarze (z ostatnich miesięcy), konsekwentnie potwierdzają tę odpowiedź, wskazując na integrację Cloud Composer z Dataproc i jego zdolność do obsługi złożonych harmonogramów.

Kluczowe argumenty za odpowiedzią C:

  • Orkiestracja złożonych przepływów pracy: Cloud Composer umożliwia definiowanie zależności między zadaniami, co jest niezbędne do obsługi sekwencyjnych i współbieżnych zadań.
  • Harmonogramowanie: Cloud Composer oferuje wbudowane możliwości harmonogramowania zadań, w tym uruchamianie ich w określonych odstępach czasu lub w odpowiedzi na zdarzenia.
  • Integracja z Dataproc: Cloud Composer integruje się natywnie z Cloud Dataproc, co upraszcza konfigurację i uruchamianie zadań Spark.

Analiza innych odpowiedzi:

  • A. Cloud Dataproc Workflow Templates: Chociaż Workflow Templates mogą automatyzować uruchamianie zadań, brakuje im możliwości definiowania zależności i koordynowania zadań współbieżnych. Użytkownicy wskazują, że same w sobie nie posiadają mechanizmu harmonogramowania.
  • B. Initialization action: Akcje inicjalizacyjne służą do konfigurowania klastra podczas jego tworzenia, a nie do automatyzacji wykonywania zadań po uruchomieniu klastra.
  • D. Bash script: Skrypt Bash może być niewystarczający do zarządzania złożonymi zależnościami i monitorowania, a także trudniejszy w utrzymaniu niż DAG w Cloud Composer.

Wiele komentarzy wskazuje, że początkowo rozważano odpowiedź A, ale ostatecznie uznano ją za niewystarczającą ze względu na brak możliwości harmonogramowania i zarządzania zależnościami. Link do dokumentacji Cloud Composer i Dataproc, udostępniony przez użytkowników, dodatkowo potwierdza, że Cloud Composer jest zalecanym rozwiązaniem do automatyzacji złożonych przepływów pracy z Dataproc.
