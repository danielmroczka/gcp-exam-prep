You are preparing an organization-wide dataset. You need to preprocess customer data stored in a restricted bucket in Cloud Storage. The data will be used to create consumer analyses. You need to comply with data privacy requirements.

What should you do?

  • A. Use Dataflow and the Cloud Data Loss Prevention API to mask sensitive data. Write the processed data in BigQuery. Most Voted
  • B. Use customer-managed encryption keys (CMEK) to directly encrypt the data in Cloud Storage. Use federated queries from BigQuery. Share the encryption key by following the principle of least privilege.
  • C. Use the Cloud Data Loss Prevention API and Dataflow to detect and remove sensitive fields from the data in Cloud Storage. Write the filtered data in BigQuery.
  • D. Use Dataflow and Cloud KMS to encrypt sensitive fields and write the encrypted data in BigQuery. Share the encryption key by following the principle of least privilege.

Odpowiedź: A.

Wykorzystanie Dataflow w połączeniu z Cloud Data Loss Prevention (DLP) API jest najlepszym rozwiązaniem, ponieważ pozwala na skuteczne maskowanie danych wrażliwych podczas przetwarzania. Dataflow umożliwia skalowalne przetwarzanie danych w chmurze, a Cloud DLP API identyfikuje i modyfikuje dane wrażliwe zgodnie z zdefiniowanymi regułami (np. maskowanie numerów kart kredytowych, adresów e-mail). Przetworzone, zanonimizowane dane mogą być bezpiecznie zapisane w BigQuery do dalszych analiz. To podejście zapewnia zgodność z wymaganiami dotyczącymi prywatności danych, jednocześnie umożliwiając wykorzystanie danych do celów analitycznych. Zapisanie danych w BigQuery pozwala na łatwe i efektywne wykonywanie zapytań i tworzenie raportów.

A. Jest to poprawne, ponieważ maskowanie danych jest bardziej odpowiednie dla analiz konsumenckich niż całkowite usuwanie.
B. CMEK chroni dane w spoczynku, ale nie rozwiązuje problemu wrażliwych danych widocznych podczas przetwarzania i analizy.
C. Usunięcie wrażliwych pól może prowadzić do utraty istotnych informacji potrzebnych do analizy, a samo usunięcie nie gwarantuje zgodności z przepisami o prywatności.
D. Szyfrowanie wrażliwych pól nie ukrywa ich treści, a jedynie utrudnia dostęp do nich osobom nieupoważnionym, co nie spełnia wymagań dotyczących prywatności w kontekście analiz konsumenckich.

Dodatkowe źródła:

*   **Cloud Data Loss Prevention (DLP):** https://cloud.google.com/dlp
*   **Dataflow:** https://cloud.google.com/dataflow
*   **BigQuery:** https://cloud.google.com/bigquery
*   **Cloud KMS:** https://cloud.google.com/kms

Analiza i Wyjaśnienie

Sugerowana odpowiedź: A

Odpowiedź A jest najbardziej odpowiednia, ponieważ łączy w sobie skalowalność przetwarzania danych (Dataflow) z możliwościami identyfikacji i modyfikacji danych wrażliwych (Cloud DLP API). Maskowanie danych, w przeciwieństwie do usuwania, pozwala na zachowanie użyteczności danych do celów analitycznych, co jest kluczowe w kontekście tworzenia analiz konsumenckich. Użytkownik AlizCert trafnie zauważa, że całkowite usunięcie danych PII uniemożliwiłoby rozróżnienie między klientami, co zniweczyłoby cel analizy.

Komentarze użytkowników, szczególnie te nowsze (np. raaad, Matt_108), potwierdzają poprawność odpowiedzi A, podkreślając jej zalety w zakresie ochrony prywatności, redukcji wrażliwości danych i zachowania ich użyteczności. Użytkownik raaad dobrze podsumowuje korzyści płynące z maskowania.

Dlaczego inne odpowiedzi są nieprawidłowe:

  • B: Szyfrowanie CMEK chroni dane w spoczynku, ale nie rozwiązuje problemu danych wrażliwych podczas przetwarzania i analizy. Samo szyfrowanie nie ukrywa treści danych.
  • C: Usunięcie wrażliwych pól może prowadzić do utraty istotnych informacji potrzebnych do analizy. Jak zauważa AlizCert, całkowite usunięcie PII uniemożliwia analizę na poziomie pojedynczego klienta.
  • D: Szyfrowanie za pomocą Cloud KMS, podobnie jak CMEK, nie maskuje danych wrażliwych, a jedynie utrudnia dostęp do nich osobom nieupoważnionym.

Użytkownik ML6 słusznie zauważa, że dane w Cloud Storage są domyślnie szyfrowane, co dodatkowo podkreśla, że odpowiedź B i D nie adresują głównego problemu, jakim jest ochrona prywatności podczas przetwarzania i analizy danych.
