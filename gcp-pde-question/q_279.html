You want to store your team’s shared tables in a single dataset to make data easily accessible to various analysts. You want to make this data readable but unmodifiable by analysts. At the same time, you want to provide the analysts with individual workspaces in the same project, where they can create and store tables for their own use, without the tables being accessible by other analysts. What should you do?

  • A. Give analysts the BigQuery Data Viewer role at the project level. Create one other dataset, and give the analysts the BigQuery Data Editor role on that dataset.
  • B. Give analysts the BigQuery Data Viewer role at the project level. Create a dataset for each analyst, and give each analyst the BigQuery Data Editor role at the project level.
  • C. Give analysts the BigQuery Data Viewer role on the shared dataset. Create a dataset for each analyst, and give each analyst the BigQuery Data Editor role at the dataset level for their assigned dataset. Most Voted
  • D. Give analysts the BigQuery Data Viewer role on the shared dataset. Create one other dataset and give the analysts the BigQuery Data Editor role on that dataset.

Odpowiedź: C.

Opcja C jest poprawna, ponieważ zapewnia zarówno dostęp do odczytu do wspólnych tabel, jak i izolację przestrzeni roboczych dla analityków. Przyznanie roli BigQuery Data Viewer na wspólnym zbiorze danych pozwala analitykom przeglądać dane, ale nie modyfikować ich. Tworzenie oddzielnych zbiorów danych dla każdego analityka i przyznanie im roli BigQuery Data Editor na poziomie zbioru danych daje im pełną kontrolę nad swoimi własnymi tabelami, jednocześnie uniemożliwiając dostęp do nich innym analitykom. To rozwiązanie spełnia wszystkie wymagania zadania: dostęp do odczytu do wspólnych danych, brak możliwości ich modyfikacji przez analityków oraz izolowane przestrzenie robocze.

A. Przyznanie roli Data Editor na poziomie projektu dałoby analitykom możliwość modyfikacji wszystkich danych w projekcie, co jest sprzeczne z wymaganiami.
B. Przyznanie roli Data Editor na poziomie projektu również dałoby analitykom zbyt szerokie uprawnienia i naruszyłoby izolację przestrzeni roboczych.
D. Tworzenie jednego zbioru danych dla analityków z rolą Data Editor pozwoliłoby im modyfikować dane w tym zbiorze, co nie spełnia wymagań dotyczących braku możliwości modyfikacji danych przez analityków.

Sugerowana odpowiedź: C

Odpowiedź C jest prawidłowa, ponieważ w pełni spełnia wszystkie wymagania postawione w pytaniu. Kluczowe jest zapewnienie dostępu do odczytu do wspólnych tabel dla wszystkich analityków, przy jednoczesnym braku możliwości ich modyfikacji. Rola BigQuery Data Viewer na wspólnym zbiorze danych gwarantuje to. Dodatkowo, pytanie wymaga izolacji przestrzeni roboczych dla każdego analityka, gdzie będą mogli tworzyć i przechowywać własne tabele bez dostępu dla innych. Tworzenie oddzielnych zbiorów danych dla każdego analityka i przyznanie im roli BigQuery Data Editor na poziomie ich zbioru danych zapewnia tę izolację i pełną kontrolę nad ich własnymi zasobami.

Komentarze użytkowników, takie jak Sofiia98, dodatkowo podkreślają, że analitycy nie mają wglądu w indywidualne zbiory danych innych analityków, co jest istotne dla spełnienia wymagań dotyczących izolacji.
