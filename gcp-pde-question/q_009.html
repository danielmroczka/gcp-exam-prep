Your company is using WILDCARD tables to query data across multiple tables with similar names. The SQL statement is currently failing with the following error:

Which table name will make the SQL statement work correctly?

Odpowiedź: D.

Wyjaśnienie: W BigQuery, aby użyć wildcardów do zapytania tabel o podobnych nazwach, należy zastosować znak gwiazdki (`*`) bezpośrednio po nazwie częściowej, bez ujmowania jej w cudzysłów. Opcja D poprawnie używa `gsod*` jako części nazwy tabeli, wskazując na wszystkie tabele zaczynające się od "gsod". Taki zapis pozwala BigQuery na dynamiczne rozpoznanie i skonsolidowanie danych z wielu tabel pasujących do wzorca. Jest to standardowy mechanizm stosowany w zapytaniach wielotabelowych przy użyciu wildcardów.

Dlaczego pozostałe opcje są gorsze/błędne:

A. `'bigquery-public-data.noaa_gsod.gsod'` – Podanie dokładnej nazwy tabeli bez wildcarda nie jest użyciem mechanizmu WILDCARD i odnosi się tylko do jednej konkretnej tabeli, co może nie spełniać celu zapytania "wielu tabel".
B. `bigquery-public-data.noaa_gsod.gsod*` – Brak cudzysłowów wokół pełnej nazwy tabeli w tym kontekście może być interpretowany przez niektóre narzędzia lub dialekty SQL jako próba użycia symbolu specjalnego w nazwie tabeli, a nie jako wildcard.
C. `'bigquery-public-data.noaa_gsod.gsod'*` – Ujęcie wildcarda w cudzysłów traktuje go jako literalny znak, a nie jako symbol do rozszerzenia nazwy tabeli, co uniemożliwia poprawne dopasowanie wielu tabel.

Analiza i Wyjaśnienie

Sugerowana odpowiedź: D

Prawidłową odpowiedzią jest D. `bigquery-public-data.noaa_gsod.gsod*` (z użyciem backticków `).

Dlaczego?

W Google BigQuery (Standard SQL), aby odpytać wiele tabel przy użyciu znaku wieloznacznego (wildcard), nazwa tabeli zawierająca gwiazdkę musi być ujęta w backticki (`). Jest to wymóg składniowy, ponieważ gwiazdka `*` jest znakiem specjalnym. Poprawne zapytanie wyglądałoby tak: SELECT * FROM `bigquery-public-data.noaa_gsod.gsod*` WHERE ...

Warto zauważyć, że w opcjach odpowiedzi często pojawiają się błędy typograficzne (np. użycie pojedynczego cudzysłowu ' zamiast backticka `). Jednak intencją pytania jest sprawdzenie wiedzy o konieczności umieszczenia gwiazdki *wewnątrz* identyfikatora tabeli, co robi tylko opcja B i D, przy czym D (w domyśle poprawiona na backticki) jest formą akceptowaną przez egzamin jako "ta z cudzysłowem". W rzeczywistości BQ wymaga backticków.

Dlaczego inne odpowiedzi są nieprawidłowe?

  • A. Brak gwiazdki: Odpyta tylko jedną tabelę o nazwie `gsod`, a nie zestaw tabel.
  • B. Brak otoczenia znakiem (cudzysłów/backtick): Spowoduje błąd składni (Syntax Error), parser SQL nie zrozumie gwiazdki w tym miejscu.
  • C. Gwiazdka poza cudzysłowem: Traktuje string jako nazwę i próbuje pomnożyć? To niepoprawna składnia dla Wildcard Tables.
