You are running a streaming pipeline with Dataflow and are using hopping windows to group the data as the data arrives. You noticed that some data is arriving late but is not being marked as late data, which is resulting in inaccurate aggregations downstream. You need to find a solution that allows you to capture the late data in the appropriate window. What should you do?

  • A. Use watermarks to define the expected data arrival window. Allow late data as it arrives. Most Voted
  • B. Change your windowing function to tumbling windows to avoid overlapping window periods.
  • C. Change your windowing function to session windows to define your windows based on certain activity.
  • D. Expand your hopping window so that the late data has more time to arrive within the grouping.

Odpowiedź: A.

Wykorzystanie watermarków jest kluczowe w przetwarzaniu strumieniowym, aby radzić sobie z opóźnionymi danymi. Watermarki definiują punkt w czasie, po którym nie oczekuje się już przychodzenia danych dla danego okna. Pozwalają one Dataflow na prawidłowe oznaczanie danych jako opóźnionych i uwzględnianie ich w obliczeniach, nawet jeśli przyjdą po zamknięciu okna. Dzięki temu agregacje downstream będą dokładniejsze, ponieważ uwzględnią wszystkie dostępne dane, a nie tylko te, które dotarły na czas. Watermarki są dynamiczne i mogą być aktualizowane w miarę przetwarzania strumienia, co pozwala na adaptację do zmieniających się warunków sieciowych i opóźnień.

B. Tumbling windows eliminują nakładanie się okien, ale nie rozwiązują problemu opóźnionych danych – dane, które przyjdą po zamknięciu okna, zostaną po prostu odrzucone.
C. Session windows są przydatne do grupowania danych na podstawie aktywności, ale nie rozwiązują problemu opóźnionych danych, a wręcz mogą go pogłębić, jeśli aktywność jest sporadyczna.
D. Rozszerzenie hopping window może pomóc w pewnych przypadkach, ale jest to rozwiązanie nieefektywne i może prowadzić do nadmiernego zużycia zasobów, ponieważ okna będą musiały przechowywać dane przez dłuższy czas.

Sugerowana odpowiedź: A

Odpowiedź A jest prawidłowa. Kluczowym elementem w przetwarzaniu strumieniowym, szczególnie w przypadku opóźnionych danych, są watermarki. Watermarki definiują moment w czasie, po którym Dataflow przestaje oczekiwać nowych danych dla danego okna czasowego. Pozwalają one na uwzględnienie opóźnionych danych w obliczeniach, co jest niezbędne do uzyskania dokładnych agregacji. Umożliwiają również konfigurację potoku w celu akceptowania i prawidłowego przetwarzania danych, które przychodzą po przekroczeniu watermarku, umieszczając je w odpowiednim oknie.

Większość użytkowników (Matt_108, Sofiia98, scaenruy, JyoGCP, Pime13, raaad, m_a_p_s) wskazuje odpowiedź A i odwołuje się do dokumentacji Google Cloud Dataflow, która szczegółowo wyjaśnia koncepcję watermarków i ich zastosowanie w przetwarzaniu strumieniowym.

Dlaczego inne odpowiedzi są nieprawidłowe:

  • B: Tumbling windows nie rozwiązują problemu opóźnionych danych, a wręcz je ignorują.
  • C: Session windows są przydatne w innych scenariuszach, ale nie pomagają w radzeniu sobie z opóźnionymi danymi.
  • D: Rozszerzenie okna czasowego może pomóc, ale jest to nieefektywne rozwiązanie, które prowadzi do zwiększonego zużycia zasobów.
