You are migrating your data warehouse to BigQuery. You have migrated all of your data into tables in a dataset. Multiple users from your organization will be using the data. They should only see certain tables based on their team membership. How should you set user permissions?

  • A. Assign the users/groups data viewer access at the table level for each table Most Voted
  • B. Create SQL views for each team in the same dataset in which the data resides, and assign the users/groups data viewer access to the SQL views
  • C. Create authorized views for each team in the same dataset in which the data resides, and assign the users/groups data viewer access to the authorized views
  • D. Create authorized views for each team in datasets created for each team. Assign the authorized views data viewer access to the dataset in which the data resides. Assign the users/groups data viewer access to the datasets in which the authorized views reside

Odpowiedź: A.

Najprostszym i najbardziej efektywnym sposobem na ograniczenie dostępu do tabel w BigQuery dla różnych zespołów jest przypisanie użytkownikom/grupom uprawnień "Data Viewer" na poziomie tabeli. Pozwala to na precyzyjną kontrolę, które tabele każdy zespół może przeglądać, bez konieczności tworzenia dodatkowych widoków czy zestawów danych. Jest to również łatwe w utrzymaniu i skalowaniu, szczególnie gdy liczba tabel i zespołów rośnie.  BigQuery oferuje granularne uprawnienia, które umożliwiają zarządzanie dostępem do danych w sposób elastyczny i bezpieczny.  Użycie uprawnień na poziomie tabeli minimalizuje ryzyko nieautoryzowanego dostępu do wrażliwych danych.

B. Tworzenie standardowych widoków SQL nie zapewnia mechanizmu kontroli dostępu – dostęp do widoku oznacza dostęp do danych, które widok reprezentuje.
C. Authorized views są przydatne do udostępniania danych zewnętrznym użytkownikom, a nie do kontrolowania dostępu wewnątrz organizacji.
D. Tworzenie oddzielnych zestawów danych dla każdego zespołu jest niepraktyczne i utrudnia zarządzanie danymi oraz potencjalnie zwiększa koszty przechowywania.

Analiza i Wyjaśnienie

Sugerowana odpowiedź: A

Najbardziej poprawną odpowiedzią jest A. Dyskusja użytkowników początkowo była podzielona, ale z biegiem czasu, a zwłaszcza w nowszych komentarzach (od 2023 roku), przeważa opinia, że BigQuery umożliwia teraz przypisywanie uprawnień na poziomie tabeli. Użytkownik squishy_fishy wskazuje na dokumentację Google Cloud, potwierdzającą tę możliwość. Dodatkowo, użytkownicy SamuelTsch, zellck i MaxNRG również potwierdzają, że opcja A jest obecnie poprawna.

Przypisanie uprawnień "Data Viewer" na poziomie tabeli jest najprostszym i najbardziej efektywnym sposobem na ograniczenie dostępu do danych dla różnych zespołów. Pozwala to na precyzyjną kontrolę, które tabele każdy zespół może przeglądać, bez konieczności tworzenia dodatkowych widoków czy zestawów danych. Jest to również łatwe w utrzymaniu i skalowaniu.

Dlaczego inne odpowiedzi są nieprawidłowe:

  • B: Standardowe widoki SQL nie zapewniają mechanizmu kontroli dostępu.
  • C: Authorized views są bardziej odpowiednie do udostępniania danych zewnętrznym użytkownikom lub do kontroli dostępu na poziomie wierszy/kolumn, a nie do podstawowego podziału dostępu na tabele wewnątrz organizacji.
  • D: Tworzenie oddzielnych zestawów danych dla każdego zespołu jest niepraktyczne, zwiększa koszty i komplikuje zarządzanie danymi. Dodatkowo, jak zauważa squishy_fishy, pytanie wyraźnie mówi o jednym zestawie danych.

Warto zauważyć, że pierwotnie (w marcu 2020 roku, kiedy pytanie zostało opublikowane) opcja A mogła być nieprawidłowa. Jednak funkcjonalność BigQuery została rozszerzona, umożliwiając teraz granularne uprawnienia na poziomie tabeli.
