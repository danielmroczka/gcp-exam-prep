You have data pipelines running on BigQuery, Dataflow, and Dataproc. You need to perform health checks and monitor their behavior, and then notify the team managing the pipelines if they fail. You also need to be able to work across multiple projects. Your preference is to use managed products or features of the platform. What should you do?

  • A. Export the information to Cloud Monitoring, and set up an Alerting policy Most Voted
  • B. Run a Virtual Machine in Compute Engine with Airflow, and export the information to Cloud Monitoring
  • C. Export the logs to BigQuery, and set up App Engine to read that information and send emails if you find a failure in the logs
  • D. Develop an App Engine application to consume logs using GCP API calls, and send emails if you find a failure in the logs

Odpowiedź: A.

Cloud Monitoring jest natywnym, w pełni zarządzanym rozwiązaniem do monitorowania i alertowania w Google Cloud Platform. Pozwala na zbieranie metryk, logów i zdarzeń z BigQuery, Dataflow i Dataproc bez konieczności zarządzania infrastrukturą. Ustawienie polityki alertowania w Cloud Monitoring umożliwia automatyczne powiadamianie zespołu w przypadku wykrycia awarii lub nieprawidłowego zachowania pipeline'ów. Co ważne, Cloud Monitoring działa w wielu projektach, co spełnia wymaganie zadania. Wykorzystanie zarządzanych usług minimalizuje nakład pracy operacyjnej i zapewnia skalowalność.

A. Jest to najprostsze i najbardziej efektywne rozwiązanie, wykorzystujące natywne narzędzia GCP.
B. Użycie maszyny wirtualnej z Airflow wprowadza dodatkowy element do zarządzania i nie jest preferowanym podejściem, gdy dostępne są zarządzane usługi.
C. Eksportowanie logów do BigQuery i użycie App Engine do ich analizy jest bardziej skomplikowane i mniej efektywne niż bezpośrednie wykorzystanie Cloud Monitoring.
D. Rozwijanie aplikacji App Engine do konsumowania logów przez API jest bardziej czasochłonne i wymaga więcej kodu niż wykorzystanie Cloud Monitoring.
