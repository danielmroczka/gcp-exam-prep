Your large e-commerce company uses BigQuery for its data warehousing needs. They have two primary types of workloads: - Analyst dashboards: Interactive queries throughout the day requiring low latency. - ETL jobs: Large, complex batch queries that load and transform data overnight. Analysts are reporting inconsistent performance during peak business hours. You need to ensure reliable performance for both workload types while optimizing costs. What should you do?

  • A. Use BigQuery editions to create an autoscaling reservation with dedicated slots for analyst dashboards, while running ETL jobs with a separate reservation or using the on-demand billing model.
  • B. Implement a custom autoscaling solution using Cloud Run functions to dynamically adjust available BigQuery slots based on time of day.
  • C. Migrate all ETL jobs to Cloud SQL to reduce BigQuery's workload.
  • D. Run all analyst dashboard queries as interactive queries and configure ETL jobs to run as batch queries.
Show Suggested Answer
Suggested Answer: A

Wyjaśnienie

Correct Answer: A

BigQuery Reservations (now part of BigQuery Editions) allow you to allocate dedicated capacity (slots) to specific workloads. By creating separate reservations for Analyst Dashboards and ETL jobs, you ensure that high-priority interactive queries aren't starved of resources by background ETL tasks. Autoscaling within a reservation allows BigQuery to dynamically adjust the number of slots based on demand, optimizing costs.

Why other options are incorrect:
B. Custom autoscaling with Cloud Run is complex and unnecessary, as BigQuery now provides native autoscaling reservations.
C. Cloud SQL is not suitable for large-scale data warehousing and complex ETL jobs that are currently running in BigQuery.
D. While batch queries are scheduled to run when resources are available, they don't guarantee performance for interactive queries if the system is overall congested. Reservations provide more robust isolation.
