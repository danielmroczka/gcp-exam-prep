You need to modernize your existing on-premises data strategy. Your organization currently uses:
• Apache Hadoop clusters for processing multiple large data sets, including on-premises Hadoop Distributed File System (HDFS) for data replication.
• Apache Airflow to orchestrate hundreds of ETL pipelines with thousands of job steps.

You need to set up a new architecture in Google Cloud that can handle your Hadoop workloads and requires minimal changes to your existing orchestration processes. What should you do?

  • A. Use Bigtable for your large workloads, with connections to Cloud Storage to handle any HDFS use cases. Orchestrate your pipelines with Cloud Composer.
  • B. Use Dataproc to migrate Hadoop clusters to Google Cloud, and Cloud Storage to handle any HDFS use cases. Orchestrate your pipelines with Cloud Composer. Most Voted
  • C. Use Dataproc to migrate Hadoop clusters to Google Cloud, and Cloud Storage to handle any HDFS use cases. Convert your ETL pipelines to Dataflow.
  • D. Use Dataproc to migrate your Hadoop clusters to Google Cloud, and Cloud Storage to handle any HDFS use cases. Use Cloud Data Fusion to visually design and deploy your ETL pipelines.

Odpowiedź: B.

Dataproc jest usługą Google Cloud zaprojektowaną specjalnie do uruchamiania obciążeń Hadoop i Spark, co pozwala na płynną migrację istniejących klastrów Hadoop bez konieczności przepisywania aplikacji. Cloud Storage może efektywnie zastąpić HDFS, zapewniając skalowalność i niezawodność przechowywania danych. Cloud Composer jest zarządzaną instancją Apache Airflow, co minimalizuje zmiany w istniejących procesach orkiestracji ETL. Wykorzystanie tych usług pozwala na modernizację infrastruktury Hadoop w chmurze Google przy zachowaniu istniejących procesów i minimalizując nakład pracy.

A. Bigtable nie jest odpowiednikiem Hadoop do przetwarzania dużych zbiorów danych, a Cloud Storage nie w pełni zastąpi funkcjonalności HDFS.
C. Konwersja istniejących pipeline'ów Airflow do Dataflow wymagałaby znacznych zmian w kodzie i architekturze.
D. Cloud Data Fusion, choć potężne narzędzie ETL, wymagałoby przeprojektowania istniejących pipeline'ów Airflow, co jest sprzeczne z wymaganiem minimalnych zmian.

Analiza i Wyjaśnienie

Sugerowana odpowiedź: B

Odpowiedź B jest prawidłowa, ponieważ najlepiej odpowiada na wszystkie wymagania zadania. Organizacja potrzebuje zmodernizować swoją infrastrukturę Hadoop, minimalizując zmiany w istniejących procesach orkiestracji ETL. Dataproc to usługa Google Cloud zaprojektowana do uruchamiania obciążeń Hadoop i Spark, umożliwiając migrację istniejących klastrów Hadoop bez przepisywania aplikacji. Cloud Storage skutecznie zastępuje HDFS, zapewniając skalowalność i niezawodność przechowywania danych. Co kluczowe, Cloud Composer jest zarządzaną instancją Apache Airflow, co oznacza, że istniejące pipeline'y Airflow mogą być używane z minimalnymi zmianami.

Komentarze użytkowników w dużej mierze potwierdzają tę odpowiedź. Większość wskazuje na Dataproc, Cloud Storage i Cloud Composer jako kluczowe elementy rozwiązania. Użytkownik 'datasmg' trafnie podsumowuje: "You can use Dataproc for doing Apache Hadoop process, then Cloud Storage to replace the HDFS, and using Cloud Composer (built in Apache Airflow) for orchestrator."

Dlaczego inne odpowiedzi są nieprawidłowe:

  • A: Bigtable nie jest odpowiednikiem Hadoop do przetwarzania dużych zbiorów danych, a Cloud Storage nie w pełni zastąpi funkcjonalności HDFS.
  • C: Konwersja istniejących pipeline'ów Airflow do Dataflow wymagałaby znacznych zmian w kodzie i architekturze, co jest sprzeczne z wymaganiem minimalnych zmian.
  • D: Cloud Data Fusion, choć potężne narzędzie ETL, wymagałoby przeprojektowania istniejących pipeline'ów Airflow, co również jest sprzeczne z wymaganiem minimalnych zmian.
