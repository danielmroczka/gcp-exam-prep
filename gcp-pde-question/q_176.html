You have uploaded 5 years of log data to Cloud Storage. A user reported that some data points in the log data are outside of their expected ranges, which indicates errors. You need to address this issue and be able to run the process again in the future while keeping the original data for compliance reasons. What should you do?

  • A. Import the data from Cloud Storage into BigQuery. Create a new BigQuery table, and skip the rows with errors.
  • B. Create a Compute Engine instance and create a new copy of the data in Cloud Storage. Skip the rows with errors.
  • C. Create a Dataflow workflow that reads the data from Cloud Storage, checks for values outside the expected range, sets the value to an appropriate default, and writes the updated records to a new dataset in Cloud Storage. Most Voted
  • D. Create a Dataflow workflow that reads the data from Cloud Storage, checks for values outside the expected range, sets the value to an appropriate default, and writes the updated records to the same dataset in Cloud Storage.

Odpowiedź: C.

Opcja C jest najlepszym rozwiązaniem, ponieważ wykorzystuje Dataflow do transformacji danych w sposób nieniszczący. Dataflow pozwala na odczyt danych z Cloud Storage, identyfikację błędnych wartości, ich korektę (ustawienie wartości domyślnych) i zapisanie *zmodyfikowanych* danych do *nowego* zbioru danych w Cloud Storage. To spełnia wymagania zadania: adresuje problem błędnych danych, umożliwia ponowne uruchomienie procesu w przyszłości i zachowuje oryginalne dane dla celów zgodności. Użycie Dataflow zapewnia skalowalność i elastyczność w przetwarzaniu dużych zbiorów danych logów.  Dodatkowo, tworzenie nowego zbioru danych minimalizuje ryzyko utraty oryginalnych danych w przypadku błędów podczas transformacji.

A. Import do BigQuery i pomijanie błędnych wierszy nie zachowuje oryginalnych danych, co jest sprzeczne z wymaganiami zgodności.
B. Tworzenie instancji Compute Engine i kopiowanie danych jest mniej skalowalne i bardziej kosztowne niż użycie Dataflow.
D. Zapisywanie zmodyfikowanych danych do tego samego zbioru danych w Cloud Storage nadpisuje oryginalne dane, co narusza wymagania zgodności.

Dodatkowe źródła:

*   **Google Cloud Dataflow:** https://cloud.google.com/dataflow
*   **BigQuery:** https://cloud.google.com/bigquery
*   **Cloud Storage:** https://cloud.google.com/storage

Sugerowana odpowiedź: C

Odpowiedź C jest zdecydowanie najlepszym rozwiązaniem, ponieważ spełnia wszystkie wymagania zadania. Kluczowe jest zachowanie oryginalnych danych dla celów zgodności (compliance), co wyklucza opcje A i D, które modyfikują lub usuwają dane. Opcja B, choć zachowuje oryginalne dane, jest mniej efektywna i skalowalna niż Dataflow, szczególnie przy dużych zbiorach danych logów. Dataflow umożliwia transformację danych w sposób nieniszczący, odczytując dane z Cloud Storage, korygując błędy i zapisując zmodyfikowane dane do nowego zbioru danych w Cloud Storage.

Komentarze użytkowników w dużej mierze potwierdzają tę ocenę. Większość głosów popiera odpowiedź C, a argumenty zawarte w komentarzach (np. od MaxNRG, AWSandeep) trafnie wskazują na wady pozostałych opcji. Komentarz FP77 zwraca uwagę na nieprecyzyjne sformułowanie "dataset" w kontekście Cloud Storage, ale mimo to poprawnie identyfikuje odpowiedź C jako najbardziej adekwatną.

Komentarz ea2023 sugeruje opcję D z włączoną wersjonowaniem w Cloud Storage. Choć wersjonowanie jest dobrym rozwiązaniem, nie eliminuje problemu potencjalnego nadpisania danych podczas transformacji. Dataflow z zapisem do nowego zbioru danych zapewnia większe bezpieczeństwo i klarowność.

Podsumowując, Dataflow jest narzędziem zaprojektowanym do przetwarzania strumieniowego i wsadowego dużych zbiorów danych, co czyni go idealnym wyborem do tego zadania. Zapewnia skalowalność, elastyczność i możliwość zachowania oryginalnych danych, co jest kluczowe dla spełnienia wymagań zadania.
