Odpowiedź: B

Wyjaśnienie: Cloud Composer jest zaprojektowany specjalnie do orkiestracji danych w środowisku hybrydowym, co oznacza, że łączy w sobie funkcje zarządzania przepływem danych, automatyzacji i integracji z różnymi usługami chmurowymi.  Umożliwia to efektywne zarządzanie i monitorowanie całego procesu przetwarzania danych, w tym transfer danych między usługami różnych dostawców chmury.  Cloud Composer oferuje wbudowane mechanizmy do zarządzania workflow, co jest kluczowe w złożonych scenariuszach hybrydowych.  Ponadto, jest zaprojektowany z myślą o skalowalności i wydajności, co jest istotne dla dużych i dynamicznych danych.  W przeciwieństwie do innych opcji, Cloud Composer jest dedykowany do orkiestracji, a nie tylko do przetwarzania danych, co czyni go idealnym wyborem dla tego zadania.

Pozostałe opcje są mniej odpowiednie: A. Cloud Dataflow - Chociaż Cloud Dataflow może być używany do przetwarzania danych, nie jest to dedykowany orkiestrator, a raczej narzędzie do przetwarzania danych w ramach przepływu. B. Cloud Composer - Jest to dedykowany orkiestrator, który zapewnia centralne zarządzanie i monitorowanie całego procesu przetwarzania danych. C. Cloud Dataprep - Cloud Dataprep jest narzędziem do czyszczenia i przygotowania danych, a nie do orkiestracji przepływu danych. D. Cloud Dataproc - Cloud Dataproc jest narzędziem do uruchamiania uruchomionych aplikacji, a nie do orkiestracji przepływu danych.

Analiza i Wyjaśnienie

Sugerowana odpowiedź: B

Prawidłową odpowiedzią jest B. Cloud Composer.

Dlaczego?

Wymagania: "hybrid cloud initiative", "complex data pipeline that moves data between cloud provider services", "orchestrate the entire pipeline". Cloud Composer (zarządzany Apache Airflow) jest najlepszym narzędziem do orkiestracji w środowiskach multi-cloud i hybrydowych. Airflow posiada ogromną bibliotekę operatorów (konektorów), które pozwalają na komunikację nie tylko z usługami Google Cloud, ale również z AWS, Azure oraz systemami on-premise. Dzięki temu możesz stworzyć jeden centralny proces (DAG), który wyzwala zadanie w chmurze A, czeka na wynik, a następnie przesyła go do chmury B.

Dlaczego inne odpowiedzi są nieprawidłowe?

  • A. Cloud Dataflow: Służy do przetwarzania danych (ETL/Streaming). Choć może czytać/pisać do różnych źródeł, nie jest orkiestratorem workflow, który zarządza wysokopoziomową logiką między różnymi, niezależnymi usługami.
  • C. Cloud Dataprep: Narzędzie do wizualnego czyszczenia danych (Data Wrangling). Nie służy do orkiestracji globalnych potoków hybrydowych.
  • D. Cloud Dataproc: Usługa do uruchamiania klastrów Hadoop/Spark. To narzędzie wykonawcze dla konkretnego typu obliczeń, a nie system do zarządzania całym przepływem pracy między różnymi dostawcami chmury.
