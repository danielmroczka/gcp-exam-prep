A TensorFlow machine learning model on Compute Engine virtual machines (n2-standard-32) takes two days to complete training. The model has custom TensorFlow operations that must run partially on a CPU. You want to reduce the training time in a cost-effective manner. What should you do?

  • A. Change the VM type to n2-highmem-32.
  • B. Change the VM type to e2-standard-32.
  • C. Train the model using a VM with a GPU hardware accelerator. Most Voted
  • D. Train the model using a VM with a TPU hardware accelerator.

Odpowiedź: C.

Wykorzystanie GPU (Graphics Processing Unit) jako akceleratora sprzętowego jest najbardziej efektywnym sposobem na przyspieszenie treningu modeli uczenia maszynowego, szczególnie tych opartych na TensorFlow. GPU są zoptymalizowane do wykonywania operacji macierzowych, które są kluczowe w procesie uczenia maszynowego. Chociaż model wymaga częściowego działania na CPU, większość obliczeń może być delegowana do GPU, co znacząco skróci czas treningu. Wybór GPU jest bardziej opłacalny niż TPU, szczególnie jeśli model nie jest w pełni zoptymalizowany pod kątem TPU. Dodatkowo, GPU oferują dobrą elastyczność i są szeroko wspierane przez TensorFlow.

A. Zmiana na n2-highmem-32 zwiększy dostępną pamięć, ale nie przyspieszy obliczeń, a więc nie skróci czasu treningu.
B. E2-standard-32 to tańsza opcja, ale oferuje mniejszą moc obliczeniową niż n2-standard-32, co prawdopodobnie wydłuży czas treningu.
D. TPU (Tensor Processing Unit) są bardzo wydajne, ale droższe i wymagają znacznej optymalizacji modelu, aby w pełni wykorzystać ich potencjał, co może być nieopłacalne w tym przypadku.

*   **Google Cloud TPUs:** https://cloud.google.com/tpu
*   **Compute Engine Machine Types:** https://cloud.google.com/compute/docs/machine-types

Sugerowana odpowiedź: C

Najbardziej efektywnym i opłacalnym rozwiązaniem w celu skrócenia czasu treningu modelu TensorFlow, który częściowo musi działać na CPU, jest wykorzystanie maszyny wirtualnej z akceleratorem sprzętowym GPU. Dyskusja użytkowników w zdecydowanej większości (100% głosów) potwierdza tę odpowiedź. Użytkownicy, w tym gudiking, Atnafu, zellck i inni, powołują się na dokumentację Google Cloud dotyczącą TPU, wskazując, że GPU są bardziej odpowiednie dla modeli z dużą liczbą niestandardowych operacji TensorFlow, które muszą działać przynajmniej częściowo na CPU. GPU są zoptymalizowane do operacji macierzowych, które są kluczowe w uczeniu maszynowym, a jednocześnie pozwalają na uruchamianie kodu na CPU, gdy jest to wymagane.

Dlaczego inne odpowiedzi są nieprawidłowe:

  • A (n2-highmem-32): Zwiększenie pamięci nie przyspieszy obliczeń, a więc nie skróci czasu treningu.
  • B (e2-standard-32): Tańsza opcja, ale oferuje mniejszą moc obliczeniową, co prawdopodobnie wydłuży czas treningu.
  • D (TPU): Chociaż TPU są bardzo wydajne, są droższe od GPU i wymagają znacznej optymalizacji modelu, aby w pełni wykorzystać ich potencjał. Dokumentacja Google Cloud wskazuje, że GPU są lepszym wyborem, gdy model zawiera niestandardowe operacje TensorFlow, które muszą działać na CPU.

Kluczowym fragmentem informacji w treści pytania jest wymóg, aby model działał częściowo na CPU. To eliminuje TPU jako optymalne rozwiązanie, ponieważ TPU są zoptymalizowane pod kątem pełnego wykorzystania i mogą mieć problemy z operacjami CPU.
