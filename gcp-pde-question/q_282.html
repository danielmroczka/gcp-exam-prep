You are using a Dataflow streaming job to read messages from a message bus that does not support exactly-once delivery. Your job then applies some transformations, and loads the result into BigQuery. You want to ensure that your data is being streamed into BigQuery with exactly-once delivery semantics. You expect your ingestion throughput into BigQuery to be about 1.5 GB per second. What should you do?

  • A. Use the BigQuery Storage Write API and ensure that your target BigQuery table is regional. Most Voted
  • B. Use the BigQuery Storage Write API and ensure that your target BigQuery table is multiregional.
  • C. Use the BigQuery Streaming API and ensure that your target BigQuery table is regional.
  • D. Use the BigQuery Streaming API and ensure that your target BigQuery table is multiregional.

Odpowiedź: A.

Wykorzystanie BigQuery Storage Write API w połączeniu z regionalną tabelą BigQuery jest najlepszym rozwiązaniem dla osiągnięcia dokładnie-raz (exactly-once) semantyki dostarczania danych w Dataflow, szczególnie przy wysokiej przepustowości (1.5 GB/s). Storage Write API oferuje transakcyjne zapisy, co gwarantuje, że dane są zapisywane w BigQuery tylko raz, nawet w przypadku awarii. Regionalna lokalizacja tabeli minimalizuje opóźnienia i potencjalne problemy z replikacją, co jest kluczowe dla wysokiej przepustowości.  Dataflow automatycznie obsługuje deduplikację i ponowne próby w przypadku błędów, zapewniając integralność danych.  Ważne jest, aby pamiętać, że Storage Write API jest zoptymalizowane pod kątem wysokiej przepustowości i niskich opóźnień.

A. Jest to gorsze, ponieważ Storage Write API jest bardziej wydajne i zapewnia dokładnie-raz semantykę.
B. Jest to gorsze, ponieważ multiregionalna tabela może wprowadzać dodatkowe opóźnienia i komplikacje w kontekście wysokiej przepustowości.
C. Jest to gorsze, ponieważ BigQuery Streaming API nie gwarantuje dokładnie-raz semantyki dostarczania danych.
D. Jest to gorsze, ponieważ połączenie Streaming API i multiregionalnej tabeli nie rozwiązuje problemu dokładnie-raz semantyki i może pogorszyć wydajność.
