Your company is using WILDCARD tables to query data across multiple tables with similar names (e.g., `table_20230101`, `table_20230102`). You want to use the `_TABLE_SUFFIX` pseudo-column in your `WHERE` clause to filter the tables efficiently. Which SQL statement syntax is correct for filtering tables from the last 7 days?

  • A. SELECT * FROM `project.dataset.table_*` WHERE _TABLE_SUFFIX BETWEEN '20230101' AND '20230107'
  • B. SELECT * FROM `project.dataset.table_*` WHERE table_name LIKE '202301%'
  • C. SELECT * FROM `project.dataset.table_*` WHERE _PARTITIONTIME > TIMESTAMP_SUB(CURRENT_TIMESTAMP(), INTERVAL 7 DAY)
  • D. SELECT * FROM `project.dataset.table_*` WHERE suffix >= '20230101'
Show Suggested Answer
Suggested Answer: A

Wyjaśnienie

Correct Answer: A

When using BigQuery wildcard tables, the `_TABLE_SUFFIX` pseudo-column contains the portion of the table name matched by the asterisk (*). You can use this column in a `WHERE` clause to limit the number of tables scanned. The standard syntax is to compare the `_TABLE_SUFFIX` (which is a string) against the desired values or range using `BETWEEN` or comparison operators.

Why other options are incorrect:
B. `table_name` is not a pseudo-column provided by BigQuery for wildcard filtering; `_TABLE_SUFFIX` is the correct one.
C. `_PARTITIONTIME` is used for ingestion-time partitioned tables, not for wildcard tables where multiple physical tables exist.
D. The pseudo-column name must be exactly `_TABLE_SUFFIX`, not just `suffix`.
