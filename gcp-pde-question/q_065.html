You are building a data pipeline on Google Cloud. You need to prepare data using a casual method for a machine-learning process. You want to support a logistic regression model. You also need to monitor and adjust for null values, which must remain real-valued and cannot be removed. What should you do?

  • A. Use Cloud Dataprep to find null values in sample source data. Convert all nulls to 'none' using a Cloud Dataproc job.

Odpowiedź: B.

**Wyjaśnienie:**
Cloud Dataprep jest idealnym narzędziem do „nieformalnego” (casual) i wizualnego przygotowania danych, pozwalając na szybką identyfikację brakujących wartości w próbkach danych. Zgodnie z wymaganiami, wartości null muszą pozostać liczbami rzeczywistymi (real-valued), co wyklucza zamianę ich na tekst. Zastąpienie wartości null zerem (0) w Cloud Dataprep jest prostą operacją, która spełnia ten wymóg i pozwala na zachowanie rekordów w modelu regresji logistycznej. Regresja logistyczna wymaga danych numerycznych, a Dataprep umożliwia monitorowanie rozkładu tych wartości w czasie rzeczywistym przed uruchomieniem pełnego zadania przetwarzania. Jest to rozwiązanie szybsze i bardziej intuicyjne niż pisanie dedykowanego kodu w Dataflow czy Dataproc.

**Dlaczego pozostałe opcje są gorsze/błędne:**
* **A:** Ta opcja jest błędna, ponieważ konwersja wartości null na ciąg znaków 'none' sprawia, że dane przestają być numeryczne (real-valued), co uniemożliwi ich bezpośrednie użycie w modelu regresji logistycznej.
* **C:** Użycie Cloud Dataflow do samego znajdowania wartości null jest zbyt skomplikowane w porównaniu do wizualnego interfejsu Dataprep, a konwersja na 'none' narusza wymóg zachowania wartości liczbowych.
* **D:** Choć konwersja na 0 jest poprawna merytorycznie, użycie Cloud Dataflow i niestandardowych skryptów nie pasuje do wymogu „nieformalnej metody” (casual method), gdyż wymaga znacznie większego nakładu pracy programistycznej niż Dataprep.

**Dodatkowe źródła:**
* Dokumentacja Cloud Dataprep - obsługa wartości brakujących
* Zasady przygotowania danych pod regresję logistyczną

Analiza i Wyjaśnienie

Sugerowana odpowiedź: B

Prawidłową odpowiedzią jest B. Use Cloud Dataprep to find null values... Convert all nulls to 0....

Dlaczego?

Wymagania: 1. "Casual method": Sugeruje użycie narzędzia wizualnego, szybkiego, niewymagającego pisania złożonego kodu (low-code/no-code). Cloud Dataprep (od Trifacta) jest właśnie takim narzędziem do eksploracji i czyszczenia danych. 2. "Support logistic regression": Model ten wymaga danych numerycznych. 3. "Null values... must remain real-valued": Wartości NULL muszą zostać zastąpione liczbą (imputacja), a nie usunięte. Zero (0) jest liczbą rzeczywistą ("real-valued"), natomiast 'none' jest stringiem. 4. "Monitor...": Dataprep oferuje wizualne profilowanie danych (histogramy), co pozwala monitorować rozkład wartości null.

Dlaczego inne odpowiedzi są nieprawidłowe?

  • A. Convert to 'none': 'none' to tekst. Regresja logistyczna na wejściu numerycznym wyrzuci błąd przy stringu.
  • C. Cloud Dataflow + 'none': Dataflow to narzędzie inżynierskie (high-code), mniej "casual" niż Dataprep. Konwersja na 'none' jest błędna (patrz A).
  • D. Cloud Dataflow + custom script: Pisanie skryptów w Dataflow nie jest "casual method". Jest to poprawne technicznie, ale bardziej pracochłonne niż wyklikanie tego w Dataprep.
