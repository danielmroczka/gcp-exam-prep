You are migrating your on-premises data warehouse to BigQuery. One of the upstream data sources resides on a MySQL. database that runs in your on-premises data center with no public IP addresses. You want to ensure that the data ingestion into BigQuery is done securely and does not go through the public internet. What should you do?

  • A. Update your existing on-premises ETL tool to write to BigQuery by using the BigQuery Open Database Connectivity (ODBC) driver. Set up the proxy parameter in the simba.googlebigqueryodbc.ini file to point to your data center’s NAT gateway.
  • B. Use Datastream to replicate data from your on-premises MySQL database to BigQuery. Set up Cloud Interconnect between your on-premises data center and Google Cloud. Use Private connectivity as the connectivity method and allocate an IP address range within your VPC network to the Datastream connectivity configuration. Use Server-only as the encryption type when setting up the connection profile in Datastream. Most Voted
  • C. Use Datastream to replicate data from your on-premises MySQL database to BigQuery. Use Forward-SSH tunnel as the connectivity method to establish a secure tunnel between Datastream and your on-premises MySQL database through a tunnel server in your on-premises data center. Use None as the encryption type when setting up the connection profile in Datastream.
  • D. Use Datastream to replicate data from your on-premises MySQL database to BigQuery. Gather Datastream public IP addresses of the Google Cloud region that will be used to set up the stream. Add those IP addresses to the firewall allowlist of your on-premises data center. Use IP Allowlisting as the connectivity method and Server-only as the encryption type when setting up the connection profile in Datastream.

Odpowiedź: B.

Opcja B jest poprawna, ponieważ wykorzystuje Cloud Interconnect do stworzenia prywatnego połączenia między Twoim centrum danych a Google Cloud, eliminując potrzebę przesyłania danych przez publiczny Internet. Datastream, w połączeniu z Private connectivity, zapewnia bezpieczną replikację danych z MySQL do BigQuery. Użycie Server-only encryption dodatkowo zabezpiecza dane podczas transferu. Cloud Interconnect gwarantuje stabilne i bezpieczne połączenie, idealne do regularnej replikacji danych. Konfiguracja IP address range w VPC zapewnia izolację i kontrolę nad ruchem sieciowym.

Analiza i Wyjaśnienie

Sugerowana odpowiedź: B

Odpowiedź B jest prawidłowa, ponieważ spełnia wszystkie wymagania zadania. Kluczowe jest zapewnienie bezpiecznego transferu danych z on-premise MySQL do BigQuery bez przechodzenia przez publiczny Internet. Cloud Interconnect tworzy dedykowane, prywatne połączenie sieciowe między Twoim centrum danych a Google Cloud, eliminując potrzebę korzystania z publicznej infrastruktury. Datastream, w połączeniu z opcją Private connectivity, umożliwia bezpieczną replikację danych. Wybór Server-only encryption dodatkowo zabezpiecza dane podczas przesyłania.

Użytkownicy w dyskusji jednogłośnie potwierdzają poprawność odpowiedzi B, wskazując na korzyści płynące z wykorzystania Datastream i Cloud Interconnect. Użytkownik raaad podkreśla, że Cloud Interconnect zapewnia prywatne i bezpieczne połączenie, a Datastream umożliwia bezpieczny transfer danych. Użytkownik datapassionate dodaje, że Datastream jest bezproblemowym rozwiązaniem do replikacji danych bezpośrednio do BigQuery.

Dlaczego inne odpowiedzi są nieprawidłowe:

  • A: Użycie NAT gateway nadal wymaga przechodzenia przez publiczny Internet, co narusza wymóg bezpieczeństwa.
  • C: Forward-SSH tunnel jest rozwiązaniem bardziej skomplikowanym w utrzymaniu i skalowaniu, a także mniej wydajnym niż dedykowane połączenie Cloud Interconnect.
  • D: IP Allowlisting, mimo ograniczenia dostępu, nadal naraża dane na potencjalne zagrożenia związane z publicznym Internetem.

Podsumowując, odpowiedź B jest najbardziej kompleksowym i bezpiecznym rozwiązaniem, które spełnia wszystkie wymagania zadania, co potwierdzają zarówno sugerowana odpowiedź, jak i dyskusja użytkowników.
