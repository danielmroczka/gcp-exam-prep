You are loading CSV files from Cloud Storage to BigQuery. The files have known data quality issues, including mismatched data types, such as STRINGs and
INT64s in the same column, and inconsistent formatting of values such as phone numbers or addresses. You need to create the data pipeline to maintain data quality and perform the required cleansing and transformation. What should you do?

  • A. Use Data Fusion to transform the data before loading it into BigQuery. Most Voted
  • B. Use Data Fusion to convert the CSV files to a self-describing data format, such as AVRO, before loading the data to BigQuery.
  • C. Load the CSV files into a staging table with the desired schema, perform the transformations with SQL, and then write the results to the final destination table.
  • D. Create a table with the desired schema, load the CSV files into the table, and perform the transformations in place using SQL.

Odpowiedź: A.

Data Fusion jest narzędziem ETL (Extract, Transform, Load) zaprojektowanym do rozwiązywania problemów z jakością danych i przeprowadzania złożonych transformacji. W tym scenariuszu, gdzie mamy do czynienia z różnorodnymi problemami, takimi jak niezgodność typów danych i formatowania, Data Fusion oferuje wizualny interfejs i szeroki zakres wbudowanych transformacji, które pozwalają na efektywne oczyszczenie i przekształcenie danych przed załadowaniem ich do BigQuery. Pozwala to na zapewnienie, że dane w BigQuery będą spójne i zgodne z oczekiwanym schematem. Data Fusion jest szczególnie przydatne, gdy transformacje są złożone i wymagają wielu kroków. Użycie Data Fusion minimalizuje ryzyko błędów i upraszcza proces utrzymania jakości danych.

A. Data Fusion jest najlepszym rozwiązaniem, ponieważ oferuje kompleksowe możliwości transformacji danych.
B. Konwersja do Avro sama w sobie nie rozwiązuje problemów z jakością danych, a jedynie zmienia format.
C. Transformacje SQL w tabeli staging są mniej elastyczne i trudniejsze w utrzymaniu dla złożonych transformacji.
D. Transformacje "in-place" mogą być ryzykowne i trudne do odwrócenia w przypadku błędów.

Dodatkowe źródła:

*   **Google Cloud Data Fusion:** https://cloud.google.com/data-fusion
*   **BigQuery Data Loading:** https://cloud.google.com/bigquery/docs/loading-data

Analiza i Wyjaśnienie

Sugerowana odpowiedź: A

Odpowiedź A jest najbardziej adekwatna do przedstawionego problemu. Pytanie dotyczy stworzenia pipeline do oczyszczania i transformacji danych podczas ładowania ich z Cloud Storage do BigQuery. Data Fusion jest narzędziem ETL zaprojektowanym specjalnie do tego celu. Umożliwia wizualne tworzenie potoków danych i oferuje szeroki zakres wbudowanych transformacji, które pozwalają na efektywne radzenie sobie z problemami takimi jak niezgodność typów danych i formatowania.

Dyskusja użytkowników potwierdza tę opinię. Wiele osób wskazuje na Data Fusion jako najlepsze rozwiązanie ze względu na jego funkcjonalność i możliwość budowy kompleksowych pipeline'ów. Użytkownicy, tacy jak MaxNRG, szczegółowo opisują zalety Data Fusion, w tym wizualny interfejs, wbudowane transformacje i skalowalność. Linki udostępnione przez użytkowników (np. https://cloud.google.com/data-fusion/docs/concepts/overview) również potwierdzają, że Data Fusion jest narzędziem przeznaczonym do tego typu zadań.

Dlaczego inne odpowiedzi są mniej odpowiednie:

  • B: Konwersja do Avro sama w sobie nie rozwiązuje problemów z jakością danych. Zmienia jedynie format, a transformacje nadal byłyby konieczne.
  • C: Użycie tabeli staging i transformacji SQL jest mniej elastyczne i trudniejsze w utrzymaniu dla złożonych transformacji. Chociaż jest to często stosowane podejście, nie jest tak efektywne jak Data Fusion w przypadku złożonych problemów z jakością danych.
  • D: Transformacje "in-place" są ryzykowne i trudne do odwrócenia w przypadku błędów. Zawsze bezpieczniej jest zachować surowe dane nienaruszone i wykonywać transformacje oddzielnie.

Chociaż niektórzy użytkownicy (np. squishy_fishy, Adswerve) początkowo wskazywali na odpowiedź C, ostatecznie wielu z nich (w tym sam autor pytania, AWSandeep) potwierdziło, że A jest poprawną odpowiedzią, biorąc pod uwagę wymóg stworzenia pipeline.
