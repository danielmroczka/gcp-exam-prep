You are a BigQuery admin supporting a team of data consumers who run ad hoc queries and downstream reporting in tools such as Looker. All data and users are combined under a single organizational project. You recently noticed some slowness in query results and want to troubleshoot where the slowdowns are occurring. You think that there might be some job queuing or slot contention occurring as users run jobs, which slows down access to results. You need to investigate the query job information and determine where performance is being affected. What should you do?

  • A. Use slot reservations for your project to ensure that you have enough query processing capacity and are able to allocate available slots to the slower queries.
  • B. Use Cloud Monitoring to view BigQuery metrics and set up alerts that let you know when a certain percentage of slots were used.
  • C. Use available administrative resource charts to determine how slots are being used and how jobs are performing over time. Run a query on the INFORMATION_SCHEMA to review query performance. Most Voted
  • D. Use Cloud Logging to determine if any users or downstream consumers are changing or deleting access grants on tagged resources.

Odpowiedź: C.

Opcja C jest poprawna, ponieważ oferuje najbardziej kompleksowe podejście do diagnozowania problemów z wydajnością w BigQuery. Administracyjne wykresy zasobów pozwalają na wizualizację wykorzystania slotów w czasie, co pomaga zidentyfikować okresy szczytowego obciążenia i potencjalną rywalizację o zasoby. Dodatkowo, zapytania do `INFORMATION_SCHEMA` dostarczają szczegółowych danych o wydajności poszczególnych zapytań, takich jak czas trwania, zużycie zasobów i historia wykonania, co pozwala na precyzyjne zlokalizowanie przyczyn spowolnień. Połączenie tych dwóch metod daje pełny obraz sytuacji i umożliwia skuteczne rozwiązywanie problemów.

A. Slot reservations są dobrym rozwiązaniem *po* zdiagnozowaniu problemu, ale nie pomagają w jego identyfikacji.
B. Cloud Monitoring jest przydatny do alertów, ale nie dostarcza szczegółowych informacji o wydajności poszczególnych zapytań.
D. Cloud Logging monitoruje zmiany w uprawnieniach, co jest niezwiązane z problemem spowolnienia zapytań.

Dodatkowe źródła:

*   **BigQuery Monitoring:** https://cloud.google.com/bigquery/docs/monitoring
*   **INFORMATION_SCHEMA:** https://cloud.google.com/bigquery/docs/information-schema-overview
*   **BigQuery Resource Charts:** https://cloud.google.com/bigquery/docs/resource-charts

Sugerowana odpowiedź: C

Odpowiedź C jest zdecydowanie poprawna i najskuteczniejsza w diagnozowaniu problemów z wydajnością zapytań w BigQuery w opisanym scenariuszu. Wykorzystanie administracyjnych wykresów zasobów pozwala na wizualną analizę wykorzystania slotów i wydajności zadań w czasie. To kluczowe do identyfikacji okresów szczytowego obciążenia i potencjalnej rywalizacji o zasoby, czyli dokładnie tego, co sugeruje pytanie. Dodatkowo, zapytania do INFORMATION_SCHEMA dostarczają szczegółowych danych o poszczególnych zapytaniach, takich jak czas wykonania, zużycie slotów i historia wykonania, co pozwala na precyzyjne zlokalizowanie przyczyn spowolnień.
