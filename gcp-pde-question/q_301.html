You are architecting a data transformation solution for BigQuery. Your developers are proficient with SQL and want to use the ELT development technique. In addition, your developers need an intuitive coding environment and the ability to manage SQL as code. You need to identify a solution for your developers to build these pipelines. What should you do?

  • A. Use Dataform to build, manage, and schedule SQL pipelines. Most Voted
  • B. Use Dataflow jobs to read data from Pub/Sub, transform the data, and load the data to BigQuery.
  • C. Use Data Fusion to build and execute ETL pipelines.
  • D. Use Cloud Composer to load data and run SQL pipelines by using the BigQuery job operators.

Odpowiedź: A.

Dataform jest idealnym rozwiązaniem dla tego scenariusza, ponieważ umożliwia budowanie, zarządzanie i planowanie potoków SQL. Jest zaprojektowany specjalnie dla technik ELT, gdzie transformacje odbywają się w BigQuery, co odpowiada preferencjom programistów znających SQL. Dataform pozwala na traktowanie SQL jako kodu, co ułatwia wersjonowanie, testowanie i ponowne wykorzystanie kodu. Dodatkowo, oferuje intuicyjne środowisko programistyczne, które zwiększa produktywność.  Umożliwia również zarządzanie zależnościami między transformacjami, co jest kluczowe w bardziej złożonych potokach danych.

A. Dataflow jest bardziej odpowiedni do złożonych transformacji danych, które nie mogą być łatwo wyrażone w SQL i wymagają przetwarzania strumieniowego.
B. Data Fusion jest narzędziem graficznym do budowania potoków ETL, co nie odpowiada preferencjom programistów znających SQL i potrzebie zarządzania SQL jako kodem.
C. Cloud Composer jest narzędziem do orkiestracji, a nie do budowania i zarządzania transformacjami danych, chociaż może uruchamiać zadania BigQuery.
D. Cloud Composer jest narzędziem do orkiestracji, a nie do budowania i zarządzania transformacjami danych, chociaż może uruchamiać zadania BigQuery.

Analiza i Wyjaśnienie

Sugerowana odpowiedź: A

Odpowiedź A, czyli użycie Dataform, jest zdecydowanie najbardziej odpowiednią w tym scenariuszu. Pytanie precyzyjnie określa, że programiści preferują SQL i technikę ELT (Extract, Load, Transform). Dataform został zaprojektowany specjalnie do budowania, zarządzania i planowania potoków danych w BigQuery przy użyciu SQL. Umożliwia traktowanie SQL jako kodu, co jest kluczowe dla wersjonowania, testowania i współpracy. Dodatkowo, Dataform oferuje intuicyjne środowisko programistyczne, co jest istotne dla produktywności zespołu.

Komentarze użytkowników (raaad, Pime13, JyoGCP, Matt_108, scaenruy, chickenwingz) jednogłośnie potwierdzają, że Dataform jest najlepszym rozwiązaniem, podkreślając jego zgodność z ELT, możliwość zarządzania SQL jako kodem, intuicyjne środowisko oraz funkcje planowania i orkiestracji.

Pozostałe odpowiedzi są mniej trafne:

  • B (Dataflow): Dataflow jest bardziej odpowiedni do złożonych transformacji danych, które nie mogą być łatwo wyrażone w SQL i wymagają przetwarzania strumieniowego.
  • C (Data Fusion): Data Fusion to narzędzie graficzne do budowania potoków ETL, co nie odpowiada preferencjom programistów znających SQL i potrzebie zarządzania SQL jako kodem.
  • D (Cloud Composer): Cloud Composer to narzędzie do orkiestracji, a nie do budowania i zarządzania transformacjami danych, chociaż może uruchamiać zadania BigQuery.

Podsumowując, Dataform idealnie spełnia wszystkie wymagania określone w pytaniu, co czyni go najwłaściwszym wyborem.
