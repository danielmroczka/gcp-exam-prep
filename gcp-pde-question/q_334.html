You have created an external table in BigQuery for Apache Hive partitioned data that resides in a Cloud Storage bucket, which contains a large number of files. You notice that queries against this table are slow. You want to improve the performance of these queries. What should you do?

  • A. Migrate the Hive partitioned data objects to a multi-region Cloud Storage bucket.
  • B. Create an individual external table for each Hive partition and use wildcard table queries.
  • C. Change the storage class of the Hive partitioned data objects from Coldline to Standard.
  • D. Upgrade the external table to a BigLake table and enable metadata caching for the table.
Show Suggested Answer
Suggested Answer: D

Wyjaśnienie

Correct Answer: D

BigLake tables allow BigQuery to query data in external stores (like GCS) with better performance and more control. By enabling metadata caching on a BigLake table, BigQuery avoids scanning the GCS bucket for file listings every time a query is run. This significantly speeds up queries on tables with a large number of files.

Why other options are incorrect:
A. Moving to a multi-region bucket doesn't address the metadata bottleneck of listing many files.
B. Creating many individual tables is difficult to manage and wildcard queries can still be slow if there are many files.
C. Standard storage class improves access speed slightly, but the main bottleneck for external tables is often the metadata discovery (file listing), not the data retrieval itself.
