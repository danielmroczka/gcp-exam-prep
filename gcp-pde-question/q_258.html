You have several different file type data sources, such as Apache Parquet and CSV. You want to store the data in Cloud Storage. You need to set up an object sink for your data that allows you to use your own encryption keys. You want to use a GUI-based solution. What should you do?

  • A. Use Storage Transfer Service to move files into Cloud Storage.
  • B. Use Cloud Data Fusion to move files into Cloud Storage. Most Voted

  • D. Use BigQuery Data Transfer Service to move files into BigQuery.

Odpowiedź: B.

Cloud Data Fusion oferuje graficzny interfejs użytkownika (GUI) do budowania i zarządzania potokami danych. Pozwala na integrację z różnymi źródłami danych, w tym plikami Parquet i CSV, oraz zapisywanie ich w Cloud Storage. Co ważne, Cloud Data Fusion umożliwia konfigurację szyfrowania po stronie klienta (Customer-Managed Encryption Keys - CMEK) podczas zapisu do Cloud Storage, spełniając wymaganie użycia własnych kluczy szyfrujących. Dzięki GUI, proces ten jest bardziej intuicyjny i dostępny dla użytkowników bez głębokiej wiedzy programistycznej.  Pozostałe opcje nie oferują tak kompleksowego rozwiązania z GUI i łatwą konfiguracją CMEK.

A. Storage Transfer Service skupia się na prostym kopiowaniu danych, ale nie oferuje tak elastycznej konfiguracji szyfrowania.
C. Dataflow wymaga pisania kodu (Python lub Java) i nie jest rozwiązaniem GUI.
D. BigQuery Data Transfer Service przenosi dane *do* BigQuery, a nie bezpośrednio do Cloud Storage.

Dodatkowe źródła:

*   **Cloud Data Fusion Documentation:** https://cloud.google.com/data-fusion/docs
*   **Customer-Managed Encryption Keys (CMEK) in Cloud Storage:** https://cloud.google.com/storage/docs/encryption-keys

Analiza i Wyjaśnienie

Sugerowana odpowiedź: B

Poprawną odpowiedzią jest B. Use Cloud Data Fusion to move files into Cloud Storage. Cloud Data Fusion jest w pełni zarządzaną usługą integracji danych, która oferuje graficzny interfejs użytkownika (GUI) do budowania i zarządzania potokami danych. Kluczowe jest to, że Cloud Data Fusion umożliwia konfigurację szyfrowania po stronie klienta (CMEK) podczas zapisu do Cloud Storage, co jest bezpośrednio wymagane w treści pytania. Użytkownicy, tacy jak raaad, AllenChen123, Helinia, vishavpreet i hussain.sain, poprawnie wskazali tę odpowiedź, a Helinia dodatkowo podkreśliła brak wsparcia dla CMEK w Storage Transfer Service.

Dyskusja wskazuje na pewne wątpliwości dotyczące użycia narzędzia ETL (Extract, Transform, Load) takiego jak Data Fusion do prostego transferu danych. Jednakże, pytanie wyraźnie wymaga możliwości użycia własnych kluczy szyfrujących, co eliminuje inne opcje. Użytkownik baimus słusznie zauważa, że teoretycznie można utworzyć bucket z CMEK i użyć Storage Transfer Service, ale pytanie koncentruje się na rozwiązaniu, które *umożliwia* konfigurację szyfrowania, a Data Fusion to robi w sposób zintegrowany i GUI-based.

Dlaczego inne odpowiedzi są niepoprawne:

  • A. Use Storage Transfer Service to move files into Cloud Storage: Storage Transfer Service jest GUI-based, ale, jak podkreśla Helinia i ashdam, nie obsługuje CMEK.
  • C. Use Dataflow to move files into Cloud Storage: Dataflow wymaga pisania kodu i nie jest rozwiązaniem GUI.
  • D. Use BigQuery Data Transfer Service to move files into BigQuery: Ta usługa przenosi dane do BigQuery, a nie do Cloud Storage.

Najnowsze komentarze (np. od aaaaaaaasdasdasfs) próbują argumentować za odpowiedzią A, ale są one błędne, ponieważ ignorują kluczowe wymaganie dotyczące CMEK. Większość głosów i nowsze komentarze potwierdzają poprawność odpowiedzi B.
