You are working on a linear regression model on BigQuery ML to predict a customer's likelihood of purchasing your company's products. Your model uses a city name variable as a key predictive component. In order to train and serve the model, your data must be organized in columns. You want to prepare your data using the least amount of coding while maintaining the predictable variables. What should you do?

  • A. Create a new view with BigQuery that does not include a column with city information.
  • B. Use SQL in BigQuery to transform the state column using a one-hot encoding method, and make each city a column with binary values. Most Voted
  • C. Use TensorFlow to create a categorical variable with a vocabulary list. Create the vocabulary file and upload that as part of your model to BigQuery ML.
  • D. Use Cloud Data Fusion to assign each city to a region that is labeled as 1, 2, 3, 4, or 5, and then use that number to represent the city in the model.

Odpowiedź: B.

BigQuery ML najlepiej radzi sobie z danymi w formacie kolumnowym. Aby wykorzystać zmienną "nazwa miasta" jako predyktor w modelu regresji liniowej, należy przekształcić ją na zmienne numeryczne. One-hot encoding jest idealnym rozwiązaniem, ponieważ tworzy oddzielną kolumnę binarną dla każdej unikalnej wartości miasta, co pozwala modelowi efektywnie uwzględnić wpływ lokalizacji. Użycie SQL w BigQuery do wykonania tego przekształcenia jest najmniej czasochłonne i nie wymaga pisania kodu w innych językach programowania. To podejście jest zgodne z wymaganiami BigQuery ML dotyczącymi formatu danych i pozwala na bezpośrednie wykorzystanie przekształconych danych w procesie trenowania i serwowania modelu.

A. Usunięcie informacji o mieście całkowicie uniemożliwiłoby wykorzystanie tej ważnej zmiennej w modelu.
C. TensorFlow jest bardziej skomplikowany i wymaga dodatkowego kodu oraz zarządzania plikami słownika, co jest sprzeczne z celem minimalizacji kodowania.
D. Przypisanie miast do regionów wprowadza arbitralne uproszczenie i może prowadzić do utraty istotnych informacji o wpływie konkretnych miast na prawdopodobieństwo zakupu.

Analiza i Wyjaśnienie

Sugerowana odpowiedź: B

Poprawną odpowiedzią jest B. Pytanie dotyczy efektywnego przygotowania danych kategorycznych (nazw miast) do modelu regresji liniowej w BigQuery ML, przy minimalizacji ilości kodu. One-hot encoding jest idealnym rozwiązaniem, ponieważ przekształca każdą unikalną nazwę miasta na oddzielną kolumnę binarną (0 lub 1), co pozwala modelowi uwzględnić wpływ lokalizacji. BigQuery SQL umożliwia łatwe wykonanie tego przekształcenia, co jest zgodne z wymaganiem minimalizacji kodowania.

Dyskusja użytkowników potwierdza tę odpowiedź. Wielu użytkowników (MaxNRG, barnac1es, blathul, zellck, ovokpus) wskazuje, że one-hot encoding jest standardową techniką dla danych kategorycznych i może być łatwo zaimplementowany w BigQuery za pomocą SQL. Użytkownik MaxNRG podaje nawet link do dokumentacji BigQuery ML potwierdzający tę metodę.

Dlaczego inne odpowiedzi są nieprawidłowe:

  • A: Usunięcie informacji o mieście pozbawia model ważnej zmiennej predykcyjnej.
  • C: Użycie TensorFlow jest bardziej skomplikowane i wymaga dodatkowego kodu, co jest sprzeczne z celem minimalizacji kodowania.
  • D: Przypisanie miast do regionów wprowadza uproszczenie i może prowadzić do utraty istotnych informacji. Jak zauważają użytkownicy (cetanx, tavva_prudhvi), grupowanie miast w regiony może ukryć różnice w zachowaniach zakupowych.

Chociaż niektórzy użytkownicy (np. clouditis, leandrors, Atnafu) początkowo wskazywali odpowiedź D, argumentując minimalizacją kodowania poprzez użycie Cloud Data Fusion, większość dyskusji i argumentów wskazuje, że one-hot encoding w BigQuery SQL jest bardziej efektywnym i precyzyjnym rozwiązaniem.

Należy również zwrócić uwagę na komentarze wskazujące na błąd w treści pytania (cajica, sergiomujica, knith66), sugerujący, że powinno być "city" zamiast "state" w opcji B. Zakładając, że jest to błąd w transkrypcji, odpowiedź B staje się jeszcze bardziej oczywista.
