Question 92 / 201

You are developing the deployment and testing strategies for your CI/CD pipeline in Google Cloud. You must be able to: • Reduce the complexity of release deployments and minimize the duration of deployment rollbacks. • Test real production traffic with a gradual increase in the number of affected users. You want to select a deployment and testing strategy that meets your requirements. What should you do?

  • A. Recreate deployment and canary testing
  • B. Blue/green deployment and canary testing
  • C. Rolling update deployment and A/B testing
  • D. Rolling update deployment and shadow testing
Show Suggested Answer
Suggested Answer: B 
Odpowiedź: D.

Wyjaśnienie: Opisany atak (wykonywanie komend w przeglądarce ofiary) to Cross-Site Scripting (XSS). Aby go wykryć i naprawić:
1. Używamy Web Security Scanner (część SCC) w środowisku stagingowym, aby automatycznie znaleźć miejsca podatne na XSS (identyfikacja).
2. Stosujemy "contextual auto-escaping" w silniku szablonów aplikacji (np. Jinja2, Go Templates), co zapewnia, że dane od użytkownika są traktowane jako tekst, a nie jako kod wykonywalny (naprawa).

A. IAP chroni przed nieuprawnionym dostępem, ale nie naprawia błędów w kodzie samej aplikacji.
B. Cloud Armor może blokować ataki, ale nie "naprawia" (fix) podatności u źródła.
C. Sprawdzanie bibliotek jest dobre, ale XSS często wynika z logiki wyświetlania danych przez własny kod dewelopera.

Analiza i Wyjaśnienie

Zapewnienie bezpieczeństwa aplikacji webowych wymaga narzędzi DAST (Dynamic Application Security Testing). Web Security Scanner jest natywnym rozwiązaniem GCP dla tego zadania.
