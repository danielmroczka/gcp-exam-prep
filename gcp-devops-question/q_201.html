Question 201 / 201

Cloud Build

You are managing a CI/CD pipeline that uses Cloud Build to deploy applications to multiple environments. You need to ensure that only approved changes are deployed to production. What should you do?

  • A. Use Cloud Build triggers with branch protection rules.
  • B. Use Binary Authorization with Cloud Build.
  • C. Use Cloud Deploy with approval gates.
  • D. Use Cloud Functions to manually approve each deployment.
Show Suggested Answer
Suggested Answer: C 
Odpowiedź: A.

Wyjaśnienie: Ransomware polega na zaszyfrowaniu lub usunięciu danych. Funkcja "Object Versioning" sprawia, że każda próba nadpisania czy usunięcia pliku tworzy nową wersję, zachowując oryginał. Funkcja "Bucket Lock" (w trybie retencji) uniemożliwia usunięcie tych wersji przez określony czas, nawet przez hakera z uprawnieniami administratora. To zapewnia niezmienność danych (Immutability) i możliwość ich odzyskania po ataku.

B. CMEK szyfruje dane, ale nie chroni przed ich złośliwym skasowaniem lub ponownym zaszyfrowaniem przez hakerów.
C. VPC Service Controls chroni przed kradzieżą danych (wyciekiem), ale nie przed ich zniszczeniem wewnątrz projektu.
D. Cloud Armor chroni przed atakami sieciowymi na aplikacje WWW, nie chroni plików w storage przed manipulacją przez przejęte konto serwisowe.

Analiza i Wyjaśnienie

Strategia Anti-Ransomware w chmurze opiera się na Immutable Storage. Połączenie wersjonowania i blokady bucketu (WORM - Write Once, Read Many) jest jedyną skuteczną obroną przed destrukcyjnym działaniem złośliwego oprogramowania.
