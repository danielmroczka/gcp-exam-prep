Question 194 / 201

You are developing a Node.js utility on a workstation in Cloud Workstations by using Code OSS. The utility is a simple web page, and you have already confirmed that all necessary firewall rules are in place. You tested the application by starting it on port 3000 on your workstation in Cloud Workstations, but you need to be able to access the web page from your local machine. You need to follow Google-recommended security practices. What should you do?

  • A. Use a browser running on a bastion host VM.
  • B. Run the gcloud compute start-iap-tunnel command to the Cloud Workstations VM.
  • C. Allow public IP addresses in the Cloud Workstations configuration.
  • D. Click the preview link in the Code OSS panel.
Show Suggested Answer
Suggested Answer: D 
Odpowiedź: A.

Wyjaśnienie: Policy Analyzer jest jedynym narzędziem w Google Cloud, które potrafi "rozwinąć" strukturę dziedziczenia IAM. Potrafi on przeanalizować całe drzewo od Organizacji, przez Foldery, aż po Projekt, i pokazać, że użytkownik ma dostęp do danej tabeli BigQuery, bo odziedziczył rolę np. z poziomu działu (Folderu).

B. Policy Troubleshooter sprawdza tylko jeden, konkretny przypadek "czy X ma dostęp do Y".
C. IAM Recommender sugeruje usuwanie ról, których nikt nie używa.
D. Audit Logs rejestrują fakt nadania roli, ale nie analizują logicznej struktury dziedziczenia wszystkich ról w czasie rzeczywistym.

Analiza i Wyjaśnienie

Audytorzy często pytają: "Kto MA dostęp?", a nie "Kto DOSTAŁ dostęp?". Policy Analyzer dostarcza odpowiedzi na to pierwsze, znacznie trudniejsze pytanie, biorąc pod uwagę złożoność dziedziczenia w GCP.
