Question 162 / 201

You are configuring a CI pipeline. The build step for your CI pipeline integration testing requires access to APIs inside your private VPC network. Your security team requires that you do not expose API traffic publicly. You need to implement a solution that minimizes management overhead. What should you do?

  • A. Use Cloud Build private pools to connect to the private VPC.
  • B. Use Spinnaker for Google Cloud to connect to the private VPC.
  • C. Use Cloud Build as a pipeline runner. Configure Internal HTTP(S) Load Balancing for API access.
  • D. Use Cloud Build as a pipeline runner. Configure External HTTP(S) Load Balancing with a Google Cloud Armor policy for API access.
Show Suggested Answer
Suggested Answer: A 
Odpowiedź: C.

Wyjaśnienie: Fizyczne klucze bezpieczeństwa (Security Keys) oparte na standardzie FIDO2/WebAuthn są jedyną metodą uwierzytelniania dwuskładnikowego całkowicie odporną na phishing. W przeciwieństwie do kodów SMS czy aplikacji, klucz sprzętowy weryfikuje domenę strony logowania, co uniemożliwia hakerowi przechwycenie poświadczeń na podrobionej stronie.

A, B i D. Kody przesyłane przez SMS, e-mail lub generowane w aplikacji mogą zostać wyłudzone przez hakerów za pomocą socjotechniki i wpisane na fałszywej stronie logowania w czasie rzeczywistym.

Analiza i Wyjaśnienie

Google przeprowadziło badanie na własnych pracownikach: od momentu wymuszenia kluczy bezpieczeństwa (Titan), liczba udanych ataków phishingowych spadła do zera. Jest to obecnie najsilniejszy standard ochrony kont.
