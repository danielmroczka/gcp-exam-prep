Question 69 / 201

You manage several production systems that run on Compute Engine in the same Google Cloud Platform (GCP) project. Each system has its own set of dedicated Compute Engine instances. You want to know how must it costs to run each of the systems. What should you do?

  • A. In the Google Cloud Platform Console, use the Cost Breakdown section to visualize the costs per system.
  • B. Assign all instances a label specific to the system they run. Configure BigQuery billing export and query costs per label.
  • C. Enrich all instances with metadata specific to the system they run. Configure Stackdriver Logging to export to BigQuery, and query costs based on the metadata.
  • D. Name each virtual machine (VM) after the system it runs. Set up a usage report export to a Cloud Storage bucket. Configure the bucket as a source in BigQuery to query costs based on VM name.
Show Suggested Answer
Suggested Answer: B 
Odpowiedź: A.

Wyjaśnienie: Zgodnie z opisem w pytaniu 68, GCDS jest standardowym mostem łączącym on-premises directory z chmurą Google. Pozwala zachować istniejący katalog jako jedyne źródło prawdy (Source of Truth), przesyłając dane o tożsamościach w jedną stronę (do Google).

B. Cloud Identity to nazwa samej usługi tożsamości, a nie narzędzia łączącego.
C. SAML służy do logowania (authentication), ale nie do automatycznego tworzenia i usuwania kont (provisioning). Do pełnego wdrożenia potrzebujesz zazwyczaj i GCDS (konta) i SAML (hasła).
D. Pub/Sub to usługa przesyłania wiadomości, nie ma związku z zarządzaniem kontami użytkowników.

Analiza i Wyjaśnienie

W dużych firmach synchronizacja kont (Provisioning) jest równie ważna co logowanie (SSO). Bez GCDS administratorzy musieliby ręcznie zakładać konta w Google dla każdego nowego pracownika.
