Question 1 / 201

GKE Kubernetes

You support a Node.js application running on Google Kubernetes Engine (GKE) in production. The application makes several HTTP requests to dependent applications. You want to anticipate which dependent applications might cause performance issues. What should you do?

  • A. Instrument all applications with Stackdriver Profiler.
  • B. Instrument all applications with Stackdriver Trace and review inter-service HTTP requests.
  • C. Use Stackdriver Debugger to review the execution of logic within each application to instrument all applications.
  • D. Modify the Node.js application to log HTTP request and response times to dependent applications. Use Stackdriver Logging to find dependent applications that are performing poorly.
Show Suggested Answer
Suggested Answer: B 
Odpowiedź: B.

Wyjaśnienie: Stackdriver Trace (obecnie Cloud Trace) jest narzędziem przeznaczonym do profilowania latencji w systemach rozproszonych. Pozwala on na śledzenie drogi żądania przez wiele mikroserwisów, co jest kluczowe w architekturze GKE. Dzięki niemu można dokładnie zobaczyć, ile czasu zajmuje każde wywołanie HTTP między usługami i zidentyfikować "wąskie gardła".

A. Stackdriver Profiler (Cloud Profiler) służy do analizy zużycia zasobów (CPU, pamięć) wewnątrz pojedynczej aplikacji, a nie do śledzenia komunikacji między nimi.
C. Stackdriver Debugger pozwala na inspekcję stanu działającej aplikacji bez jej zatrzymywania, ale nie służy do monitorowania wydajności komunikacji sieciowej.
D. Ręczne logowanie czasów jest czasochłonne i mniej efektywne niż gotowe rozwiązanie typu Trace, które dostarcza wizualizację całego łańcucha wywołań.

Analiza i Wyjaśnienie

W środowisku mikroserwisowym na GKE komunikacja między usługami jest kluczowym elementem wpływającym na wydajność. Cloud Trace jest standardowym rozwiązaniem Google Cloud do rozwiązywania problemów z opóźnieniami (latencją) rozproszoną.

  • Trace: Dostarcza wykresy "waterfall", pokazujące czas trwania każdego skoku (hop) żądania.
  • Profiler: Optymalizuje koszt wykonania kodu (np. znajduje funkcje obciążające CPU).
  • Logging: Jest zbyt generyczne – analiza milionów logów w poszukiwaniu opóźnień bez korelacji (którą daje Trace) jest nieefektywna.
