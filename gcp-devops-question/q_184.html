Question 184 / 201

GKE

You manage a retail website for your company. The website consists of several microservices running in a GKE Standard node pool with node autoscaling enabled. Each microservice has resource limits and a Horizontal Pod Autoscaler configured. During a busy period, you receive alerts for one of the microservices. When you check the Pods, half of them have the status OOMKilled, and the number of Pods is at the minimum autoscaling limit. You need to resolve the issue. What should you do?

  • A. Update the node pool to use a machine type with more memory.
  • B. Increase the maximum number of nodes in the node pool.
  • C. Increase the maximum replica limit of the Horizontal Pod Autoscaler.
  • D. Increase the memory resource limit of the microservice.
Show Suggested Answer
Suggested Answer: D 
Odpowiedź: B.

Wyjaśnienie: Komunikacja między usługami wewnątrz GCP (Server-to-service) powinna opierać się na tożsamościach przypisanych do zasobów obliczeniowych (Instance Service Account). Biblioteki Google (Application Default Credentials) automatycznie pobierają tokeny z serwera metadanych instancji, co eliminuje potrzebę stosowania jakichkolwiek haseł czy kluczy statycznych.

A. Klucze w zmiennych środowiskowych to ryzyko wycieku przez logi lub błędy aplikacji.
C. Używanie haseł użytkowników dla komunikacji systemowej jest niezgodne z architekturą chmurową.
D. Klucze API są przeznaczone dla publicznych usług (np. Maps API) i nie zapewniają bezpiecznej autoryzacji IAM dla usług wewnętrznych.

Analiza i Wyjaśnienie

Application Default Credentials (ADC) to mechanizm, który każdy inżynier DevOps musi znać. Pozwala on na pisanie kodu, który działa tak samo na lokalnym laptopie i w chmurze, bez żadnych haseł w konfiguracji.
